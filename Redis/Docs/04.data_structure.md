## String Commands

Strings are the most basic Redis data type, representing a sequence of bytes. They can store text, serialized objects, or binary data up to 512MB.

### SET
**Description:** Sets a key to hold a string value. If the key already exists, it overwrites the value.

**When to use:** Storing user sessions, caching API responses, configuration values.

**Syntax:**
```bash
SET key value [EX seconds] [PX milliseconds] [NX|XX]
```

**Example:**
```bash
SET user:1000:name "John Doe"
# Output: OK

SET session:abc123 "user_data" EX 3600
# Output: OK (expires in 1 hour)
```

---

### GET
**Description:** Retrieves the value of a key. Returns nil if the key doesn't exist.

**When to use:** Fetching cached data, retrieving user information.

**Syntax:**
```bash
GET key
```

**Example:**
```bash
GET user:1000:name
# Output: "John Doe"

GET nonexistent_key
# Output: (nil)
```

---

### APPEND
**Description:** Appends a value to the end of an existing string. If the key doesn't exist, it creates a new string.

**When to use:** Building log entries, concatenating text data.

**Syntax:**
```bash
APPEND key value
```

**Example:**
```bash
SET message "Hello"
APPEND message " World"
GET message
# Output: "Hello World"
```

---

### INCR
**Description:** Increments the integer value of a key by one. Creates the key if it doesn't exist (starts at 0).

**When to use:** Counters, view counts, rate limiting.

**Syntax:**
```bash
INCR key
```

**Example:**
```bash
INCR page:views
# Output: 1
INCR page:views
# Output: 2
```

---

### DECR
**Description:** Decrements the integer value of a key by one.

**When to use:** Countdown timers, inventory management.

**Syntax:**
```bash
DECR key
```

**Example:**
```bash
SET stock:item123 50
DECR stock:item123
# Output: 49
```

---

### INCRBY
**Description:** Increments the integer value of a key by a specified amount.

**When to use:** Adding points, bulk counter updates.

**Syntax:**
```bash
INCRBY key increment
```

**Example:**
```bash
INCRBY user:1000:points 100
# Output: 100
```

---

### DECRBY
**Description:** Decrements the integer value of a key by a specified amount.

**When to use:** Deducting points, bulk inventory reduction.

**Syntax:**
```bash
DECRBY key decrement
```

**Example:**
```bash
SET wallet:balance 1000
DECRBY wallet:balance 250
# Output: 750
```

---

### INCRBYFLOAT
**Description:** Increments the float value of a key by a specified floating-point amount.

**When to use:** Financial calculations, precise measurements.

**Syntax:**
```bash
INCRBYFLOAT key increment
```

**Example:**
```bash
SET temperature 20.5
INCRBYFLOAT temperature 1.3
# Output: "21.8"
```

---

### GETSET
**Description:** Sets a new value and returns the old value atomically.

**When to use:** Atomic swap operations, rotating tokens.

**Syntax:**
```bash
GETSET key value
```

**Example:**
```bash
SET current_leader "Alice"
GETSET current_leader "Bob"
# Output: "Alice"
```

---

### MGET
**Description:** Retrieves the values of multiple keys in a single operation.

**When to use:** Batch retrieval, reducing network round trips.

**Syntax:**
```bash
MGET key1 key2 key3 ...
```

**Example:**
```bash
MGET user:1:name user:2:name user:3:name
# Output: 
# 1) "John"
# 2) "Jane"
# 3) "Bob"
```

---

### MSET
**Description:** Sets multiple key-value pairs in a single atomic operation.

**When to use:** Batch updates, initializing multiple values.

**Syntax:**
```bash
MSET key1 value1 key2 value2 ...
```

**Example:**
```bash
MSET user:1:name "Alice" user:1:email "alice@example.com" user:1:age "25"
# Output: OK
```

---

### STRLEN
**Description:** Returns the length of the string value stored at a key.

**When to use:** Validation, checking data size.

**Syntax:**
```bash
STRLEN key
```

**Example:**
```bash
SET description "Redis is fast"
STRLEN description
# Output: 13
```

---

### SETEX
**Description:** Sets a key with a value and expiration time in seconds.

**When to use:** Temporary data, session tokens, OTP codes.

**Syntax:**
```bash
SETEX key seconds value
```

**Example:**
```bash
SETEX otp:user123 300 "123456"
# Output: OK (expires in 5 minutes)
```

---

### SETNX
**Description:** Sets a key only if it doesn't already exist (SET if Not eXists).

**When to use:** Distributed locks, preventing duplicate entries.

**Syntax:**
```bash
SETNX key value
```

**Example:**
```bash
SETNX lock:resource1 "locked"
# Output: 1 (success)
SETNX lock:resource1 "locked"
# Output: 0 (key already exists)
```

---

### GETRANGE
**Description:** Returns a substring of the string stored at a key.

**When to use:** Extracting portions of text, pagination.

**Syntax:**
```bash
GETRANGE key start end
```

**Example:**
```bash
SET bio "Hello World from Redis"
GETRANGE bio 0 4
# Output: "Hello"
```

---

### SETRANGE
**Description:** Overwrites part of a string at a specified offset.

**When to use:** Partial updates, binary data manipulation.

**Syntax:**
```bash
SETRANGE key offset value
```

**Example:**
```bash
SET message "Hello World"
SETRANGE message 6 "Redis"
GET message
# Output: "Hello Redis"
```

## List Commands

Lists are ordered collections of strings, sorted by insertion order. They support operations at both ends.

### LPUSH
**Description:** Inserts one or more values at the head (left) of a list.

**When to use:** Activity feeds, task queues, recent items.

**Syntax:**
```bash
LPUSH key value1 [value2 ...]
```

**Example:**
```bash
LPUSH notifications "New message" "Friend request"
# Output: 2 (list length)
```

---

### RPUSH
**Description:** Inserts one or more values at the tail (right) of a list.

**When to use:** Job queues, logs, chronological data.

**Syntax:**
```bash
RPUSH key value1 [value2 ...]
```

**Example:**
```bash
RPUSH tasks "task1" "task2" "task3"
# Output: 3
```

---

### LPOP
**Description:** Removes and returns the first element from the list.

**When to use:** Processing queue items, stack operations.

**Syntax:**
```bash
LPOP key [count]
```

**Example:**
```bash
RPUSH queue "job1" "job2" "job3"
LPOP queue
# Output: "job1"
```

---

### RPOP
**Description:** Removes and returns the last element from the list.

**When to use:** FIFO queue processing, removing oldest items.

**Syntax:**
```bash
RPOP key [count]
```

**Example:**
```bash
RPUSH messages "msg1" "msg2" "msg3"
RPOP messages
# Output: "msg3"
```

---

### LRANGE
**Description:** Returns a range of elements from a list.

**When to use:** Pagination, displaying recent items.

**Syntax:**
```bash
LRANGE key start stop
```

**Example:**
```bash
RPUSH posts "post1" "post2" "post3" "post4" "post5"
LRANGE posts 0 2
# Output:
# 1) "post1"
# 2) "post2"
# 3) "post3"

LRANGE posts 0 -1  # Get all elements
```

---

### LLEN
**Description:** Returns the length of a list.

**When to use:** Checking queue size, pagination metadata.

**Syntax:**
```bash
LLEN key
```

**Example:**
```bash
RPUSH items "a" "b" "c"
LLEN items
# Output: 3
```

---

### LINDEX
**Description:** Returns the element at a specific index in a list.

**When to use:** Random access to list elements.

**Syntax:**
```bash
LINDEX key index
```

**Example:**
```bash
RPUSH colors "red" "green" "blue"
LINDEX colors 1
# Output: "green"
```

---

### LSET
**Description:** Sets the value of an element at a specific index.

**When to use:** Updating specific list items.

**Syntax:**
```bash
LSET key index value
```

**Example:**
```bash
RPUSH fruits "apple" "banana" "cherry"
LSET fruits 1 "blueberry"
LRANGE fruits 0 -1
# Output: ["apple", "blueberry", "cherry"]
```

---

### LINSERT
**Description:** Inserts a value before or after an existing element.

**When to use:** Inserting items at specific positions.

**Syntax:**
```bash
LINSERT key BEFORE|AFTER pivot value
```

**Example:**
```bash
RPUSH mylist "Hello" "World"
LINSERT mylist BEFORE "World" "Beautiful"
# Output: 3
LRANGE mylist 0 -1
# Output: ["Hello", "Beautiful", "World"]
```

---

### LREM
**Description:** Removes elements from a list that match a value.

**When to use:** Filtering list items, removing duplicates.

**Syntax:**
```bash
LREM key count value
```

**Example:**
```bash
RPUSH tags "redis" "database" "redis" "cache" "redis"
LREM tags 2 "redis"  # Remove first 2 occurrences
# Output: 2
LRANGE tags 0 -1
# Output: ["database", "cache", "redis"]
```

---

### LTRIM
**Description:** Trims a list to a specified range, removing all elements outside.

**When to use:** Maintaining fixed-size lists, keeping recent N items.

**Syntax:**
```bash
LTRIM key start stop
```

**Example:**
```bash
RPUSH recent "item1" "item2" "item3" "item4" "item5"
LTRIM recent 0 2  # Keep only first 3 items
LRANGE recent 0 -1
# Output: ["item1", "item2", "item3"]
```

---

### BLPOP
**Description:** Blocking version of LPOP. Waits until an element is available or timeout occurs.

**When to use:** Worker queues, real-time job processing.

**Syntax:**
```bash
BLPOP key [key ...] timeout
```

**Example:**
```bash
BLPOP task_queue 30  # Wait up to 30 seconds
# Blocks until an item is pushed to task_queue
```

---

### BRPOP
**Description:** Blocking version of RPOP.

**When to use:** Blocking queue consumers.

**Syntax:**
```bash
BRPOP key [key ...] timeout
```

**Example:**
```bash
BRPOP job_queue 0  # Wait indefinitely
```

---

### RPOPLPUSH
**Description:** Atomically removes the last element from one list and pushes it to another.

**When to use:** Reliable queue processing, moving items between stages.

**Syntax:**
```bash
RPOPLPUSH source destination
```

**Example:**
```bash
RPUSH pending "task1" "task2"
RPOPLPUSH pending processing
# Output: "task2"
# pending: ["task1"]
# processing: ["task2"]
```

---

### BRPOPLPUSH
**Description:** Blocking version of RPOPLPUSH.

**When to use:** Reliable distributed job queues.

**Syntax:**
```bash
BRPOPLPUSH source destination timeout
```

**Example:**
```bash
BRPOPLPUSH incoming processing 10
```

---

### LPOS
**Description:** Returns the index of matching elements in a list.

**When to use:** Finding element positions, searching lists.

**Syntax:**
```bash
LPOS key element [RANK rank] [COUNT num] [MAXLEN len]
```

**Example:**
```bash
RPUSH mylist "a" "b" "c" "b" "d"
LPOS mylist "b"
# Output: 1
LPOS mylist "b" COUNT 2
# Output: [1, 3]
```

## Set Commands

Sets are unordered collections of unique strings. Perfect for tracking unique items and membership testing.

### SADD
**Description:** Adds one or more members to a set.

**When to use:** Tracking unique visitors, tags, permissions.

**Syntax:**
```bash
SADD key member [member ...]
```

**Example:**
```bash
SADD user:1000:tags "developer" "python" "redis"
# Output: 3
SADD user:1000:tags "python"  # Duplicate ignored
# Output: 0
```

---

### SMEMBERS
**Description:** Returns all members of a set.

**When to use:** Listing all unique items, displaying tags.

**Syntax:**
```bash
SMEMBERS key
```

**Example:**
```bash
SADD languages "python" "javascript" "go"
SMEMBERS languages
# Output:
# 1) "python"
# 2) "javascript"
# 3) "go"
```

---

### SISMEMBER
**Description:** Checks if a value is a member of a set.

**When to use:** Permission checks, membership validation.

**Syntax:**
```bash
SISMEMBER key member
```

**Example:**
```bash
SADD premium_users "user123" "user456"
SISMEMBER premium_users "user123"
# Output: 1 (true)
SISMEMBER premium_users "user789"
# Output: 0 (false)
```

---

### SREM
**Description:** Removes one or more members from a set.

**When to use:** Revoking permissions, removing tags.

**Syntax:**
```bash
SREM key member [member ...]
```

**Example:**
```bash
SADD fruits "apple" "banana" "cherry"
SREM fruits "banana"
# Output: 1
SMEMBERS fruits
# Output: ["apple", "cherry"]
```

---

### SCARD
**Description:** Returns the number of members in a set.

**When to use:** Counting unique items, set size validation.

**Syntax:**
```bash
SCARD key
```

**Example:**
```bash
SADD visitors "ip1" "ip2" "ip3"
SCARD visitors
# Output: 3
```

---

### SPOP
**Description:** Removes and returns one or more random members from a set.

**When to use:** Random selection, lottery systems.

**Syntax:**
```bash
SPOP key [count]
```

**Example:**
```bash
SADD deck "A♠" "K♠" "Q♠" "J♠"
SPOP deck
# Output: "K♠" (random)
```

---

### SRANDMEMBER
**Description:** Returns one or more random members without removing them.

**When to use:** Random sampling, showing random content.

**Syntax:**
```bash
SRANDMEMBER key [count]
```

**Example:**
```bash
SADD quotes "quote1" "quote2" "quote3" "quote4"
SRANDMEMBER quotes 2
# Output: ["quote3", "quote1"] (random)
```

---

### SMOVE
**Description:** Moves a member from one set to another.

**When to use:** Transferring items between categories.

**Syntax:**
```bash
SMOVE source destination member
```

**Example:**
```bash
SADD active_users "user1" "user2"
SADD inactive_users "user3"
SMOVE active_users inactive_users "user2"
# Output: 1
```

---

### SUNION
**Description:** Returns the union of multiple sets.

**When to use:** Combining categories, merging groups.

**Syntax:**
```bash
SUNION key [key ...]
```

**Example:**
```bash
SADD team1 "Alice" "Bob" "Charlie"
SADD team2 "Bob" "David" "Eve"
SUNION team1 team2
# Output: ["Alice", "Bob", "Charlie", "David", "Eve"]
```

---

### SUNIONSTORE
**Description:** Stores the union of multiple sets in a new set.

**When to use:** Creating combined datasets.

**Syntax:**
```bash
SUNIONSTORE destination key [key ...]
```

**Example:**
```bash
SADD skills1 "Python" "Java"
SADD skills2 "JavaScript" "Python"
SUNIONSTORE all_skills skills1 skills2
# Output: 3
```

---

### SINTER
**Description:** Returns the intersection of multiple sets (common members).

**When to use:** Finding common elements, shared interests.

**Syntax:**
```bash
SINTER key [key ...]
```

**Example:**
```bash
SADD group1 "Alice" "Bob" "Charlie"
SADD group2 "Bob" "Charlie" "David"
SINTER group1 group2
# Output: ["Bob", "Charlie"]
```

---

### SINTERSTORE
**Description:** Stores the intersection of multiple sets in a new set.

**When to use:** Creating filtered datasets.

**Syntax:**
```bash
SINTERSTORE destination key [key ...]
```

**Example:**
```bash
SADD python_devs "Alice" "Bob" "Charlie"
SADD redis_devs "Bob" "Charlie" "David"
SINTERSTORE python_redis_devs python_devs redis_devs
# Output: 2
```

---

### SDIFF
**Description:** Returns members in the first set that are not in the other sets.

**When to use:** Finding differences, exclusions.

**Syntax:**
```bash
SDIFF key [key ...]
```

**Example:**
```bash
SADD all_users "user1" "user2" "user3" "user4"
SADD banned_users "user2" "user4"
SDIFF all_users banned_users
# Output: ["user1", "user3"]
```

---

### SDIFFSTORE
**Description:** Stores the difference of multiple sets in a new set.

**When to use:** Creating exclusion lists.

**Syntax:**
```bash
SDIFFSTORE destination key [key ...]
```

**Example:**
```bash
SADD invited "Alice" "Bob" "Charlie"
SADD attended "Alice" "Charlie"
SDIFFSTORE missed invited attended
# Output: 1  (Bob)
```

---

### SSCAN
**Description:** Iterates over set members using a cursor.

**When to use:** Processing large sets without blocking.

**Syntax:**
```bash
SSCAN key cursor [MATCH pattern] [COUNT count]
```

**Example:**
```bash
SSCAN large_set 0 COUNT 10
# Output: ["5", ["member1", "member2", ...]]
```

## Sorted Set (ZSET) Commands

Sorted sets are collections of unique members, each associated with a score used for ordering.

### ZADD
**Description:** Adds one or more members with scores to a sorted set.

**When to use:** Leaderboards, priority queues, time series data.

**Syntax:**
```bash
ZADD key [NX|XX] [GT|LT] [CH] [INCR] score member [score member ...]
```

**Example:**
```bash
ZADD leaderboard 100 "player1" 250 "player2" 180 "player3"
# Output: 3
ZADD leaderboard 220 "player4"
# Output: 1
```

---

### ZRANGE
**Description:** Returns members in a sorted set by index range (low to high score).

**When to use:** Getting top N items, pagination.

**Syntax:**
```bash
ZRANGE key start stop [WITHSCORES]
```

**Example:**
```bash
ZADD scores 10 "Alice" 20 "Bob" 15 "Charlie"
ZRANGE scores 0 -1 WITHSCORES
# Output:
# 1) "Alice"
# 2) "10"
# 3) "Charlie"
# 4) "15"
# 5) "Bob"
# 6) "20"
```

---

### ZREVRANGE
**Description:** Returns members in reverse order (high to low score).

**When to use:** Top scores, recent items (when using timestamp scores).

**Syntax:**
```bash
ZREVRANGE key start stop [WITHSCORES]
```

**Example:**
```bash
ZADD highscores 1500 "Pro1" 2000 "Pro2" 1800 "Pro3"
ZREVRANGE highscores 0 1 WITHSCORES
# Output: ["Pro2", "2000", "Pro3", "1800"]  (top 2)
```

---

### ZRANK
**Description:** Returns the rank (index) of a member in ascending order.

**When to use:** Finding position in rankings.

**Syntax:**
```bash
ZRANK key member
```

**Example:**
```bash
ZADD exam_scores 85 "Alice" 92 "Bob" 78 "Charlie"
ZRANK exam_scores "Alice"
# Output: 1 (0-indexed, Charlie=0, Alice=1, Bob=2)
```

---

### ZREVRANK
**Description:** Returns the rank of a member in descending order.

**When to use:** Finding position from top.

**Syntax:**
```bash
ZREVRANK key member
```

**Example:**
```bash
ZADD ratings 4.5 "Product1" 4.8 "Product2" 4.2 "Product3"
ZREVRANK ratings "Product2"
# Output: 0 (highest rated)
```

---

### ZSCORE
**Description:** Returns the score of a member in a sorted set.

**When to use:** Getting individual scores, checking values.

**Syntax:**
```bash
ZSCORE key member
```

**Example:**
```bash
ZADD points 150 "user123"
ZSCORE points "user123"
# Output: "150"
```

---

### ZINCRBY
**Description:** Increments the score of a member by a given amount.

**When to use:** Updating scores, adding points.

**Syntax:**
```bash
ZINCRBY key increment member
```

**Example:**
```bash
ZADD player_points 100 "player1"
ZINCRBY player_points 50 "player1"
ZSCORE player_points "player1"
# Output: "150"
```

---

### ZCARD
**Description:** Returns the number of members in a sorted set.

**When to use:** Counting participants, set size validation.

**Syntax:**
```bash
ZCARD key
```

**Example:**
```bash
ZADD participants 1 "p1" 2 "p2" 3 "p3"
ZCARD participants
# Output: 3
```

---

### ZCOUNT
**Description:** Returns the count of members with scores within a range.

**When to use:** Filtering by score range, statistics.

**Syntax:**
```bash
ZCOUNT key min max
```

**Example:**
```bash
ZADD ages 25 "Alice" 30 "Bob" 22 "Charlie" 35 "David"
ZCOUNT ages 25 30
# Output: 2 (Alice and Bob)
```

---

### ZREM
**Description:** Removes one or more members from a sorted set.

**When to use:** Deleting entries, removing participants.

**Syntax:**
```bash
ZREM key member [member ...]
```

**Example:**
```bash
ZADD tournament 100 "player1" 200 "player2"
ZREM tournament "player1"
# Output: 1
```

---

### ZREMRANGEBYRANK
**Description:** Removes members within a rank range.

**When to use:** Removing bottom performers, trimming lists.

**Syntax:**
```bash
ZREMRANGEBYRANK key start stop
```

**Example:**
```bash
ZADD scores 10 "a" 20 "b" 30 "c" 40 "d"
ZREMRANGEBYRANK scores 0 1  # Remove lowest 2
# Output: 2
```

---

### ZREMRANGEBYSCORE
**Description:** Removes members with scores in a given range.

**When to use:** Removing old entries, score-based cleanup.

**Syntax:**
```bash
ZREMRANGEBYSCORE key min max
```

**Example:**
```bash
ZADD timestamps 1000 "old1" 2000 "old2" 5000 "recent"
ZREMRANGEBYSCORE timestamps 0 3000  # Remove old
# Output: 2
```

---

### ZRANGEBYSCORE
**Description:** Returns members with scores within a range.

**When to use:** Range queries, filtering by score.

**Syntax:**
```bash
ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]
```

**Example:**
```bash
ZADD prices 10.5 "item1" 25.0 "item2" 15.75 "item3" 30.0 "item4"
ZRANGEBYSCORE prices 15 25 WITHSCORES
# Output: ["item3", "15.75", "item2", "25"]
```

---

### ZREVRANGEBYSCORE
**Description:** Returns members in a score range in reverse order.

**When to use:** Filtering highest scores in a range.

**Syntax:**
```bash
ZREVRANGEBYSCORE key max min [WITHSCORES] [LIMIT offset count]
```

**Example:**
```bash
ZADD temperatures 20 "Mon" 25 "Tue" 22 "Wed" 28 "Thu"
ZREVRANGEBYSCORE temperatures 26 20
# Output: ["Tue", "Wed", "Mon"]
```

---

### ZPOPMIN
**Description:** Removes and returns members with the lowest scores.

**When to use:** Priority queues, processing lowest priority items.

**Syntax:**
```bash
ZPOPMIN key [count]
```

**Example:**
```bash
ZADD tasks 1 "low_priority" 5 "medium" 10 "high"
ZPOPMIN tasks
# Output: ["low_priority", "1"]
```

---

### ZPOPMAX
**Description:** Removes and returns members with the highest scores.

**When to use:** Processing highest priority items.

**Syntax:**
```bash
ZPOPMAX key [count]
```

**Example:**
```bash
ZADD urgent_tasks 10 "task1" 20 "task2" 15 "task3"
ZPOPMAX urgent_tasks 2
# Output: ["task2", "20", "task3", "15"]
```

---

### BZPOPMIN
**Description:** Blocking version of ZPOPMIN.

**When to use:** Blocking priority queue consumers.

**Syntax:**
```bash
BZPOPMIN key [key ...] timeout
```

**Example:**
```bash
BZPOPMIN priority_queue 10
# Waits up to 10 seconds for an item
```

---

### BZPOPMAX
**Description:** Blocking version of ZPOPMAX.

**When to use:** Blocking high-priority queue processing.

**Syntax:**
```bash
BZPOPMAX key [key ...] timeout
```

**Example:**
```bash
BZPOPMAX urgent_queue 0  # Wait indefinitely
```

---

### ZUNIONSTORE
**Description:** Computes the union of multiple sorted sets and stores the result.

**When to use:** Combining leaderboards, merging scores.

**Syntax:**
```bash
ZUNIONSTORE destination numkeys key [key ...] [WEIGHTS weight] [AGGREGATE SUM|MIN|MAX]
```

**Example:**
```bash
ZADD quiz1 80 "Alice" 90 "Bob"
ZADD quiz2 85 "Alice" 75 "Bob"
ZUNIONSTORE total_scores 2 quiz1 quiz2 AGGREGATE SUM
ZRANGE total_scores 0 -1 WITHSCORES
# Output: ["Bob", "165", "Alice", "165"]
```

---

### ZINTERSTORE
**Description:** Computes the intersection of multiple sorted sets and stores the result.

**When to use:** Finding common members across sorted sets.

**Syntax:**
```bash
ZINTERSTORE destination numkeys key [key ...] [WEIGHTS weight] [AGGREGATE SUM|MIN|MAX]
```

**Example:**
```bash
ZADD set1 1 "a" 2 "b" 3 "c"
ZADD set2 2 "b" 3 "c" 4 "d"
ZINTERSTORE result 2 set1 set2
# Output: 2 (b and c are common)
```

---

### ZSCAN
**Description:** Iterates over sorted set members using a cursor.

**When to use:** Processing large sorted sets incrementally.

**Syntax:**
```bash
ZSCAN key cursor [MATCH pattern] [COUNT count]
```
