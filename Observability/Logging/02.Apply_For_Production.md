## 1. Why Zap?

Plain `log.Printf` is a string. Machines cannot parse it, filter it, or alert on it.

```go
// ❌ Plain text logging — useless in production
log.Printf("payment failed for user %d, order %d, amount %.2f", userID, orderID, amount)
// Output: 2024/01/15 02:47:13 payment failed for user 42, order 789, amount 59.99
// Problem: to filter by user_id you must grep "user 42" — matches "user 420" too.
// Cannot count failures per minute. Cannot build alerts. Cannot query in Grafana.

// ✅ Zap structured logging — every field is named and typed
logger.Error("Payment failed",
    zap.Int64("user_id", userID),
    zap.Int64("order_id", orderID),
    zap.Float64("amount_usd", amount),
    zap.String("gateway", "stripe"),
    zap.Error(err),
)
// Output: {"level":"error","ts":"2024-01-15T02:47:13.241Z","msg":"Payment failed",
//          "user_id":42,"order_id":789,"amount_usd":59.99,"gateway":"stripe",
//          "error":"card declined [insufficient_funds]"}
// Now: filter by user_id=42 (exact), count errors per minute, build Grafana alerts.
```

Zap is the fastest structured logger in Go:

- **Zero heap allocations** for typed fields (`zap.String`, `zap.Int64`)
- **~100× faster** than `log.Printf` for high-throughput services
- **AtomicLevel** — change log level at runtime without restart

## 3. The Logger Package

This is the foundation. Every other file in your project imports from here.

```go
// pkg/logger/logger.go
package logger

import (
	"context"
	"os"

	"go.uber.org/zap"
	"go.uber.org/zap/zapcore"
)

// contextKey is unexported to prevent collisions with other packages
// that also store values in context.Context.
type contextKey struct{}

var loggerKey = contextKey{}

// Config holds everything the logger needs.
type Config struct {
	Level       string // "debug" | "info" | "warn" | "error"
	Format      string // "json" (production) | "console" (development)
	ServiceName string // e.g. "payment-service"
	Version     string // e.g. "2.3.1" — from your build pipeline
	Environment string // "development" | "staging" | "production"
	OutputPath  string // file path for rotation; empty = stdout only
}

// New creates a production-ready Zap logger.
//
// Returns:
//   - *zap.Logger: the base logger, inject into your dependency tree
//   - zap.AtomicLevel: change log level at runtime via HTTP without restart
//   - error: only non-nil if the logger cannot be constructed
func New(cfg Config) (*zap.Logger, zap.AtomicLevel, error) {
	// AtomicLevel is thread-safe and can be updated while the service runs.
	// Expose it on an internal HTTP endpoint for on-demand DEBUG logging.
	atomicLevel := zap.NewAtomicLevel()
	if err := atomicLevel.UnmarshalText([]byte(cfg.Level)); err != nil {
		// Unknown level string — fall back to INFO rather than crashing.
		atomicLevel.SetLevel(zapcore.InfoLevel)
	}

	encoderCfg := zapcore.EncoderConfig{
		// Field names in the JSON output. Change these if your log aggregator
		// (Loki, Elasticsearch) expects different key names.
		TimeKey:       "ts",
		LevelKey:      "level",
		CallerKey:     "caller",
		FunctionKey:   zapcore.OmitKey, // omit — too verbose
		MessageKey:    "msg",
		StacktraceKey: "stacktrace",

		LineEnding:     zapcore.DefaultLineEnding,
		EncodeLevel:    zapcore.LowercaseLevelEncoder,  // "info" not "INFO"
		EncodeTime:     zapcore.ISO8601TimeEncoder,      // "2024-01-15T02:47:13.241Z"
		EncodeDuration: zapcore.MillisDurationEncoder,  // 145.3 (ms as a float)
		EncodeCaller:   zapcore.ShortCallerEncoder,     // "handler/user.go:42"
	}

	var enc zapcore.Encoder
	if cfg.Format == "console" {
		// Human-readable for local development: coloured level, no JSON brackets.
		encoderCfg.EncodeLevel = zapcore.CapitalColorLevelEncoder
		enc = zapcore.NewConsoleEncoder(encoderCfg)
	} else {
		// JSON for production: every field is named and machine-parseable.
		enc = zapcore.NewJSONEncoder(encoderCfg)
	}

	// Build the output cores. Always write to stdout (container best practice).
	// Optionally also write to a rotating file for local persistence.
	cores := []zapcore.Core{
		zapcore.NewCore(enc, zapcore.AddSync(os.Stdout), atomicLevel),
	}

	if cfg.OutputPath != "" {
		// Rotating file writer — see Section 15 for full Lumberjack config.
		fileCore := zapcore.NewCore(
			zapcore.NewJSONEncoder(encoderCfg),
			zapcore.AddSync(newRotatingFile(cfg.OutputPath)),
			atomicLevel,
		)
		cores = append(cores, fileCore)
	}

	core := zapcore.NewTee(cores...) // write to ALL cores simultaneously

	log := zap.New(core,
		zap.AddCaller(), // adds "caller":"handler/user.go:42" to every entry
		zap.AddStacktrace(zapcore.ErrorLevel), // stack trace on ERROR and above only

		// Tier 2 fields: application identity, added ONCE at startup.
		// These appear on EVERY log entry without any extra code at call sites.
		zap.Fields(
			zap.String("app", cfg.ServiceName),
			zap.String("version", cfg.Version),
			zap.String("env", cfg.Environment),
		),
	)

	return log, atomicLevel, nil
}

// InjectContext stores logger in ctx so it flows through the entire
// request lifecycle without being passed as a function argument everywhere.
//
// Usage: in HTTP middleware, call this once after creating the request-scoped logger.
// Every handler and service down the chain calls FromContext to retrieve it.
func InjectContext(ctx context.Context, log *zap.Logger) context.Context {
	return context.WithValue(ctx, loggerKey, log)
}

// FromContext retrieves the request-scoped logger from ctx.
// If no logger was stored (e.g. background worker without middleware),
// returns fallback — this never panics.
//
// Usage pattern in every handler, service, and repository:
//
//	log := logger.FromContext(ctx, h.log)
//	log.Info("doing something", zap.String("key", "value"))
func FromContext(ctx context.Context, fallback *zap.Logger) *zap.Logger {
	if l, ok := ctx.Value(loggerKey).(*zap.Logger); ok {
		return l
	}
	return fallback
}
```

### Why context propagation instead of a global logger?

```go
// ❌ Global logger — the common mistake
var globalLog *zap.Logger  // shared, no request context

func CreateOrder(ctx context.Context, req Request) error {
    globalLog.Info("creating order") // which request? which user? unknown.
    return nil
}

// ✅ Context propagation — the production pattern
func CreateOrder(ctx context.Context, req Request) error {
    log := logger.FromContext(ctx, fallback)
    // This log already carries: request_id, trace_id, user_id
    // because the middleware attached them before this function was called.
    log.Info("creating order",
        zap.String("order_id", req.OrderID),
    )
    return nil
}
```

Context propagation means: **add context once (in middleware), use it everywhere**. You never repeat `zap.String("request_id", ...)` in every log call.

## 4. Context Propagation — The Core Pattern

The middleware injects a rich logger once. Every downstream function extracts it.

```
HTTP Request enters
       │
       ▼
TracingMiddleware        → adds trace_id, span_id to logger
       │
       ▼
RequestIDMiddleware      → adds request_id to logger, stores in context
       │
       ▼
AuthMiddleware           → adds user_id, role to logger, stores in context
       │
       ▼
Handler                  → calls logger.FromContext(ctx, h.log)
       │
       ▼
Service                  → calls logger.FromContext(ctx, s.log)
       │
       ▼
Repository               → calls logger.FromContext(ctx, r.log)

Every log entry at every layer automatically carries:
  app, version, env        ← from zap.Fields() at startup
  trace_id, span_id        ← from TracingMiddleware
  request_id               ← from RequestIDMiddleware
  user_id, role            ← from AuthMiddleware
  order_id (if relevant)   ← from logger.With() at the start of the operation
```

## 5. HTTP Middleware

```go
// internal/middleware/logging.go
package middleware

import (
	"net/http"
	"time"

	"github.com/google/uuid"
	"go.uber.org/zap"

	"myapp/pkg/logger"
)

// RequestLogger is the primary HTTP logging middleware.
// It creates a request-scoped child logger and logs one structured
// entry at the end of every request.
//
// Apply this AFTER your tracing middleware so that trace_id is already
// in the logger when RequestLogger runs.
func RequestLogger(base *zap.Logger) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			// Skip noisy health/readiness probes.
			// They generate hundreds of log entries per minute with zero signal.
			if r.URL.Path == "/health" || r.URL.Path == "/ready" || r.URL.Path == "/metrics" {
				next.ServeHTTP(w, r)
				return
			}

			// Honour upstream request IDs (from API gateway, load balancer).
			// This links this service's logs to the upstream service's logs.
			requestID := r.Header.Get("X-Request-ID")
			if requestID == "" {
				requestID = uuid.New().String()
			}
			// Always echo it back so the client can reference it in bug reports.
			w.Header().Set("X-Request-ID", requestID)

			// Create a child logger with request-scoped fields.
			// From this line on, every log entry during this request carries
			// request_id, method, and path — without any extra code.
			reqLogger := base.With(
				zap.String("request_id", requestID),
				zap.String("method", r.Method),
				zap.String("path", r.URL.Path),
			)

			// Store in context — downstream code calls logger.FromContext(ctx, fallback).
			ctx := logger.InjectContext(r.Context(), reqLogger)

			// statusCapture intercepts WriteHeader to record the status code.
			sw := &statusCapture{ResponseWriter: w, statusCode: http.StatusOK}

			start := time.Now()
			next.ServeHTTP(sw, r.WithContext(ctx))
			duration := time.Since(start)

			// Log ONE entry per request AFTER the handler returns.
			// Log AFTER, not before, so we can include status code and duration.
			fields := []zap.Field{
				zap.Int("status", sw.statusCode),
				zap.Duration("duration", duration),
				zap.Int64("bytes_written", sw.bytesWritten),
				zap.String("user_agent", r.UserAgent()),
			}

			switch {
			case sw.statusCode >= 500:
				// Server error: something in OUR code or infrastructure broke.
				// This is an ERROR — it affects users, it needs attention.
				reqLogger.Error("HTTP request failed", fields...)

			case sw.statusCode >= 400:
				// Client error: the client sent a bad request.
				// This is a WARN — expected, but worth monitoring for spikes.
				reqLogger.Warn("HTTP client error", fields...)

			default:
				// Normal operation: request served successfully.
				reqLogger.Info("HTTP request served", fields...)
			}
		})
	}
}

// Recovery catches panics, logs them as errors, and returns 500.
// Always use this middleware — panics in handlers would silently kill goroutines.
// Place Recovery FIRST in the middleware chain so it catches panics from all
// other middleware and the handler itself.
func Recovery(base *zap.Logger) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			defer func() {
				if rec := recover(); rec != nil {
					log := logger.FromContext(r.Context(), base)
					log.Error("Panic recovered — returning 500",
						zap.Any("panic", rec),
						zap.Stack("stacktrace"),
					)
					http.Error(w,
						`{"success":false,"error":{"code":"INTERNAL_ERROR","message":"An unexpected error occurred"}}`,
						http.StatusInternalServerError,
					)
				}
			}()
			next.ServeHTTP(w, r)
		})
	}
}

// statusCapture wraps http.ResponseWriter to record the response status code
// and number of bytes written. WriteHeader is called at most once.
type statusCapture struct {
	http.ResponseWriter
	statusCode   int
	bytesWritten int64
}

func (sc *statusCapture) WriteHeader(code int) {
	sc.statusCode = code
	sc.ResponseWriter.WriteHeader(code)
}

func (sc *statusCapture) Write(b []byte) (int, error) {
	n, err := sc.ResponseWriter.Write(b)
	sc.bytesWritten += int64(n)
	return n, err
}
```

## 6. Log Level Rules

Learn these rules. Wrong log levels are the #1 logging mistake in production.

```go
// ─── DEBUG ──────────────────────────────────────────────────────────────────
// Developer detail. DISABLED in production (level = "info").
// Freely verbose — no storage cost when disabled.
// Use for: variable values, cache keys, algorithm steps, query parameters.

log.Debug("Cache key computed",
    zap.String("key", "user:42:profile"),
    zap.Duration("compute_time", 2*time.Microsecond),
)

log.Debug("Executing SQL query",
    zap.String("operation", "SELECT"),
    zap.String("table", "orders"),
    zap.Int64("user_id", 42),
)

// ─── INFO ────────────────────────────────────────────────────────────────────
// A meaningful business event completed normally.
// Use for: successful requests, orders placed, users registered, service started.
// Reading only INFO logs should tell a coherent story of what the service did.

log.Info("Order placed successfully",
    zap.String("order_id", "ord-789"),
    zap.String("user_id", "usr-42"),   // user_id ✅  email ❌  (see Section 8)
    zap.Float64("total_usd", 59.99),
    zap.Int("item_count", 3),
    zap.Duration("processing_time", 145*time.Millisecond),
)

log.Info("User registered",
    zap.String("user_id", newUser.ID),
    zap.String("role", "customer"),
    // Never log: email, name, phone, address — all PII
)

// ─── WARN ────────────────────────────────────────────────────────────────────
// Unexpected, BUT the system recovered. No user was harmed.
// Use for: retries that succeeded, fallbacks triggered, slow queries, deprecated API called.
// Key test: did the system handle it without user impact? → WARN.

log.Warn("Payment gateway retry succeeded",
    zap.String("gateway", "stripe"),
    zap.Int("attempt", 2),
    zap.Duration("total_elapsed", 892*time.Millisecond),
    zap.String("order_id", "ord-789"),
)

log.Warn("Slow database query — check index",
    zap.String("table", "orders"),
    zap.String("operation", "SELECT"),
    zap.Duration("elapsed", 320*time.Millisecond),
    zap.String("suggestion", "add index on user_id column"),
)

// ─── ERROR ───────────────────────────────────────────────────────────────────
// A user-facing operation FAILED. Something is broken. Users were affected.
// Use for: unhandled exceptions, DB failures, payment failures, external API down.
// Every ERROR should be actionable — someone should investigate it.
// Rule: log an error ONCE at the layer with the most context (see Section 9).

log.Error("Payment charge failed — order unpaid",
    zap.String("order_id", "ord-789"),
    zap.String("user_id", "usr-42"),
    zap.String("gateway", "stripe"),
    zap.String("decline_code", "insufficient_funds"),
    zap.Int("retry_count", 2),
    zap.Duration("elapsed", 1243*time.Millisecond),
    zap.Error(err), // always include the underlying error
)

// ─── FATAL ───────────────────────────────────────────────────────────────────
// The service CANNOT start or continue. Calls os.Exit(1).
// Use ONLY at startup for unrecoverable failures.
// Never use mid-operation — return an error instead.

log.Fatal("Cannot connect to PostgreSQL — service cannot start",
    zap.String("host", "postgres:5432"),
    zap.Int("max_open_conns", 25),
    zap.Error(err),
)
```

## 7. What to Log at Service Boundaries

The **boundary rule**: log where work enters and exits a component. Do not log every line inside a function.

```go
// internal/user/handler/user_handler.go
package handler

import (
	"encoding/json"
	"net/http"

	"go.uber.org/zap"

	"myapp/internal/user/service"
	"myapp/pkg/logger"
)

type UserHandler struct {
	service service.UserService
	log     *zap.Logger
}

func New(svc service.UserService, log *zap.Logger) *UserHandler {
	// Tier 2 context: component name, added once in the constructor.
	return &UserHandler{
		service: svc,
		log:     log.With(zap.String("handler", "user")),
	}
}

// Register handles POST /api/v1/users/register
func (h *UserHandler) Register(w http.ResponseWriter, r *http.Request) {
	// Extract the request-scoped logger (carries request_id, trace_id from middleware).
	log := logger.FromContext(r.Context(), h.log)

	var req RegisterRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		log.Debug("Failed to decode register request body", zap.Error(err))
		http.Error(w, "invalid request body", http.StatusBadRequest)
		return
	}

	user, err := h.service.Register(r.Context(), req)
	if err != nil {
		// Error was already logged in the service layer with full business context.
		// DO NOT log here again — that creates duplicate entries. (see Section 9)
		http.Error(w, "registration failed", http.StatusInternalServerError)
		return
	}

	// Log the successful boundary crossing: request entered, work done, response sent.
	log.Info("User registered via API",
		zap.String("user_id", user.ID),
		// Do NOT log user.Email — it is PII
	)

	w.WriteHeader(http.StatusCreated)
	json.NewEncoder(w).Encode(user)
}
```

```go
// internal/user/service/user_service.go
package service

import (
	"context"
	"time"

	"go.uber.org/zap"

	"myapp/pkg/logger"
)

type UserService struct {
	repo UserRepository
	log  *zap.Logger
}

func New(repo UserRepository, log *zap.Logger) *UserService {
	return &UserService{
		repo: repo,
		log:  log.With(zap.String("service", "user")),
	}
}

func (s *UserService) Register(ctx context.Context, req RegisterRequest) (*User, error) {
	// Extract request-scoped logger — it already carries request_id and trace_id.
	// Add the operation-specific context with With() so every subsequent log
	// in THIS function carries it without repeating it.
	log := logger.FromContext(ctx, s.log)

	// Hash password — no logging needed here (pure CPU work)
	hash, err := hashPassword(req.Password)
	if err != nil {
		log.Error("Failed to hash password",
			zap.Error(err),
		)
		return nil, err
	}

	user := &User{
		ID:        generateID(),
		Email:     req.Email,
		PassHash:  hash,
		CreatedAt: time.Now().UTC(),
	}

	if err := s.repo.Create(ctx, user); err != nil {
		// This is the ONE place we log this error — the service layer has the
		// most business context. The handler above does NOT log it again.
		log.Error("Failed to create user in database",
			zap.String("user_id", user.ID),
			// Never log user.Email — PII
			zap.Error(err),
		)
		return nil, err
	}

	log.Info("User registered successfully",
		zap.String("user_id", user.ID),
		// Never log email — use user_id as the opaque identifier
	)

	return user, nil
}
```

## 8. What NEVER to Log

This section is the most important. Logging wrong data causes data breaches, GDPR violations, and legal liability. Review every log statement you write.

```go
// ══════════════════════════════════════════════════════════════════════════════
// AUTHENTICATION AND SECRETS — Never, under any circumstances
// ══════════════════════════════════════════════════════════════════════════════

// ❌ Passwords
log.Info("user login", zap.String("password", req.Password))          // NO
log.Info("user login", zap.String("password_hash", hash))             // NO — even hashed

// ❌ JWT tokens, session tokens, API keys
log.Info("authenticated", zap.String("token", jwtToken))              // NO
log.Info("api call", zap.String("api_key", key))                      // NO

// ✅ Safe alternative — log the token's identifier, never the token itself
log.Info("authenticated",
    zap.String("user_id", claims.UserID),
    zap.String("token_id", claims.ID), // jti claim — an opaque ID, not the secret
    zap.Time("expires_at", claims.ExpiresAt.Time),
)

// ══════════════════════════════════════════════════════════════════════════════
// PAYMENT DATA — PCI-DSS compliance
// ══════════════════════════════════════════════════════════════════════════════

// ❌ Full card numbers
log.Info("payment", zap.String("card_number", "4111111111111111"))    // NO
// ❌ CVV — NEVER, not even masked
log.Info("payment", zap.String("cvv", "***"))                         // NO — even masked!
// ❌ Full card + expiry together (combined = PAN)
log.Info("payment", zap.String("card", "4111xxxx1111"), zap.String("expiry", "12/26"))

// ✅ Safe alternative
log.Info("payment attempted",
    zap.String("card_brand", "visa"),
    zap.String("card_last4", "1111"), // last 4 only — safe
    zap.String("order_id", orderID),
    zap.Float64("amount_usd", 59.99),
)
```

## 9. Error Logging — Log Once Rule

**The most common logging mistake**: logging the same error at multiple layers.

```go
// ❌ Wrong — the same error is logged THREE times
// This makes it look like three failures happened when only one did.

// repository layer
func (r *Repo) FindUser(ctx context.Context, id string) (*User, error) {
    err := r.db.QueryRow(...)
    if err != nil {
        r.log.Error("db query failed", zap.Error(err))  // LOG 1
        return nil, err
    }
}

// service layer
func (s *Service) GetUser(ctx context.Context, id string) (*User, error) {
    user, err := r.repo.FindUser(ctx, id)
    if err != nil {
        s.log.Error("failed to find user", zap.Error(err)) // LOG 2 (same error!)
        return nil, err
    }
}

// handler layer
func (h *Handler) GetUser(w http.ResponseWriter, r *http.Request) {
    user, err := h.service.GetUser(r.Context(), id)
    if err != nil {
        h.log.Error("handler error", zap.Error(err)) // LOG 3 (same error!)
        http.Error(w, "error", 500)
    }
}
```

```go
// ✅ Correct — log ONCE at the layer with the most business context
// Wrap errors upward to add context without logging.

// Repository: execute the query, WRAP and return error without logging.
// The repository has no business context (no user_id, no order_id).
func (r *Repo) FindUser(ctx context.Context, id string) (*User, error) {
    var user User
    err := r.db.QueryRowContext(ctx, "SELECT id FROM users WHERE id=$1", id).
        Scan(&user.ID)
    if err != nil {
        if errors.Is(err, sql.ErrNoRows) {
            return nil, ErrNotFound // domain error, no logging
        }
        // Wrap: adds context to the error message without logging.
        return nil, fmt.Errorf("find user by id: %w", err)
    }
    return &user, nil
}

// Service: has business context (user_id), knows this is a final failure.
// Log HERE — once — with full context.
func (s *Service) GetUser(ctx context.Context, id string) (*User, error) {
    log := logger.FromContext(ctx, s.log)

    user, err := s.repo.FindUser(ctx, id)
    if err != nil {
        // Log HERE with full context. No other layer should log this.
        log.Error("Failed to retrieve user",
            zap.String("user_id", id),
            zap.String("operation", "GetUser"),
            zap.Error(err),
        )
        return nil, err
    }
    return user, nil
}

// Handler: error was already logged. Just respond to the client.
func (h *Handler) GetUser(w http.ResponseWriter, r *http.Request) {
    user, err := h.service.GetUser(r.Context(), id)
    if err != nil {
        // NO log here — already logged with full context in the service.
        // Just send the appropriate HTTP status.
        if errors.Is(err, ErrNotFound) {
            http.Error(w, "user not found", http.StatusNotFound)
        } else {
            http.Error(w, "internal error", http.StatusInternalServerError)
        }
        return
    }
    json.NewEncoder(w).Encode(user)
}
```

## 10. Database Operation Logging

```go
// internal/user/repository/user_repo.go
package repository

import (
	"context"
	"database/sql"
	"errors"
	"fmt"
	"time"

	"go.uber.org/zap"

	"myapp/pkg/logger"
)

type UserRepository struct {
	db  *sql.DB
	log *zap.Logger
}

func New(db *sql.DB, log *zap.Logger) *UserRepository {
	return &UserRepository{
		db:  db,
		log: log.With(zap.String("repository", "user")),
	}
}

func (r *UserRepository) Create(ctx context.Context, user *User) error {
	log := logger.FromContext(ctx, r.log)

	const query = `
		INSERT INTO users (id, email, password_hash, created_at)
		VALUES ($1, $2, $3, $4)`

	// Log at DEBUG before the query — visible only during active investigation.
	// Never log the full query in production — it may contain parameter values.
	log.Debug("Executing INSERT query", zap.String("table", "users"))

	start := time.Now()
	_, err := r.db.ExecContext(ctx, query, user.ID, user.Email, user.PassHash, user.CreatedAt)
	elapsed := time.Since(start)

	if err != nil {
		// Log error HERE with all available context.
		// Use zap.Error(err) — it adds "error":"<message>" as a typed field.
		log.Error("Failed to insert user",
			zap.String("user_id", user.ID),
			// Never log user.Email — PII
			zap.Duration("elapsed", elapsed),
			zap.Error(err),
		)
		return fmt.Errorf("create user: %w", err)
	}

	// Log slow queries as WARN — they indicate missing indexes or growing tables.
	// The threshold (100ms) should be tuned to your SLA.
	if elapsed > 100*time.Millisecond {
		log.Warn("Slow database INSERT — check query plan",
			zap.String("table", "users"),
			zap.String("operation", "INSERT"),
			zap.Duration("elapsed", elapsed),
			zap.String("suggestion", "EXPLAIN ANALYZE the query"),
		)
	} else {
		// Fast query succeeds → DEBUG only. Not visible in production.
		log.Debug("INSERT completed",
			zap.String("user_id", user.ID),
			zap.Duration("elapsed", elapsed),
		)
	}

	return nil
}

func (r *UserRepository) FindByID(ctx context.Context, id string) (*User, error) {
	log := logger.FromContext(ctx, r.log)

	start := time.Now()
	var user User
	err := r.db.QueryRowContext(ctx,
		"SELECT id, email, created_at FROM users WHERE id = $1", id,
	).Scan(&user.ID, &user.Email, &user.CreatedAt)
	elapsed := time.Since(start)

	if err != nil {
		if errors.Is(err, sql.ErrNoRows) {
			// Not found is not an error — it is expected. Log at DEBUG.
			log.Debug("User not found",
				zap.String("user_id", id),
				zap.Duration("elapsed", elapsed),
			)
			return nil, ErrNotFound
		}
		log.Error("Failed to query user by ID",
			zap.String("user_id", id),
			zap.Duration("elapsed", elapsed),
			zap.Error(err),
		)
		return nil, fmt.Errorf("find user by id: %w", err)
	}

	if elapsed > 100*time.Millisecond {
		log.Warn("Slow database SELECT", zap.Duration("elapsed", elapsed))
	}

	return &user, nil
}
```

## 12. Background Worker Logging

Background workers fail silently. Unlike HTTP requests that produce a 500 response, a cron job that crashes at midnight produces nothing visible unless you log it.

```go
// internal/worker/invoice_worker.go
package worker

import (
	"context"
	"time"

	"go.uber.org/zap"

	"myapp/pkg/logger"
)

type InvoiceWorker struct {
	repo InvoiceRepository
	log  *zap.Logger
}

// Run is called by your cron scheduler or ticker.
func (w *InvoiceWorker) Run(ctx context.Context) {
	// Add worker context to the logger.
	log := w.log.With(zap.String("worker", "invoice-generator"))

	// Log START — proves the worker actually ran.
	log.Info("Invoice generation worker started")
	start := time.Now()

	generated, failed, err := w.generatePendingInvoices(ctx, log)

	if err != nil {
		log.Error("Invoice generation worker failed with fatal error",
			zap.Duration("elapsed", time.Since(start)),
			zap.Int("generated_before_failure", generated),
			zap.Error(err),
		)
		return
	}

	// Log SUMMARY at completion — tells an operator at a glance whether
	// the batch ran normally without counting individual log lines.
	log.Info("Invoice generation worker completed",
		zap.Duration("elapsed", time.Since(start)),
		zap.Int("invoices_generated", generated),
		zap.Int("invoices_failed", failed),
	)
}

func (w *InvoiceWorker) generatePendingInvoices(ctx context.Context, log *zap.Logger) (generated, failed int, err error) {
	orders, err := w.repo.FindPendingOrders(ctx)
	if err != nil {
		return 0, 0, fmt.Errorf("fetch pending orders: %w", err)
	}

	log.Info("Fetched orders for invoicing", zap.Int("count", len(orders)))

	for _, order := range orders {
		// Per-item logging at DEBUG — visible only during investigation.
		// Never log at INFO inside a loop — 10,000 orders = 10,000 INFO entries.
		itemLog := log.With(zap.String("order_id", order.ID))
		itemLog.Debug("Generating invoice")

		if err := w.generateInvoice(ctx, order); err != nil {
			// Log per-item errors at ERROR but continue processing other items.
			itemLog.Error("Failed to generate invoice",
				zap.String("user_id", order.UserID),
				zap.Error(err),
			)
			failed++
			continue
		}

		generated++
	}

	return generated, failed, nil
}
```

## 13. Goroutine Logging

When you start a goroutine, the logger context must be explicitly passed in — goroutines do not inherit context.

```go
// ❌ Wrong — goroutine has no request context
for _, order := range orders {
    go func() {
        // Which order? The logger doesn't know.
        log.Info("Processing order in background")
        process(order)
    }()
}

// ✅ Correct — pass the enriched logger explicitly
for _, order := range orders {
    order := order // capture loop variable (required in Go < 1.22)
    orderLog := log.With(zap.String("order_id", order.ID))

    go func(log *zap.Logger) {
        log.Info("Processing order in background goroutine")
        if err := process(order); err != nil {
            log.Error("Background processing failed", zap.Error(err))
            return
        }
        log.Info("Background processing completed")
    }(orderLog) // pass as argument — safe for goroutines
}

// ✅ Correct — for goroutines that receive context
func (w *Worker) startJob(ctx context.Context, job Job) {
    jobLog := logger.FromContext(ctx, w.log).With(
        zap.String("job_type", job.Type),
        zap.String("job_id", job.ID),
    )

    go func() {
        // Use jobLog (a snapshot), not a context-derived logger.
        // The request context may be cancelled before the goroutine completes.
        // jobLog is safe to use after ctx is cancelled.
        jobLog.Info("Background job started")
        if err := w.executeJob(job); err != nil {
            jobLog.Error("Background job failed", zap.Error(err))
            return
        }
        jobLog.Info("Background job completed")
    }()
}
```

## 14. Dynamic Log Level at Runtime

In production, you sometimes need DEBUG logs to investigate an issue without restarting the service. `zap.AtomicLevel` makes this safe.

```go
// In main.go — expose the level endpoint on the INTERNAL port
// This is returned by logger.New() as the second return value.

// Enable DEBUG logging (no restart needed):
//   curl -X PUT http://your-service:9090/internal/log-level -d '{"level":"debug"}'
//
// Revert to INFO:
//   curl -X PUT http://your-service:9090/internal/log-level -d '{"level":"info"}'
//
// Check current level:
//   curl http://your-service:9090/internal/log-level

http.Handle("/internal/log-level", atomicLevel)
```

The `atomicLevel.ServeHTTP` handler is built into Zap — no extra code needed. Changes take effect immediately on all goroutines — zero service disruption.

## 16. Wiring Everything in main.go

```go
// cmd/server/main.go
package main

import (
	"context"
	"fmt"
	"net/http"
	"os"
	"os/signal"
	"syscall"
	"time"

	"github.com/go-chi/chi/v5"
	"go.uber.org/zap"

	"myapp/internal/middleware"
	userhandler "myapp/internal/user/handler"
	userrepo "myapp/internal/user/repository"
	usersvc "myapp/internal/user/service"
	"myapp/pkg/logger"
)

func main() {
	// ── Step 1: Logger — first thing, before EVERYTHING else ──────────
	log, atomicLevel, err := logger.New(logger.Config{
		Level:       getEnv("LOG_LEVEL", "info"),
		Format:      getEnv("LOG_FORMAT", "json"),   // "console" for local dev
		ServiceName: "my-service",
		Version:     getEnv("APP_VERSION", "1.0.0"), // injected by CI/CD
		Environment: getEnv("APP_ENV", "production"),
		OutputPath:  getEnv("LOG_FILE", ""),          // empty = stdout only
	})
	if err != nil {
		fmt.Fprintf(os.Stderr, "FATAL: failed to create logger: %v\n", err)
		os.Exit(1)
	}
	// Sync flushes any buffered log entries before the process exits.
	// Always defer this immediately after creating the logger.
	defer log.Sync()

	// Log startup configuration — everything EXCEPT secrets.
	// This lets you verify what config the service is running with.
	log.Info("Application starting",
		zap.String("version", getEnv("APP_VERSION", "1.0.0")),
		zap.String("env", getEnv("APP_ENV", "production")),
		zap.String("log_level", getEnv("LOG_LEVEL", "info")),
		zap.String("db_host", getEnv("DB_HOST", "localhost")),
		// NEVER log: DB_PASSWORD, JWT_SECRET, API_KEY, REDIS_PASSWORD
	)

	// ── Step 2: Database ──────────────────────────────────────────────
	db, err := connectDB()
	if err != nil {
		log.Fatal("Cannot connect to database", zap.Error(err))
	}
	defer db.Close()
	log.Info("Database connection established",
		zap.String("host", getEnv("DB_HOST", "localhost")),
		zap.Int("max_open_conns", 25),
	)

	// ── Step 3: Wire dependencies ─────────────────────────────────────
	userRepo := userrepo.New(db, log)
	userSvc  := usersvc.New(userRepo, log)
	userH    := userhandler.New(userSvc, log)

	// ── Step 4: Router with middleware stack ──────────────────────────
	r := chi.NewRouter()

	// Middleware order matters — outermost runs first on request,
	// last on response.
	r.Use(middleware.Recovery(log))           // catches panics in ALL other middleware
	r.Use(middleware.RequestLogger(log))      // request_id + one log entry per request

	// Health probes — not logged, not authenticated
	r.Get("/health", func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Content-Type", "application/json")
		fmt.Fprintf(w, `{"status":"ok"}`)
	})

	// Dynamic log level — internal only, NEVER on public port
	r.Mount("/internal/log-level", atomicLevel)

	// API routes
	r.Route("/api/v1", func(r chi.Router) {
		r.Post("/users/register", userH.Register)
		r.Post("/users/login",    userH.Login)
		// ... more routes
	})

	// ── Step 5: HTTP server ───────────────────────────────────────────
	server := &http.Server{
		Addr:         ":" + getEnv("HTTP_PORT", "8080"),
		Handler:      r,
		ReadTimeout:  10 * time.Second,
		WriteTimeout: 30 * time.Second,
		IdleTimeout:  60 * time.Second,
	}

	go func() {
		log.Info("HTTP server started", zap.String("addr", server.Addr))
		if err := server.ListenAndServe(); err != http.ErrServerClosed {
			log.Fatal("HTTP server crashed", zap.Error(err))
		}
	}()

	// ── Step 6: Graceful shutdown ─────────────────────────────────────
	quit := make(chan os.Signal, 1)
	signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
	sig := <-quit

	log.Info("Shutdown signal received — draining connections",
		zap.String("signal", sig.String()),
	)

	ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
	defer cancel()

	if err := server.Shutdown(ctx); err != nil {
		log.Error("Graceful shutdown failed", zap.Error(err))
	}

	log.Info("Application shutdown complete")
}

func getEnv(key, def string) string {
	if v := os.Getenv(key); v != "" {
		return v
	}
	return def
}
```

## 17. Testing Your Logger

```go
// pkg/logger/logger_test.go
package logger_test

import (
	"context"
	"testing"

	"github.com/stretchr/testify/assert"
	"github.com/stretchr/testify/require"
	"go.uber.org/zap"
	"go.uber.org/zap/zapcore"
	"go.uber.org/zap/zaptest/observer"
)

// newObservedLogger creates a logger that captures entries in memory
// instead of writing to stdout. Use this in tests to assert on log output.
func newObservedLogger() (*zap.Logger, *observer.ObservedLogs) {
	// observer.New captures all log entries at INFO and above.
	core, logs := observer.New(zapcore.InfoLevel)
	return zap.New(core), logs
}

func TestLogger_ContextPropagation(t *testing.T) {
	log, logs := newObservedLogger()

	// Simulate what middleware does: store a child logger in context.
	childLog := log.With(zap.String("request_id", "req-abc123"))
	ctx := InjectContext(context.Background(), childLog)

	// Simulate what a service does: extract the logger from context.
	extracted := FromContext(ctx, log)
	extracted.Info("test entry", zap.String("user_id", "usr-42"))

	require.Equal(t, 1, logs.Len())

	entry := logs.All()[0]
	fields := make(map[string]interface{})
	for _, f := range entry.Context {
		fields[f.Key] = f.String
	}

	assert.Equal(t, "test entry", entry.Message)
	assert.Equal(t, "req-abc123", fields["request_id"])
	assert.Equal(t, "usr-42",     fields["user_id"])
}

func TestLogger_FallbackWhenNoLoggerInContext(t *testing.T) {
	fallback, logs := newObservedLogger()

	// Empty context — no logger stored.
	extracted := FromContext(context.Background(), fallback)

	// Should return fallback, not panic.
	extracted.Info("fallback entry")

	assert.Equal(t, 1, logs.Len())
	assert.Equal(t, "fallback entry", logs.All()[0].Message)
}

func TestLogger_LogLevelBelowMinimumIsDropped(t *testing.T) {
	// Observer is set to INFO level — DEBUG entries should be dropped.
	core, logs := observer.New(zapcore.InfoLevel)
	log := zap.New(core)

	log.Debug("this should not appear")
	log.Info("this should appear")

	assert.Equal(t, 1, logs.Len())
	assert.Equal(t, "this should appear", logs.All()[0].Message)
}

func TestLogger_ErrorIncludesStack(t *testing.T) {
	// Create logger with stacktrace enabled on ERROR.
	core, logs := observer.New(zapcore.DebugLevel)
	log := zap.New(core, zap.AddStacktrace(zapcore.ErrorLevel))

	log.Error("something broke")

	require.Equal(t, 1, logs.Len())
	// Stack trace field should be present on ERROR entries.
	entry := logs.All()[0]
	hasStack := false
	for _, f := range entry.Context {
		if f.Key == "stacktrace" {
			hasStack = true
		}
	}
	assert.True(t, hasStack, "expected stacktrace field on ERROR entry")
}
```

## 18. Production Checklist

```
FOUNDATION
  ✅ Using Zap — never log.Printf, fmt.Println, or log.Println in production code
  ✅ JSON format in production, console format in development
  ✅ ISO8601 UTC timestamps on every entry (EncodeTime: zapcore.ISO8601TimeEncoder)
  ✅ Caller field enabled (zap.AddCaller())
  ✅ Stack trace on ERROR and above only (zap.AddStacktrace(zapcore.ErrorLevel))
  ✅ defer log.Sync() called in main() immediately after logger creation
  ✅ AtomicLevel returned from logger.New() and exposed on /internal/log-level

LOG LEVELS
  ✅ Production minimum level: INFO (never DEBUG in production)
  ✅ DEBUG: developer details only — cache keys, SQL params, algorithm steps
  ✅ INFO: business events — user actions, successful operations
  ✅ WARN: recovered anomalies — retries, fallbacks, slow queries
  ✅ ERROR: user-facing failures — DB errors, payment failures
  ✅ FATAL: startup failures only — cannot connect to DB, cannot bind port
  ✅ HTTP 5xx → ERROR, 4xx → WARN, 2xx → INFO (in middleware)

STRUCTURE AND FIELDS
  ✅ snake_case for all field names, consistently across the entire codebase
```

_Good logging is not about logging more — it is about logging the right things, in the right places, with the right context, at the right level. A service with 100 well-crafted log entries per request is infinitely more useful in a crisis than a service with 10,000 entries that say nothing._
