# React Native Animation: A Complete Deep Dive

> A thorough, beginner-friendly guide to every major animation system in React Native — the built-in Animated API, LayoutAnimation, and the modern Reanimated 3 library — explained from first principles with production-grade examples.

---

## Preface — Why Animation Is Not Just "Making Things Move"

Animation in a mobile application is not decoration. It is communication. When a button shrinks slightly under a finger, it confirms that the press was registered. When a list item slides out as it is deleted, the user's eye knows where the other items went. When a loading spinner pulses, the user knows the application is working and has not frozen. When a card flips to reveal its back face, the brain is prepared to see different content. Every animation carries meaning — and a missing animation is itself a message, usually the wrong one.

This guide covers the complete animation ecosystem in React Native, from the foundational `Animated` API that ships inside React Native itself, to `LayoutAnimation` for automatic layout transitions, to the professional-grade `react-native-reanimated` library that runs animations entirely on the native UI thread and is the tool of choice for any animation that demands absolute smoothness.

# Part I — The Animated API: React Native's Built-in Animation Engine

## Chapter 1: How the Animated API Works Under the Hood

Before writing any animation code, you need to understand the fundamental design decision that shapes the entire Animated API: **the separation between animation values and rendered styles**.

In a conventional React application, you animate by calling `setState` repeatedly — perhaps using `setInterval` or `requestAnimationFrame` — updating some numeric value, and letting React re-render the component with each new value. This works, but it has a critical performance flaw: every frame of the animation requires a full round-trip through the React rendering pipeline, across the JavaScript-to-Native bridge. When the JavaScript thread is busy — handling user input, fetching data, processing logic — this bridge crossing may be delayed, and frames are dropped. The animation stutters.

The Animated API solves this by creating **special value objects** — `Animated.Value` and `Animated.ValueXY` — that are not ordinary JavaScript numbers. They are objects that have a direct, persistent connection to the native rendering layer. When you animate an `Animated.Value` from 0 to 1, the animation driver can run entirely on the native thread, updating the rendered view at full frame rate without ever touching the JavaScript thread. This is enabled by the `useNativeDriver: true` option, which is one of the most important performance switches in all of React Native.

```
Animated API Architecture

┌─────────────────────────────────────────────────────────────────────┐
│                         JavaScript Thread                           │
│                                                                     │
│   const opacity = new Animated.Value(0);  ◄── Create value          │
│                                                                     │
│   Animated.timing(opacity, {                                        │
│     toValue: 1,                                                     │
│     duration: 300,                                                  │
│     useNativeDriver: true,  ◄── Key flag                            │
│   }).start();                                                       │
│                                                                     │
│   // JS thread is now FREE. Animation runs below. ✅                │
└──────────────────────────┬──────────────────────────────────────────┘
                           │  Native Driver takes over
                           ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      Native UI Thread                               │
│                                                                     │
│   Frame 1: opacity = 0.0  → update view                             │
│   Frame 2: opacity = 0.1  → update view                             │
│   Frame 3: opacity = 0.3  → update view     ← 60fps, always         │
│   ...                                                               │
│   Frame N: opacity = 1.0  → update view                             │
│                                                                     │
│   No JS involvement after start(). Pure native speed. ✅            │
└─────────────────────────────────────────────────────────────────────┘

⚠️  useNativeDriver: false (JS-driven):
    Every frame: Native → JS Bridge → JS thread → JS Bridge → Native
    → Can drop frames if JS is busy
    → Only use when animating layout properties (width, height, padding)
       which the native driver cannot handle.
```

## Chapter 2: Animated.Value — The Atom of All Animations

`Animated.Value` is the single most important class in the Animated API. Everything else — every animation type, every interpolation, every combination — operates on `Animated.Value` objects. Think of an `Animated.Value` as a special number that knows how to be animated smoothly and how to drive visual properties of React Native views.

You create one with `new Animated.Value(initialValue)` and use `useRef` to keep it stable across re-renders. Without `useRef`, a new `Animated.Value` would be created on every render, destroying the animation state.

```
Animated.Value — The Complete Picture

┌──────────────────────────────────────────────────────────────────────┐
│  const fadeAnim = useRef(new Animated.Value(0)).current;             │
│                                                                      │
│  Creation:                                                           │
│  ─────────                                                           │
│  new Animated.Value(0)    ← starts at 0                              │
│  new Animated.Value(1)    ← starts at 1                              │
│  new Animated.Value(100)  ← starts at 100 (pixels, degrees, etc.)    │
│                                                                      │
│  Reading the current value (for logic, not rendering):               │
│  ──────────────────────────────────────────────────────              │
│  let current;                                                        │
│  fadeAnim.addListener(({ value }) => { current = value; });          │
│  fadeAnim.removeAllListeners();                                      │
│                                                                      │
│  Directly setting without animation:                                 │
│  ────────────────────────────────────                                │
│  fadeAnim.setValue(0.5);  ← Instant snap, no animation               │
│                                                                      │
│  Interpolating into a derived value:                                 │
│  ────────────────────────────────────                                │

│                                                                      │
│  Used in styles via Animated.View:                                   │
│  ──────────────────────────────────                                  │

│  </Animated.View>                                                    │
└──────────────────────────────────────────────────────────────────────┘
```

## Chapter 3: The Three Animation Types — timing, spring, decay

The Animated API provides exactly three primitive animation types, and every animation you will ever build is constructed from these three building blocks. Understanding each deeply is the foundation of all animation work.

**`Animated.timing()`** is the workhorse. It moves a value from its current state to a target value over a specified duration, following an easing curve. The easing curve determines the character of the motion: `Easing.linear` moves at constant speed (robotic), `Easing.ease` starts fast and decelerates (natural), `Easing.bounce` overshoots and bounces back (playful), `Easing.elastic` overshoots dramatically (springy and energetic). The React Native `Easing` module provides a comprehensive library of curves, and you can compose your own with `Easing.bezier()`.

**`Animated.spring()`** simulates physical spring physics. Instead of specifying a duration, you specify physical properties: `mass` (how heavy the object feels), `stiffness` (how tight the spring is), and `damping` (how much the spring resists oscillation — high damping means it settles quickly, low damping means it bounces for a long time). Spring animations feel organic and alive in a way that timing animations cannot fully replicate.

**`Animated.decay()`** simulates an object being thrown and gradually slowing to a stop due to friction. You give it an initial `velocity` and a `deceleration` factor, and it decelerates naturally until it stops. This is used for momentum-based interactions — the natural continuation of a flick gesture.

```
Animation Types — Visual Character

Animated.timing (Easing.ease):

  1 ┤                          ╭──────────
    │                      ╭───╯
    │                  ╭───╯
    │              ╭───╯
  0 ┤──────────────╯
    └────────────────────────────────────► Time
       slow start    ──► fast ──► slows near end

Animated.spring (damping: 10):
 Value
  1 ┤              ╭──╮    ╭──╮ ─────────
    │           ╭──╯  ╰────╯  ╰─
    │        ╭──╯
    │    ╭───╯
  0 ┤────╯
    └────────────────────────────────────► Time
       fast takeoff    overshoots & oscillates → settles

Animated.decay (velocity: 500):

300┤╭─
   │  ╲

       fast at start, decelerating to stop
```

```javascript
import React, { useRef, useEffect } from 'react';
import {
  View, Text, TouchableOpacity, StyleSheet,
  Animated, Easing,
} from 'react-native';

// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 1: The Three Animation Types Side by Side
// ─────────────────────────────────────────────────────────────────────────────

const AnimationTypesDemo = () => {
  const timingAnim = useRef(new Animated.Value(0)).current;
  const springAnim = useRef(new Animated.Value(0)).current;
  const decayAnim  = useRef(new Animated.Value(0)).current;

  const runAllAnimations = () => {
    // Reset all values to their start positions instantly
    timingAnim.setValue(0);
    springAnim.setValue(0);
    decayAnim.setValue(0);

    // ── Timing: Moves the box 250px to the right over 800ms ─────────────────
    Animated.timing(timingAnim, {
      toValue: 250,
      duration: 800,
      easing: Easing.out(Easing.cubic),
      // Easing.out(fn) means the easing is applied to the END of the animation,
      // creating a deceleration effect. Easing.in() applies to the start (acceleration).
      // Easing.inOut() applies to both (slow-fast-slow, very natural).
      useNativeDriver: true,
    }).start();

    // ── Spring: Bounces to 250px with physical spring behavior ───────────────
    Animated.spring(springAnim, {
      toValue: 250,
      mass: 1,
      stiffness: 100,  // How strong the spring pulls toward the target
      damping: 8,      // Low damping = more bouncing oscillation
      useNativeDriver: true,
    }).start();

    // ── Decay: Throw the box with initial velocity, it coasts and slows ──────
    // Note: decay needs to be triggered right after setValue because it
    // uses velocity from a gesture, not a toValue.
    Animated.decay(decayAnim, {
      velocity: 1.2,        // pixels per millisecond (initial speed)
      deceleration: 0.997,  // How quickly it slows (0.997 is the default)
      useNativeDriver: true,
    }).start();
  };

  return (
    <View style={styles.container}>
      <Text style={styles.sectionTitle}>Timing</Text>
      <Animated.View style={[styles.box, styles.boxBlue, { transform: [{ translateX: timingAnim }] }]} />

      <Text style={styles.sectionTitle}>Spring</Text>
      <Animated.View style={[styles.box, styles.boxGreen, { transform: [{ translateX: springAnim }] }]} />

      <Text style={styles.sectionTitle}>Decay</Text>
      <Animated.View style={[styles.box, styles.boxPurple, { transform: [{ translateX: decayAnim }] }]} />

      <TouchableOpacity style={styles.btn} onPress={runAllAnimations}>
        <Text style={styles.btnText}>▶ Run All</Text>
      </TouchableOpacity>
    </View>
  );
};
```

## Chapter 4: Interpolation — The Swiss Army Knife of Animation

Interpolation is arguably the most powerful concept in the Animated API. It allows you to take a single `Animated.Value` and **map it to completely different output ranges** — transforming a value that goes from 0 to 1 into a color that shifts from red to blue, a rotation that spins 360 degrees, a translation that moves 200 pixels, an opacity that fades, and a scale that grows — all simultaneously, all driven by the same single underlying value. This dramatically simplifies complex, coordinated animations.

The `interpolate()` method accepts an `inputRange` (what values the source animated value will pass through) and an `outputRange` (what values to map each input to). The library smoothly interpolates between these control points. You can have as many points as you need — not just two endpoints but multiple breakpoints that create complex, multi-stage curves.

```
Interpolation — Mapping One Value to Many Properties

  Source Animated.Value (scrollY): 0 ───────────────────► 200

  Derived opacity:
  inputRange:  [0,   200]
  outputRange: [1,   0  ]          1 ────╲────────────── 0
                                         (fades as scroll increases)

  Derived translateY (header slide up):
  inputRange:  [0,   200]
  outputRange: [0,  -60 ]          0 ────╲────────────── -60px
                                         (slides up 60px)

  Derived scale:
  inputRange:  [0,   100,  200]
  outputRange: [1,   1.2,  0.8]    1 ──╱─ 1.2 ──╲──── 0.8
                                   (grows then shrinks)

  All three derived from the SAME scrollY value.
  One value drives infinite animations simultaneously.
```

```javascript
// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 2: Interpolation — One Value Drives Everything
// A "hero" animation where a single progress value (0→1)
// simultaneously controls: opacity, scale, translateY, color, rotation.
// ─────────────────────────────────────────────────────────────────────────────

const InterpolationDemo = () => {
  const progress = useRef(new Animated.Value(0)).current;

  // ── Single source value drives FIVE different visual properties ──────────
  const animatedOpacity = progress.interpolate({
    inputRange:  [0, 1],
    outputRange: [0, 1],
  });

  const animatedScale = progress.interpolate({
    inputRange:  [0,    0.5,  1  ],
    outputRange: [0.3,  1.15, 1  ],  // Overshoots slightly at 50% then settles
  });

  const animatedTranslateY = progress.interpolate({
    inputRange:  [0,   1 ],
    outputRange: [80,  0  ],  // Rises 80px from below
  });

  const animatedRotation = progress.interpolate({
    inputRange:  [0,      1    ],
    outputRange: ['-15deg', '0deg'],  // Tilts from -15° to straight
  });

  const animatedColor = progress.interpolate({
    inputRange:  [0,         1        ],
    outputRange: ['#FF6B6B', '#6C63FF'],  // Red to purple
    // ⚠️ Color interpolation requires useNativeDriver: false
  });

  const animatedBorderRadius = progress.interpolate({
    inputRange:  [0,  1 ],
    outputRange: [50, 16],  // Circle to rounded rectangle
  });

  const runEntrance = () => {
    progress.setValue(0);
    Animated.spring(progress, {
      toValue: 1,
      mass: 1,
      stiffness: 120,
      damping: 14,
      useNativeDriver: false, // false because we're animating color & borderRadius
    }).start();
  };

  const runExit = () => {
    Animated.timing(progress, {
      toValue: 0,
      duration: 250,
      easing: Easing.in(Easing.cubic),
      useNativeDriver: false,
    }).start();
  };

  useEffect(() => { runEntrance(); }, []);

  return (
    <View style={styles.container}>
      <Animated.View
        style={[
          styles.heroCard,
          {
            opacity:          animatedOpacity,
            borderRadius:     animatedBorderRadius,
            backgroundColor:  animatedColor,
            transform: [
              { scale:      animatedScale      },
              { translateY: animatedTranslateY },
              { rotate:     animatedRotation   },
            ],
          },
        ]}
      >
        <Text style={styles.heroText}>✨ Hello!</Text>
        <Text style={styles.heroSub}>One value → many properties</Text>
      </Animated.View>

      <View style={styles.row}>
        <TouchableOpacity style={styles.btn} onPress={runEntrance}>
          <Text style={styles.btnText}>Enter ↑</Text>
        </TouchableOpacity>
        <TouchableOpacity style={[styles.btn, styles.btnDanger]} onPress={runExit}>
          <Text style={styles.btnText}>Exit ↓</Text>
        </TouchableOpacity>
      </View>
    </View>
  );
};
```

## Chapter 5: Composing Animations — sequence, parallel, stagger, loop

Individual animations animate a single value in isolation. The real artistry of animation comes from **composing** multiple animations together into choreographed sequences. The Animated API provides four composition combinators that let you orchestrate any number of animations into precise, coordinated performances.

**`Animated.sequence()`** runs animations one after another — the second begins only after the first completes, the third only after the second, and so on. This is perfect for step-by-step reveals, multi-stage transitions, and animations where each phase must wait for the previous.

**`Animated.parallel()`** runs all animations simultaneously. All start at the same moment and run concurrently. You can optionally set `stopTogether: false` so that if one animation is stopped, the others continue.

**`Animated.stagger()`** is like `Animated.parallel()` but with a time offset between each animation's start. Instead of all starting at exactly the same moment, each is delayed by a fixed number of milliseconds from the previous. This creates the elegant "cascade" or "wave" effect you see in well-designed list animations where items appear one after another with a slight delay.

**`Animated.loop()`** repeats an animation indefinitely (or a specified number of times). It is the foundation of loading spinners, pulsing indicators, and any continuously repeating animation.

```
Composition Patterns — Visual Timeline

Animated.sequence([A, B, C]):
  A: ████████
  B:         ████████
  C:                 ████████
  Time: ──────────────────────────────►

Animated.parallel([A, B, C]):
  A: ████████
  B: ████████████
  C: ██████
  Time: ──────────────────────────────►
     All start at the same moment.

Animated.stagger(100, [A, B, C, D]):
  A: ████████           (starts at 0ms)
  B:   ████████         (starts at 100ms)
  C:     ████████       (starts at 200ms)
  D:       ████████     (starts at 300ms)
  Time: ──────────────────────────────►
     Each starts 100ms after the previous. Wave effect!

Animated.loop(A):
  A: ████████████████████████████████
     ◄─────────────────────────────────
     Repeats forever until stopped.
```

```javascript
// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 3: Composition — Orchestrated Loading Animation
// A loading screen that uses sequence, parallel, stagger, and loop together.
// ─────────────────────────────────────────────────────────────────────────────

const LoadingAnimationScreen = () => {
  // Three dot values for the pulsing loader
  const dot1 = useRef(new Animated.Value(0)).current;
  const dot2 = useRef(new Animated.Value(0)).current;
  const dot3 = useRef(new Animated.Value(0)).current;

  // Logo values
  const logoScale   = useRef(new Animated.Value(0)).current;
  const logoOpacity = useRef(new Animated.Value(0)).current;

  // Content card values (for staggered entrance)
  const card1 = useRef(new Animated.Value(0)).current;
  const card2 = useRef(new Animated.Value(0)).current;
  const card3 = useRef(new Animated.Value(0)).current;
  const card4 = useRef(new Animated.Value(0)).current;

  // Spinner rotation (looped)
  const spinValue = useRef(new Animated.Value(0)).current;

  useEffect(() => {

    // ── Step 1: Logo springs in ──────────────────────────────────────────────
    const logoEntrance = Animated.parallel([
      Animated.spring(logoScale, {
        toValue: 1, mass: 1, stiffness: 150, damping: 12, useNativeDriver: true,
      }),
      Animated.timing(logoOpacity, {
        toValue: 1, duration: 400, useNativeDriver: true,
      }),
    ]);

    // ── Step 2: Three dots bounce up one by one (stagger) ────────────────────
    const dotBounce = Animated.stagger(120, [dot1, dot2, dot3].map(dot =>
      Animated.sequence([
        Animated.timing(dot, { toValue: -20, duration: 200, easing: Easing.out(Easing.quad), useNativeDriver: true }),
        Animated.spring(dot, { toValue: 0, mass: 1, stiffness: 300, damping: 15, useNativeDriver: true }),
      ])
    ));

    // ── Step 3: Content cards stagger in from below ──────────────────────────
    const cardEntrance = Animated.stagger(80, [card1, card2, card3, card4].map(card =>
      Animated.spring(card, {
        toValue: 1, mass: 1, stiffness: 180, damping: 18, useNativeDriver: true,
      })
    ));

    // ── Assemble the full sequence ────────────────────────────────────────────
    Animated.sequence([
      logoEntrance,          // First: logo appears
      dotBounce,             // Then: dots bounce (loading indicator)
      cardEntrance,          // Then: content cards appear
    ]).start();

    // ── Spinner loops continuously, independently of the sequence ────────────
    const spinAnimation = Animated.loop(
      Animated.timing(spinValue, {
        toValue: 1,
        duration: 1200,
        easing: Easing.linear,  // Linear is essential for smooth rotation loops
        useNativeDriver: true,
      })
    );
    spinAnimation.start();

    // Cleanup: stop the spinner when the component unmounts
    return () => spinAnimation.stop();

  }, []);

  const spin = spinValue.interpolate({
    inputRange:  [0, 1],
    outputRange: ['0deg', '360deg'],
  });

  const makeCardStyle = (cardAnim) => ({
    opacity: cardAnim,
    transform: [{
      translateY: cardAnim.interpolate({
        inputRange: [0, 1], outputRange: [30, 0],
      }),
    }],
  });

  return (
    <View style={styles.container}>
      {/* Logo */}
      <Animated.View style={{ opacity: logoOpacity, transform: [{ scale: logoScale }] }}>
        <Text style={styles.logo}>⚡</Text>
        <Text style={styles.logoText}>MyApp</Text>
      </Animated.View>

      {/* Spinner */}
      <Animated.View style={[styles.spinner, { transform: [{ rotate: spin }] }]} />

      {/* Bouncing Dots */}
      <View style={styles.dotsRow}>
        {[dot1, dot2, dot3].map((dot, i) => (
          <Animated.View
            key={i}
            style={[styles.dot, { transform: [{ translateY: dot }] }]}
          />
        ))}
      </View>

      {/* Staggered Content Cards */}
      <Animated.View style={[styles.card, makeCardStyle(card1)]}>
        <Text style={styles.cardText}>📊 Dashboard loaded</Text>
      </Animated.View>
      <Animated.View style={[styles.card, makeCardStyle(card2)]}>
        <Text style={styles.cardText}>👤 Profile synced</Text>
      </Animated.View>
      <Animated.View style={[styles.card, makeCardStyle(card3)]}>
        <Text style={styles.cardText}>🔔 Notifications ready</Text>
      </Animated.View>
      <Animated.View style={[styles.card, makeCardStyle(card4)]}>
        <Text style={styles.cardText}>✅ All systems go</Text>
      </Animated.View>
    </View>
  );
};
```

## Chapter 6: The useNativeDriver Rule — What Can and Cannot Be Native-Driven

Understanding the limitation of `useNativeDriver: true` is critical. The native driver can only animate properties that the native rendering layer knows how to handle without JavaScript involvement. These are exclusively the **transform** properties (`translateX`, `translateY`, `scale`, `scaleX`, `scaleY`, `rotate`, `rotateX`, `rotateY`, `rotateZ`, `skewX`, `skewY`, `perspective`) and **opacity**. That is the complete list.

Properties that affect layout — `width`, `height`, `top`, `left`, `margin`, `padding`, `flex`, `borderRadius`, `backgroundColor` — require JavaScript to compute the new layout, so they cannot be native-driven. For these, you must use `useNativeDriver: false`. The cost is that these animations may drop frames if the JavaScript thread is busy.

```
useNativeDriver: true  ← Always prefer this
Supported properties:
┌──────────────────────────────────────────────────────┐
│  transform.translateX    transform.translateY        │
│  transform.scale         transform.scaleX/Y          │
│  transform.rotate        transform.rotateX/Y/Z       │
│  transform.skewX/Y       transform.perspective       │
│  opacity                                             │
└──────────────────────────────────────────────────────┘

useNativeDriver: false  ← Only when necessary
Supported (JS-driven) properties:
┌──────────────────────────────────────────────────────┐
│  width         height       top          left        │
│  margin*       padding*     flex         flexBasis   │
│  backgroundColor            borderRadius             │
│  borderColor   color (Text) fontSize     lineHeight  │
│  zIndex        elevation                             │
└──────────────────────────────────────────────────────┘
```

# Part II — LayoutAnimation: Automatic Layout Transitions

## Chapter 7: What LayoutAnimation Does and How It Works

`LayoutAnimation` operates on a completely different philosophy from the `Animated` API. While `Animated` requires you to explicitly define what animates, what values it moves between, and how long it takes — `LayoutAnimation` is almost entirely automatic. You call `LayoutAnimation.configureNext()` before a state change that will alter the layout, and React Native automatically animates every layout change that results from that state update. Elements that appear are faded in. Elements that disappear are faded out. Elements that move change position smoothly. You write zero explicit animation code.

```
LayoutAnimation Flow

  BEFORE:                          AFTER (with LayoutAnimation):
  ┌────────────────────┐           ┌────────────────────┐
  │  [Card A]          │           │  [Card A]          │
  │  [Card B]          │  ──────►  │  [Card C]  ← animated into place
  │  [Card C]          │  delete B │  [Card D]  ← slid up smoothly
  │  [Card D]          │           └────────────────────┘
  └────────────────────┘
                                   Without LayoutAnimation:
                                   Cards C and D would JUMP instantly.
                                   With LayoutAnimation:
                                   They slide smoothly into their new positions.
```

`LayoutAnimation` is simpler but less precise than `Animated`. You cannot control individual element animations independently — all layout changes are animated with the same configuration. For simple use cases like adding/removing list items, toggling sections, or expanding/collapsing components, it is the fastest and cleanest solution available.

```javascript
// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 4: LayoutAnimation — Animated Todo List
// Every addition and deletion is automatically animated.
// ─────────────────────────────────────────────────────────────────────────────

import {
  View, Text, TouchableOpacity, TextInput,
  StyleSheet, FlatList, Platform, UIManager, LayoutAnimation,
} from 'react-native';
import { useState, useCallback } from 'react';

// ⚠️ Android requires this explicit opt-in to enable LayoutAnimation.
// iOS has it enabled by default.
if (Platform.OS === 'android') {
  UIManager.setLayoutAnimationEnabledExperimental(true);
}

const LAYOUT_CONFIGS = {
  // Built-in presets
  easeInEaseOut: LayoutAnimation.Presets.easeInEaseOut, // Smooth, natural
  linear:        LayoutAnimation.Presets.linear,         // Constant speed
  spring:        LayoutAnimation.Presets.spring,         // Bouncy

  // Custom configuration — full control over every phase
  custom: {
    duration: 350,
    create: {
      // How newly CREATED views animate in
      type:     LayoutAnimation.Types.spring,         // spring, linear, easeIn, easeOut, easeInEaseOut, keyboard
      property: LayoutAnimation.Properties.scaleXY,   // opacity, scaleXY, scaleX, scaleY
      springDamping: 0.7,
    },
    update: {
      // How EXISTING views animate to their new positions/sizes
      type:     LayoutAnimation.Types.easeInEaseOut,
      property: LayoutAnimation.Properties.opacity,
    },
    delete: {
      // How REMOVED views animate out
      type:     LayoutAnimation.Types.easeIn,
      property: LayoutAnimation.Properties.opacity,
      duration: 200,
    },
  },
};

const AnimatedTodoList = () => {
  const [todos, setTodos] = useState([
    { id: 1, text: 'Learn React Native', done: false },
    { id: 2, text: 'Build a portfolio',  done: false },
    { id: 3, text: 'Land a job',         done: false },

    if (!input.trim()) return;
    // ← Call BEFORE state change. React Native watches the next render
    // and automatically animates all layout differences.

      { id: Date.now(), text: input.trim(), done: false },
      ...prev,

    setInput('');
  }, [input]);

    LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut);
    setTodos(prev => prev.map(t =>
      t.id === id ? { ...t, done: !t.done } : t
    ).sort((a, b) => a.done - b.done)); // Done items move to bottom
  }, []);

  const deleteTodo = useCallback((id) => {
    LayoutAnimation.configureNext(LAYOUT_CONFIGS.custom);
    setTodos(prev => prev.filter(t => t.id !== id));
  }, []);

  return (
    <View style={styles.container}>
      <Text style={styles.title}>📝 Animated Todos</Text>
      <Text style={styles.subtitle}>
        {todos.filter(t => !t.done).length} remaining
      </Text>

      <View style={styles.inputRow}>
        <TextInput
          style={styles.input}
          value={input}
          onChangeText={setInput}
          placeholder="New task..."
          placeholderTextColor="#444"
          onSubmitEditing={addTodo}
          returnKeyType="done"
        />
        <TouchableOpacity style={styles.addBtn} onPress={addTodo}>
          <Text style={styles.addBtnText}>＋</Text>
        </TouchableOpacity>
      </View>

      {todos.map(todo => (
        // ⚠️ Use a regular map() rather than FlatList with LayoutAnimation,
        // because FlatList's own recycling interferes with LayoutAnimation.
        <TouchableOpacity
          key={todo.id}
          style={[styles.todoItem, todo.done && styles.todoDone]}
          onPress={() => toggleTodo(todo.id)}
          onLongPress={() => deleteTodo(todo.id)}
          activeOpacity={0.7}
        >
          <Text style={styles.todoCheckbox}>{todo.done ? '✅' : '⬜'}</Text>
          <Text style={[styles.todoText, todo.done && styles.todoTextDone]}>
            {todo.text}
          </Text>
          <TouchableOpacity onPress={() => deleteTodo(todo.id)}>
            <Text style={styles.deleteBtn}>✕</Text>
          </TouchableOpacity>
        </TouchableOpacity>
      ))}
    </View>
  );
};
```

# Part III — react-native-reanimated 3: The Professional Standard

## Chapter 8: Why Reanimated Exists — The Problem It Solves

The `Animated` API with `useNativeDriver: true` runs animations on the native thread, which is great. But it has a fundamental limitation: the **logic** that drives the animation — decisions like "if the card has been dragged more than halfway, complete the swipe; otherwise bounce back" — must still happen in JavaScript. So even though the visual frames run natively, every gesture-driven interaction requires a bridge crossing to consult JavaScript about what to do next. This creates a latency that is perceptible in complex, gesture-driven animations.

`react-native-reanimated` 3 solves this with a radical idea: **worklets**. A worklet is a small JavaScript function that is **automatically serialized and transplanted into the native UI thread's JavaScript virtual machine**. When you mark a function with the `'worklet'` directive, Reanimated compiles it and runs it directly on the UI thread — no bridge crossing, no JavaScript thread involvement whatsoever. The result is gesture-driven animations that are entirely imperceptible from fully native code.

```
Reanimated Architecture — Everything on the UI Thread

  User's Finger ──► Touch Event
                          │
  ┌─────────────────────────────────────────────────────────────────┐
  │                      UI THREAD                                  │
  │                                                                 │
  │  ┌──────────────────────────────────────────────────────────┐   │
  │  │  Gesture Handler (react-native-gesture-handler)          │   │
  │  │  Detects pan/tap/pinch natively                          │   │
  │  └───────────────────────┬──────────────────────────────────┘   │
  │                          │  gesture state (translationX, etc.)  │
  │                          ▼                                      │
  │  ┌──────────────────────────────────────────────────────────┐   │
  │  │  Reanimated Worklet (your JS logic, running HERE)        │   │
  │  │  'worklet'                                               │   │
  │  │  const handler = (event) => {                            │   │
  │  │    translateX.value = event.translationX;  ← runs here   │   │
  │  │  };                                                      │   │
  │  └───────────────────────┬──────────────────────────────────┘   │
  │                          │  shared value updated                │
  │                          ▼                                      │
  │  ┌──────────────────────────────────────────────────────────┐   │
  │  │  Animated Style Computation (also a worklet)             │   │
  │  │  useAnimatedStyle(() => ({                               │   │
  │  │    transform: [{ translateX: translateX.value }],        │   │
  │  │  }))                                                     │   │
  │  └───────────────────────┬──────────────────────────────────┘   │
  │                          │  new style object                    │
  │                          ▼                                      │
  │  ┌──────────────────────────────────────────────────────────┐   │
  │  │  Native View Update (direct, no bridge)                  │   │
  │  └──────────────────────────────────────────────────────────┘   │
  │                                                                 │
  │  JavaScript Thread: NEVER INVOLVED once gesture starts. ✅      │
  └─────────────────────────────────────────────────────────────────┘
```

## Chapter 9: Shared Values and useAnimatedStyle

The two fundamental Reanimated primitives are `useSharedValue` and `useAnimatedStyle`. A **shared value** is Reanimated's equivalent of `Animated.Value` — a piece of mutable state that lives on the UI thread and can be read and written from worklets without a bridge crossing. An **animated style** is a style object computed by a worklet on the UI thread, derived from shared values. Together, they replace the `Animated.Value` + `Animated.View` pattern with something that is both more powerful and more readable.

```javascript
// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 5: Reanimated Basics — Shared Values and useAnimatedStyle
// A card that bounces, scales, fades, and rotates on demand.
// ─────────────────────────────────────────────────────────────────────────────

import Animated, {
  useSharedValue,
  useAnimatedStyle,
  withTiming,
  withSpring,
  withSequence,
  withRepeat,
  withDelay,
  Easing,
  interpolate,
  Extrapolation,
  cancelAnimation,
  runOnJS,
} from 'react-native-reanimated';

const ReanimatedBasics = () => {
  // useSharedValue creates a value on the UI thread.
  // It is NOT a React state variable — changing it does NOT trigger re-renders.
  // It directly drives animated styles through the UI thread.
  const scale     = useSharedValue(1);
  const opacity   = useSharedValue(1);
  const rotateZ   = useSharedValue(0);  // in degrees
  const translateY= useSharedValue(0);

  // useAnimatedStyle creates a style that is recalculated on the UI thread
  // whenever any shared value it reads from changes.
  // This function is automatically compiled as a worklet.
  const animatedStyle = useAnimatedStyle(() => ({
    opacity: opacity.value,
    transform: [
      { scale:      scale.value      },
      { translateY: translateY.value },
      { rotate:     `${rotateZ.value}deg` },
    ],
  }));
  // ↑ This entire function runs on the UI thread. No bridge. No JS involvement.

  // ── Animation 1: Bounce (spring to 1.3, then back to 1) ──────────────────
  const bounce = () => {
    scale.value = withSequence(
      withSpring(1.3, { mass: 0.5, stiffness: 400 }),
      withSpring(1,   { mass: 0.5, stiffness: 200, damping: 8 })
    );
  };

  // ── Animation 2: Shake (rapid left-right timing) ──────────────────────────
  const shake = () => {
    rotateZ.value = withSequence(
      withTiming(-8,  { duration: 60 }),
      withRepeat(
        withSequence(
          withTiming(8,  { duration: 60 }),
          withTiming(-8, { duration: 60 })
        ),
        4,     // Repeat 4 times
        true   // Reverse: oscillate back and forth
      ),
      withTiming(0, { duration: 60 })
    );
  };

  // ── Animation 3: Fade Out and Up ──────────────────────────────────────────
  const fadeOut = () => {
    opacity.value    = withTiming(0, { duration: 500 });
    translateY.value = withTiming(-60, { duration: 500, easing: Easing.in(Easing.cubic) });
  };

  // ── Animation 4: Reset all values ─────────────────────────────────────────
  const reset = () => {
    scale.value      = withSpring(1);
    opacity.value    = withTiming(1, { duration: 300 });
    rotateZ.value    = withSpring(0);
    translateY.value = withSpring(0);
  };

  // ── Animation 5: Pulsing loop (attention grabber) ─────────────────────────
  const startPulse = () => {
    scale.value = withRepeat(
      withSequence(
        withTiming(1.1, { duration: 600, easing: Easing.inOut(Easing.ease) }),
        withTiming(1.0, { duration: 600, easing: Easing.inOut(Easing.ease) })
      ),
      -1,   // -1 = repeat infinitely
      false // false = don't reverse, play the sequence as-is each time
    );
  };

  const stopPulse = () => {
    // cancelAnimation stops a looping or in-progress animation
    cancelAnimation(scale);
    scale.value = withSpring(1);
  };

  return (
    <View style={styles.container}>
      {/* Animated.View from reanimated (NOT from react-native) */}
      <Animated.View style={[styles.demoCard, animatedStyle]}>
        <Text style={styles.demoCardText}>✨ Animate Me</Text>
        <Text style={styles.demoCardSub}>Powered by Reanimated 3</Text>
      </Animated.View>

      <View style={styles.btnGrid}>
        <TouchableOpacity style={styles.smallBtn} onPress={bounce}>
          <Text style={styles.smallBtnText}>Bounce</Text>
        </TouchableOpacity>
        <TouchableOpacity style={styles.smallBtn} onPress={shake}>
          <Text style={styles.smallBtnText}>Shake</Text>
        </TouchableOpacity>
        <TouchableOpacity style={styles.smallBtn} onPress={fadeOut}>
          <Text style={styles.smallBtnText}>Fade Out</Text>
        </TouchableOpacity>
        <TouchableOpacity style={styles.smallBtn} onPress={startPulse}>
          <Text style={styles.smallBtnText}>Pulse</Text>
        </TouchableOpacity>
        <TouchableOpacity style={styles.smallBtn} onPress={stopPulse}>
          <Text style={styles.smallBtnText}>Stop</Text>
        </TouchableOpacity>
        <TouchableOpacity style={[styles.smallBtn, styles.resetBtn]} onPress={reset}>
          <Text style={styles.smallBtnText}>Reset</Text>
        </TouchableOpacity>
      </View>
    </View>
  );
};
```

## Chapter 10: Keyframe Animations — Multi-Stage Choreography

Reanimated 3 introduces **Keyframes** — a declarative animation format inspired by CSS `@keyframes`. Instead of chaining `withSequence(withTiming(...), withSpring(...))`, you define percentage breakpoints along the animation's timeline and specify the exact values at each breakpoint. Reanimated smoothly interpolates between them. This makes complex, multi-stage animations dramatically more readable and maintainable.

```javascript
import Animated, {
  useSharedValue, useAnimatedStyle,
  withTiming, Keyframe,
} from 'react-native-reanimated';

// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 6: Keyframes — Complex multi-stage animation
// A notification toast that slides in, wobbles, and slides out.
// ─────────────────────────────────────────────────────────────────────────────

// ── Define the entrance keyframe ─────────────────────────────────────────────
const enterKeyframe = new Keyframe({
  // 'from' is the starting state (equivalent to 0%)
  from: {
    opacity: 0,
    transform: [
      { translateY: -80 },  // Starts 80px above
      { scale: 0.8 },
    ],
  },
  // Intermediate breakpoints (like CSS keyframes at 50%, 75%, etc.)
  60: {
    opacity: 1,
    transform: [
      { translateY: 8 },   // Overshoots slightly past its final position
      { scale: 1.02 },
    ],
    easing: Easing.out(Easing.back(1.5)),  // Custom easing for this segment
  },
  80: {
    transform: [
      { translateY: -4 },  // Bounces back up a little
      { scale: 0.99 },
    ],
  },
  // 'to' is the final state (equivalent to 100%)
  to: {
    opacity: 1,
    transform: [
      { translateY: 0 },   // Settles at final position
      { scale: 1 },
    ],
  },
}).duration(600); // Total animation duration: 600ms

// ── Define the exit keyframe ──────────────────────────────────────────────────
const exitKeyframe = new Keyframe({
  from: {
    opacity: 1,
    transform: [{ translateY: 0 }, { scale: 1 }],
  },
  40: {
    opacity: 1,
    transform: [{ translateY: -10 }, { scale: 1.03 }],
    easing: Easing.in(Easing.cubic),
  },
  to: {
    opacity: 0,
    transform: [{ translateY: -80 }, { scale: 0.8 }],
  },
}).duration(400);

// ── The Toast Component ───────────────────────────────────────────────────────
const Toast = ({ message, visible, type = 'success' }) => {
  const typeConfig = {
    success: { icon: '✅', color: '#1A3A2E' },
    error:   { icon: '❌', color: '#3A1A1A' },
    info:    { icon: '💡', color: '#1A1E3A' },
    warning: { icon: '⚠️',  color: '#3A2E1A' },
  };
  const { icon, color } = typeConfig[type];

  return (
    // Animated.View with entering/exiting props uses Keyframe animations.
    // 'entering' fires when the component MOUNTS in the React tree.
    // 'exiting'  fires when the component UNMOUNTS (before it's removed).
    // React Navigation uses this same mechanism for screen transitions.
    visible ? (
      <Animated.View
        entering={enterKeyframe}
        exiting={exitKeyframe}
        style={[styles.toast, { backgroundColor: color }]}
      >
        <Text style={styles.toastIcon}>{icon}</Text>
        <Text style={styles.toastText}>{message}</Text>
      </Animated.View>
    ) : null
  );
};

// ── Using the Toast ───────────────────────────────────────────────────────────
const ToastDemo = () => {
  const [toast, setToast] = useState({ visible: false, message: '', type: 'success' });

  const show = (message, type) => {
    setToast({ visible: true, message, type });

  return (
    <View style={styles.container}>
      <Toast {...toast} />

        <Text style={styles.btnText}>Show Success</Text>
      </TouchableOpacity>

        <Text style={styles.btnText}>Show Error</Text>
      </TouchableOpacity>
    </View>
  );
};
```

## Chapter 11: useAnimatedScrollHandler — Animations Driven by Scroll

Scroll-driven animations are among the most visually impressive effects in mobile applications — a header that collapses as you scroll, a parallax background, a progress bar that fills as you read, a floating button that hides and reappears. All of these are powered by `useAnimatedScrollHandler`, which intercepts scroll events on the UI thread and updates shared values in response.

```javascript
// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 7: Scroll-Driven Animation System
// A complete scrollable page with:
// - Collapsing hero image (parallax effect)
// - Sticky header that fades in
// - Progress bar
// - Floating action button that hides on scroll down, shows on scroll up
// ─────────────────────────────────────────────────────────────────────────────

import Animated, {
  useSharedValue,
  useAnimatedStyle,
  useAnimatedScrollHandler,
  interpolate,
  withTiming,
  Extrapolation,
} from 'react-native-reanimated';
import { useState } from 'react';

const HERO_HEIGHT     = 280;
const HEADER_HEIGHT   = 56;
const CONTENT_HEIGHT  = 1800; // Simulated long content

const ScrollDrivenPage = () => {
  const scrollY         = useSharedValue(0);
  const lastScrollY     = useSharedValue(0);
  const fabVisible      = useSharedValue(1);

  // ── Scroll Handler — runs on UI thread ────────────────────────────────────
  // onScroll fires for every scroll frame on the UI thread.
  // We update shared values directly — no bridge, no JS.
  const scrollHandler = useAnimatedScrollHandler({
    onScroll: (event) => {
      const y = event.contentOffset.y;
      scrollY.value = y;

      // Determine scroll direction to show/hide FAB
      if (y > lastScrollY.value + 5) {
        // Scrolling DOWN — hide the FAB
        fabVisible.value = withTiming(0, { duration: 200 });
      } else if (y < lastScrollY.value - 5) {
        // Scrolling UP — show the FAB
        fabVisible.value = withTiming(1, { duration: 200 });
      }
      lastScrollY.value = y;
    },
  });

  // ── Hero Image: Parallax + Fade ───────────────────────────────────────────
  const heroStyle = useAnimatedStyle(() => {
    // As user scrolls down, the hero image scrolls up at HALF the speed
    // creating a parallax depth illusion.
    const translateY = interpolate(
      scrollY.value,
      [0, HERO_HEIGHT],
      [0, -HERO_HEIGHT / 2],
      Extrapolation.CLAMP
    );
    const opacity = interpolate(
      scrollY.value,
      [0, HERO_HEIGHT * 0.7],
      [1, 0],
      Extrapolation.CLAMP
    );
    const scale = interpolate(
      scrollY.value,
      [0, HERO_HEIGHT],
      [1, 1.2],
      Extrapolation.CLAMP
    );
    return { transform: [{ translateY }, { scale }], opacity };
  });

  // ── Sticky Header: Fades and slides in from top ───────────────────────────
  const headerStyle = useAnimatedStyle(() => {
    const opacity = interpolate(
      scrollY.value,
      [HERO_HEIGHT * 0.5, HERO_HEIGHT],
      [0, 1],
      Extrapolation.CLAMP
    );
    const translateY = interpolate(
      scrollY.value,
      [HERO_HEIGHT * 0.5, HERO_HEIGHT],
      [-HEADER_HEIGHT, 0],
      Extrapolation.CLAMP
    );
    return { opacity, transform: [{ translateY }] };
  });

  // ── Progress Bar ──────────────────────────────────────────────────────────
  const progressStyle = useAnimatedStyle(() => {
    const scaleX = interpolate(
      scrollY.value,
      [0, CONTENT_HEIGHT - 600],  // 600 = approx screen height
      [0, 1],
      Extrapolation.CLAMP
    );
    return { transform: [{ scaleX }] };
  });

  // ── Floating Action Button ────────────────────────────────────────────────
  const fabStyle = useAnimatedStyle(() => ({
    opacity:  fabVisible.value,
    transform: [{
      scale: interpolate(fabVisible.value, [0, 1], [0.5, 1], Extrapolation.CLAMP),
    }],
  }));

  return (
    <View style={{ flex: 1, backgroundColor: '#080818' }}>

      {/* Reading Progress Bar — fixed at very top */}
      <Animated.View style={[styles.progressBar, progressStyle]} />

      {/* Sticky Header — invisible until scrolled */}
      <Animated.View style={[styles.stickyHeader, headerStyle]}>
        <Text style={styles.stickyHeaderTitle}>📖 Article Title</Text>
      </Animated.View>

      <Animated.ScrollView
        onScroll={scrollHandler}
        scrollEventThrottle={16}
        style={{ flex: 1 }}
      >
        {/* Hero Section */}
        <Animated.View style={[styles.hero, heroStyle]}>
          <Text style={styles.heroEmoji}>🌌</Text>
          <Text style={styles.heroTitle}>Scroll Down</Text>
          <Text style={styles.heroSub}>Watch the animations react</Text>
        </Animated.View>

        {/* Long Content */}
        <View style={styles.articleContent}>
          {Array.from({ length: 20 }, (_, i) => (
            <View key={i} style={styles.paragraph}>
              <Text style={styles.paragraphText}>
                {`Paragraph ${i + 1}: This is where your article content lives. The hero image
above uses a parallax effect — it scrolls at half speed relative to the content,
creating a sense of depth. The sticky header fades in when you've scrolled past
the hero. The progress bar grows as you read.`.trim()}
              </Text>
            </View>
          ))}
        </View>
      </Animated.ScrollView>

      {/* Floating Action Button — hides on scroll down, shows on scroll up */}
      <Animated.View style={[styles.fab, fabStyle]}>
        <Text style={styles.fabText}>↑</Text>
      </Animated.View>
    </View>
  );
};
```

## Chapter 12: Entering and Exiting Animations — Mount/Unmount Transitions

Reanimated 3's most celebrated quality-of-life feature is **Entering and Exiting animations**. In the classic Animated API, animating a component out of existence was awkward — you had to animate the value, wait for completion, then set state to remove the component. Reanimated 3 solves this elegantly: you simply provide an `entering` prop (fires when the component mounts) and an `exiting` prop (fires before the component unmounts, delaying the unmount until the animation completes). The library ships with dozens of pre-built animations — `FadeIn`, `SlideInLeft`, `ZoomIn`, `BounceIn`, `FlipInX`, and their outgoing counterparts.

```javascript
import Animated, {
  FadeIn, FadeOut,
  SlideInRight, SlideOutLeft,
  BounceIn, BounceOut,
  ZoomIn, ZoomOut,
  FlipInX, FlipOutX,
  LightSpeedInRight, LightSpeedOutRight,
  PinwheelIn, PinwheelOut,
  RotateInDownLeft, RotateOutDownRight,
} from 'react-native-reanimated';
import { useState } from 'react';

// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 8: Entering/Exiting Animations Showcase
// ─────────────────────────────────────────────────────────────────────────────

const ANIMATIONS = [
  { label: 'Fade',       enter: FadeIn,              exit: FadeOut              },
  { label: 'Slide',      enter: SlideInRight,         exit: SlideOutLeft         },
  { label: 'Bounce',     enter: BounceIn,             exit: BounceOut            },
  { label: 'Zoom',       enter: ZoomIn,               exit: ZoomOut              },
  { label: 'Flip',       enter: FlipInX,              exit: FlipOutX             },
  { label: 'Pinwheel',   enter: PinwheelIn,           exit: PinwheelOut          },
  { label: 'LightSpeed', enter: LightSpeedInRight,    exit: LightSpeedOutRight   },
  { label: 'Rotate',     enter: RotateInDownLeft,     exit: RotateOutDownRight   },
];

const EnterExitShowcase = () => {
  const [visible, setVisible] = useState(true);
  const [selectedIndex, setSelectedIndex] = useState(0);

  const selected = ANIMATIONS[selectedIndex];

  return (
    <View style={styles.container}>

      {/* Selector */}
      <View style={styles.selectorRow}>
        {ANIMATIONS.map((anim, i) => (
          <TouchableOpacity
            key={anim.label}
            style={[styles.selectorBtn, i === selectedIndex && styles.selectorBtnActive]}
            onPress={() => {
              setVisible(false);
              setTimeout(() => {
                setSelectedIndex(i);
                setVisible(true);
              }, 500); // Wait for exit animation to finish
            }}
          >
            <Text style={styles.selectorText}>{anim.label}</Text>
          </TouchableOpacity>
        ))}
      </View>

      {/* Stage */}
      <View style={styles.stage}>
        {visible && (
          // The magic: just add entering and exiting props.
          // React handles mount/unmount; Reanimated handles the animations.
          <Animated.View
            entering={selected.enter.duration(600).springify()}
            // .duration(ms) → set how long the animation takes
            // .springify()  → use spring physics instead of timing
            // .delay(ms)    → wait before starting
            // .damping(n)   → spring damping (when springified)
            exiting={selected.exit.duration(400)}
            style={styles.showcase}
          >
            <Text style={styles.showcaseEmoji}>🎭</Text>
            <Text style={styles.showcaseLabel}>{selected.label}</Text>
          </Animated.View>
        )}
      </View>

      <TouchableOpacity
        style={styles.btn}
        onPress={() => setVisible(v => !v)}
      >
        <Text style={styles.btnText}>
          {visible ? 'Animate Out ↗' : 'Animate In ↙'}
        </Text>
      </TouchableOpacity>
    </View>
  );
};
```

## Chapter 13: useAnimatedReaction — Reacting to Value Changes in Worklets

`useAnimatedReaction` is a powerful Reanimated hook that lets you watch a shared value and run a worklet whenever it changes — all on the UI thread. It is the UI-thread equivalent of `useEffect`, designed for situations where you need to trigger a secondary animation in response to a primary animation's value reaching a certain point.

```javascript
import Animated, {
  useSharedValue, useAnimatedStyle,
  useAnimatedReaction, withSpring, withTiming,
  runOnJS,
} from 'react-native-reanimated';

// ─────────────────────────────────────────────────────────────────────────────
// EXAMPLE 9: useAnimatedReaction — Chain reactions and cross-animation triggers
// A slider where crossing the 50% threshold triggers a color change animation.
// ─────────────────────────────────────────────────────────────────────────────

const ChainReactionDemo = () => {
  const sliderX    = useSharedValue(0);   // Controlled by gesture
  const glowScale  = useSharedValue(1);   // Secondary: glow effect
  const [zone, setZone] = useState('left');

  // useAnimatedReaction watches sliderX and triggers secondary effects
  // when it crosses specific thresholds. All of this runs on the UI thread.
  useAnimatedReaction(
    // Dependency function: return the value you want to watch.
    // This runs on the UI thread whenever sliderX changes.
    () => sliderX.value,

    // Reaction function: runs whenever the dependency result changes.
    // Also runs on the UI thread.
    (currentValue, previousValue) => {
      if (previousValue === null) return;

      // Trigger glow animation when crossing the center threshold
      if (currentValue > 150 && previousValue <= 150) {
        // Crossed center going right — animate glow
        glowScale.value = withSpring(1.4, { mass: 0.5, stiffness: 400 });
        setTimeout(() => {
          glowScale.value = withSpring(1);
        }, 200);
        // runOnJS allows calling React state setters from worklets
        runOnJS(setZone)('right');
      } else if (currentValue < 150 && previousValue >= 150) {
        // Crossed center going left
        glowScale.value = withSpring(1.4, { mass: 0.5, stiffness: 400 });
        setTimeout(() => {
          glowScale.value = withSpring(1);
        }, 200);
        runOnJS(setZone)('left');
      }
    }
  );

  const sliderStyle = useAnimatedStyle(() => ({
    transform: [{ translateX: sliderX.value }],
  }));

  const glowStyle = useAnimatedStyle(() => ({
    transform: [{ scale: glowScale.value }],
  }));

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Zone: {zone === 'left' ? '⬅️ Left' : '➡️ Right'}</Text>
      <View style={styles.track}>
        <View style={styles.trackCenter} />
        <Animated.View style={[styles.sliderGlow, glowStyle]} />
        <Animated.View style={[styles.sliderKnob, sliderStyle]}>
          <Text style={styles.sliderKnobText}>●</Text>
        </Animated.View>
      </View>
      <Text style={styles.hint}>Cross the center line to trigger a reaction</Text>
    </View>
  );
};
```

## Chapter 14: Animation Performance — The Complete Checklist

Understanding animation APIs is only half the battle. The other half is ensuring that your animations perform well on real devices, including lower-end Android phones. The following checklist represents accumulated wisdom from production React Native applications.

```
Animation Performance Checklist

ALWAYS:
┌──────────────────────────────────────────────────────────────────────┐
│  ✅ Use useNativeDriver: true whenever possible (Animated API)       │
│  ✅ Prefer react-native-reanimated for gesture-driven animations     │
│  ✅ Keep Animated.Values in useRef (not useState or module-level)    │
│  ✅ Extract animated components outside render functions             │
│  ✅ Use cancelAnimation() to stop loops when component unmounts      │
│  ✅ Test animations on real devices, especially low-end Android      │
│  ✅ Use Easing.linear for looping rotations (avoids micro-stutters)  │
└──────────────────────────────────────────────────────────────────────┘

AVOID:
┌──────────────────────────────────────────────────────────────────────┐
│  ❌ Creating new Animated.Value() inside render (reset every render) │
│  ❌ setState inside animation callbacks for animation state          │
│  ❌ Mixing Animated and Reanimated on the same value                 │
│  ❌ Animating layout properties (width/height) on the native driver  │
│  ❌ Very long animation lists without FlatList virtualization        │
│  ❌ Forgetting to stop loops in useEffect cleanup functions          │
│  ❌ Using InteractionManager for simple animations (overkill)        │
└──────────────────────────────────────────────────────────────────────┘

WHEN TO USE WHICH API:
┌────────────────────────┬─────────────────────────────────────────────┐
│  Use Case              │  Best Tool                                  │
├────────────────────────┼─────────────────────────────────────────────┤
│  Simple fade/slide in  │  Animated API + useNativeDriver: true       │
│  Button press effect   │  Reanimated (withSpring)                    │
│  Adding/removing items │  LayoutAnimation                            │
│  Gesture + animation   │  Reanimated + Gesture Handler               │
│  Scroll-driven effects │  Reanimated (useAnimatedScrollHandler)      │
│  Mount/unmount effects │  Reanimated (entering/exiting)              │
│  Continuous loop       │  Animated.loop or Reanimated (withRepeat)   │
│  Complex multi-step    │  Animated.sequence or Keyframe              │
│  Shared physics state  │  Reanimated (useSharedValue)                │
└────────────────────────┴─────────────────────────────────────────────┘
```

## Conclusion

Animation in React Native is a layered discipline. The **Animated API** is your built-in workhorse — reliable, well-understood, and sufficient for a large proportion of animations when `useNativeDriver: true` is used. **LayoutAnimation** is your magic wand for layout transitions — call it before a state change, and React Native handles all the motion automatically. **react-native-reanimated** is your professional-grade precision instrument — it moves all animation logic, gesture handling, and style computation to the UI thread, achieving frame rates and responsiveness that are simply impossible in pure JavaScript. Choosing the right tool for each animation is not just a performance decision — it is a design decision that determines whether your application feels alive, native, and delightful, or sluggish and mechanical.
