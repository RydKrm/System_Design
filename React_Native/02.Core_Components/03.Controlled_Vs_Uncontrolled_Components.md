
## The Central Question: Who Owns the Data?

Imagine you are filling out a paper form at a government office. In one scenario, you write whatever you like in the blank fields, and the clerk only reads the paper when you hand it in. In another scenario, the clerk watches every keystroke you make, validates each character as you type, and maintains their own copy of everything you have written at all times. Both approaches accomplish the goal of collecting your input, but the second scenario gives the clerk far more control, visibility, and power over the data at every moment.

This analogy captures the essential difference between **uncontrolled** and **controlled** components in React. In an uncontrolled component, the native element — the `TextInput`, the `Switch` — manages its own internal state, and React simply reads it when needed. In a controlled component, React itself holds the current value in state, feeds it back to the element on every render, and intercepts every change. React is the single source of truth. The element is merely a display surface.

```
┌──────────────────────────────────────────────────────────────────────┐
│          Uncontrolled vs Controlled — The Core Difference           │
│                                                                      │
│  UNCONTROLLED                                                        │
│  ┌──────────────────────────────────────────────────┐               │
│  │                                                  │               │
│  │  User types ──► TextInput (owns its own state)  │               │
│  │                     │                           │               │
│  │                     │ (read only when needed)   │               │
│  │                     ▼                           │               │
│  │                   useRef                        │               │
│  │                (passive observer)               │               │
│  └──────────────────────────────────────────────────┘               │
│                                                                      │
│  CONTROLLED                                                          │
│  ┌──────────────────────────────────────────────────┐               │
│  │                                                  │               │
│  │  User types                                      │               │
│  │      │                                          │               │
│  │      ▼                                          │               │
│  │  onChangeText fires ──► setState(newValue)      │               │
│  │                              │                  │               │
│  │                              │ (React re-renders)│              │
│  │                              ▼                  │               │
│  │                     value={state}               │               │
│  │                  ──► TextInput displays          │               │

│  └──────────────────────────────────────────────────┘               │

└──────────────────────────────────────────────────────────────────────┘
```

## Uncontrolled Components — Letting the Element Lead

In an uncontrolled component, you do not bind the element's value to React state, and you do not listen to every change. The native element — the underlying Android `EditText` or iOS `UITextField` — manages the value internally, just as it would in a purely native application. React has no awareness of what the user has typed until you explicitly ask for it.

To read the value from an uncontrolled input, you attach a `ref` to it using `useRef`. The ref gives you a direct handle to the native element, and when you need the value — typically at form submission — you read it from `ref.current.value` (on web) or call methods on the native component through the ref.

The most honest observation about uncontrolled components is that they feel simpler at first glance. You write less code — no `useState`, no `onChangeText`, no `value` prop. But this simplicity comes at a cost: you sacrifice React's awareness of the input's contents at all times, which makes validation-as-you-type, conditional rendering based on input value, and synchronized state across multiple components considerably more difficult.

### When Uncontrolled Components Are Appropriate

Uncontrolled components are appropriate when you only need the value at a single moment in time (form submission), when you are integrating with a third-party library that manages its own internal state, or when performance is a critical concern and you want to avoid re-rendering the component on every single keystroke for a non-critical input.

jsx

```jsx
import { useRef, useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, Alert, StyleSheet } from 'react-native';

export default function UncontrolledForm() {
  // useRef — does NOT cause re-renders when the input changes
  // React has no knowledge of what the user is typing at any moment
  const nameRef = useRef(null);
  const emailRef = useRef(null);
  const passwordRef = useRef(null);

  const [submitted, setSubmitted] = useState(false);

  const handleSubmit = () => {
    // We only read the values at the moment of submission —
    // not on every keystroke. React has been "passive" until now.
    const name = nameRef.current?.getNativeRef?.()?.value
      ?? nameRef.current?._lastNativeText
      ?? '';

    // In React Native, reading uncontrolled TextInput values via ref
    // is less straightforward than on the web. This is one reason
    // uncontrolled inputs are less common in React Native specifically.
    // The pattern works, but controlled inputs are strongly preferred.

    Alert.alert('Form Submitted', `Name ref exists: ${!!nameRef.current}`);
    setSubmitted(true);
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Uncontrolled Form</Text>
      <Text style={styles.note}>
        React has NO knowledge of what you type until you submit.
        No re-renders happen as you type.
      </Text>

      {/* No 'value' prop, no 'onChangeText' handler */}
      {/* The TextInput manages its own internal state */}
      <TextInput
        ref={nameRef}
        style={styles.input}
        placeholder="Full Name (uncontrolled)"
        placeholderTextColor="#aaa"
        defaultValue=""   // sets initial value without binding React state
      />

      <TextInput
        ref={emailRef}
        style={styles.input}
        placeholder="Email (uncontrolled)"
        placeholderTextColor="#aaa"
        keyboardType="email-address"
        autoCapitalize="none"
      />

      <TextInput
        ref={passwordRef}
        style={styles.input}
        placeholder="Password (uncontrolled)"
        placeholderTextColor="#aaa"
        secureTextEntry
      />

      <TouchableOpacity style={styles.submitBtn} onPress={handleSubmit}>
        <Text style={styles.submitText}>Submit</Text>
      </TouchableOpacity>

      {submitted && (
        <Text style={styles.result}>
          ✅ Submitted! React read the values only at this moment.
        </Text>
      )}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f9f9f9', padding: 24, justifyContent: 'center' },
  title: { fontSize: 22, fontWeight: '800', color: '#1a1a2e', marginBottom: 8 },
  note: { fontSize: 13, color: '#888', marginBottom: 20, lineHeight: 18 },
  input: {
    backgroundColor: '#fff', borderWidth: 1.5, borderColor: '#e0e0e0',
    borderRadius: 10, paddingHorizontal: 14, paddingVertical: 12,
    fontSize: 15, color: '#1a1a1a', marginBottom: 12,
  },
  submitBtn: {
    backgroundColor: '#6C63FF', borderRadius: 10,
    paddingVertical: 14, alignItems: 'center', marginTop: 4,
  },
  submitText: { color: '#fff', fontSize: 16, fontWeight: '700' },
  result: { marginTop: 16, color: '#27ae60', fontSize: 14, textAlign: 'center' },
});
```

## Controlled Components — React Holds the Truth

In a controlled component, React state is the definitive, authoritative source of the input's value. The `value` prop feeds the current state into the element, and the `onChangeText` callback feeds every change back into state. This creates a **data loop**: the user types → the event fires → state updates → React re-renders → the element displays the new state value. The element never holds a value of its own; it is purely a reflection of what React's state says it should display.

```
┌─────────────────────────────────────────────────────────────────┐
│               Controlled Component Data Loop                    │
│                                                                 │
│    User types 'H' into TextInput                               │
│                │                                               │
│                ▼                                               │
│    onChangeText('H') fires                                     │
│                │                                               │
│                ▼                                               │
│    setName('H') — React state updated                         │
│                │                                               │
│                ▼                                               │
│    Component re-renders                                        │
│    name = 'H' (from state)                                     │
│                │                                               │
│                ▼                                               │
│    <TextInput value="H" />  ← React feeds 'H' back to input   │
│    TextInput displays 'H' because React TOLD it to            │
│                │                                               │
│    User types 'e' → onChangeText('He') → setName('He') → ...  │
│                                                               │
│    At every moment: state === what the user sees ✅           │
└─────────────────────────────────────────────────────────────────┘
```

This loop is what makes controlled components so powerful. Because React knows the value at every keystroke, you can do things that would be impossible or very complex with uncontrolled inputs: validate the input in real time and show errors as the user types, disable the submit button until certain conditions are met, enforce a maximum character length, transform the input (e.g., automatically capitalizing the first letter or formatting a phone number), synchronize two inputs so that one reflects the other, and derive computed values based on the current input.

### When Controlled Components Are Appropriate

Controlled components are the **strongly recommended default** in React Native. The React Native documentation itself guides developers toward controlled inputs. Use controlled components for any form that requires real-time validation, any input whose current value influences other parts of the UI, any form with complex submission logic, and essentially any input in a production-quality application.

jsx

```jsx
import { useState } from 'react';
import {
  View, Text, TextInput, TouchableOpacity,
  ScrollView, StyleSheet
} from 'react-native';

// Phone number formatter — transforms raw digits into (XXX) XXX-XXXX
const formatPhone = (raw) => {
  const digits = raw.replace(/\D/g, '').slice(0, 10);
  if (digits.length <= 3) return digits;
  if (digits.length <= 6) return `(${digits.slice(0, 3)}) ${digits.slice(3)}`;
  return `(${digits.slice(0, 3)}) ${digits.slice(3, 6)}-${digits.slice(6)}`;
};

// Real-time validators
const validators = {
  name: (v) => {
    if (!v.trim()) return 'Full name is required';
    if (v.trim().length < 2) return 'Name must be at least 2 characters';
    return null;
  },
  email: (v) => {
    if (!v) return 'Email is required';
    if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v)) return 'Enter a valid email address';
    return null;
  },
  phone: (v) => {
    const digits = v.replace(/\D/g, '');
    if (!digits) return 'Phone number is required';
    if (digits.length < 10) return 'Phone must be 10 digits';
    return null;
  },
  password: (v) => {
    if (!v) return 'Password is required';
    if (v.length < 8) return `${8 - v.length} more characters needed`;
    if (!/[A-Z]/.test(v)) return 'Must contain an uppercase letter';
    if (!/[0-9]/.test(v)) return 'Must contain a number';
    return null;
  },
};

export default function ControlledForm() {
  // Every field's value lives in React state — React is the source of truth
  const [fields, setFields] = useState({
    name: '', email: '', phone: '', password: '',
  });

  // Track which fields have been "touched" (blurred at least once)
  // We only show errors on touched fields to avoid overwhelming new users
  const [touched, setTouched] = useState({
    name: false, email: false, phone: false, password: false,
  });

  const [showPassword, setShowPassword] = useState(false);
  const [submitted, setSubmitted] = useState(false);

  // Generic change handler — updates the relevant field in state
  const handleChange = (field) => (value) => {
    // Special transformation for phone: auto-format as user types
    const processed = field === 'phone' ? formatPhone(value) : value;
    setFields(prev => ({ ...prev, [field]: processed }));
  };

  // Mark a field as touched when the user leaves it
  const handleBlur = (field) => () => {
    setTouched(prev => ({ ...prev, [field]: true }));
  };

  // Compute errors for all fields
  const errors = {
    name: validators.name(fields.name),
    email: validators.email(fields.email),
    phone: validators.phone(fields.phone),
    password: validators.password(fields.password),
  };

  // Form is valid only when all validators return null
  const isValid = Object.values(errors).every(e => e === null);

  // Password strength indicator
  const passwordStrength = (() => {
    const p = fields.password;
    if (!p) return { label: '', color: '#ddd', width: '0%' };
    let score = 0;
    if (p.length >= 8) score++;
    if (/[A-Z]/.test(p)) score++;
    if (/[0-9]/.test(p)) score++;
    if (/[^A-Za-z0-9]/.test(p)) score++;
    const map = [
      { label: 'Weak', color: '#e74c3c', width: '25%' },
      { label: 'Fair', color: '#e67e22', width: '50%' },
      { label: 'Good', color: '#f1c40f', width: '75%' },
      { label: 'Strong', color: '#27ae60', width: '100%' },
    ];
    return map[score - 1] || map[0];
  })();

  const handleSubmit = () => {
    // Mark all fields as touched so all errors show
    setTouched({ name: true, email: true, phone: true, password: true });
    if (!isValid) return;
    setSubmitted(true);
  };

  if (submitted) {
    return (
      <View style={styles.successScreen}>
        <Text style={styles.successIcon}>🎉</Text>
        <Text style={styles.successTitle}>Registration Complete!</Text>
        <Text style={styles.successText}>Welcome, {fields.name.split(' ')[0]}!</Text>
        <Text style={styles.successEmail}>{fields.email}</Text>
        <TouchableOpacity
          style={styles.resetBtn}
          onPress={() => {
            setFields({ name: '', email: '', phone: '', password: '' });
            setTouched({ name: false, email: false, phone: false, password: false });
            setSubmitted(false);
          }}
        >
          <Text style={styles.resetText}>Start Over</Text>
        </TouchableOpacity>
      </View>
    );
  }

  return (
    <ScrollView style={styles.container} contentContainerStyle={styles.content}>
      <Text style={styles.title}>Controlled Form</Text>
      <Text style={styles.subtitle}>
        React knows the value of every field at every keystroke.
        Validation and formatting happen in real time.
      </Text>

      {/* Name field */}
      <View style={styles.field}>
        <Text style={styles.label}>Full Name</Text>
        <TextInput
          style={[
            styles.input,
            touched.name && errors.name && styles.inputError,
            touched.name && !errors.name && styles.inputValid,
          ]}
          value={fields.name}                   // ← controlled: React feeds value
          onChangeText={handleChange('name')}   // ← React updates state on change
          onBlur={handleBlur('name')}
          placeholder="Jane Doe"
          placeholderTextColor="#aaa"
        />
        {touched.name && errors.name && (
          <Text style={styles.errorText}>⚠ {errors.name}</Text>
        )}
        {touched.name && !errors.name && (
          <Text style={styles.validText}>✓ Looks good!</Text>
        )}
      </View>

      {/* Email field */}
      <View style={styles.field}>
        <Text style={styles.label}>Email Address</Text>
        <TextInput
          style={[
            styles.input,
            touched.email && errors.email && styles.inputError,
            touched.email && !errors.email && styles.inputValid,
          ]}
          value={fields.email}
          onChangeText={handleChange('email')}
          onBlur={handleBlur('email')}
          placeholder="you@example.com"
          placeholderTextColor="#aaa"
          keyboardType="email-address"
          autoCapitalize="none"
          autoCorrect={false}
        />
        {touched.email && errors.email && (
          <Text style={styles.errorText}>⚠ {errors.email}</Text>
        )}
        {touched.email && !errors.email && (
          <Text style={styles.validText}>✓ Valid email</Text>
        )}
      </View>

      {/* Phone field — auto-formats as you type */}
      <View style={styles.field}>
        <Text style={styles.label}>Phone Number</Text>
        <TextInput
          style={[
            styles.input,
            touched.phone && errors.phone && styles.inputError,
            touched.phone && !errors.phone && styles.inputValid,
          ]}
          value={fields.phone}            // displays formatted value: (984) 123-4567
          onChangeText={handleChange('phone')}
          onBlur={handleBlur('phone')}
          placeholder="(984) 123-4567"
          placeholderTextColor="#aaa"
          keyboardType="phone-pad"
        />
        {touched.phone && errors.phone && (
          <Text style={styles.errorText}>⚠ {errors.phone}</Text>
        )}
        {touched.phone && !errors.phone && (
          <Text style={styles.validText}>✓ Valid number</Text>
        )}
      </View>

      {/* Password field — real-time strength meter */}
      <View style={styles.field}>
        <Text style={styles.label}>Password</Text>
        <View style={styles.passwordRow}>
          <TextInput
            style={[
              styles.input,
              styles.passwordInput,
              touched.password && errors.password && styles.inputError,
              touched.password && !errors.password && styles.inputValid,
            ]}
            value={fields.password}
            onChangeText={handleChange('password')}
            onBlur={handleBlur('password')}
            placeholder="Min 8 chars, 1 uppercase, 1 number"
            placeholderTextColor="#aaa"
            secureTextEntry={!showPassword}
            autoCapitalize="none"
          />
          <TouchableOpacity
            style={styles.eyeBtn}
            onPress={() => setShowPassword(v => !v)}
          >
            <Text style={styles.eyeIcon}>{showPassword ? '🙈' : '👁️'}</Text>
          </TouchableOpacity>
        </View>

        {/* Strength bar — derived from React-controlled value in real time */}
        {fields.password.length > 0 && (
          <View style={styles.strengthContainer}>
            <View style={styles.strengthBar}>
              <View
                style={[
                  styles.strengthFill,
                  { width: passwordStrength.width, backgroundColor: passwordStrength.color },
                ]}
              />
            </View>
            <Text style={[styles.strengthLabel, { color: passwordStrength.color }]}>
              {passwordStrength.label}
            </Text>
          </View>
        )}

        {touched.password && errors.password && (
          <Text style={styles.errorText}>⚠ {errors.password}</Text>
        )}
        {touched.password && !errors.password && (
          <Text style={styles.validText}>✓ Strong password!</Text>
        )}
      </View>

      {/* Character counters — only possible with controlled inputs */}
      <View style={styles.charCounts}>
        <Text style={styles.charCount}>Name: {fields.name.length} chars</Text>
        <Text style={styles.charCount}>Email: {fields.email.length} chars</Text>
      </View>

      {/* Submit button — disabled until form is valid */}
      <TouchableOpacity
        style={[styles.submitBtn, !isValid && styles.submitDisabled]}
        onPress={handleSubmit}
        disabled={!isValid && Object.values(touched).every(Boolean)}
      >
        <Text style={styles.submitText}>
          {isValid ? 'Create Account →' : 'Fill all fields correctly'}
        </Text>
      </TouchableOpacity>

    </ScrollView>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f2f2f7' },
  content: { padding: 24, paddingBottom: 50 },
  title: { fontSize: 26, fontWeight: '800', color: '#1a1a2e', marginBottom: 6 },
  subtitle: { fontSize: 13, color: '#777', lineHeight: 19, marginBottom: 24 },
  field: { marginBottom: 18 },
  label: { fontSize: 13, fontWeight: '600', color: '#555', marginBottom: 6,
```

## Controlled vs Uncontrolled — Side-by-Side Comparison

```
┌────────────────────────────────────────────────────────────────────────┐
│              Controlled vs Uncontrolled — Full Comparison             │
├─────────────────────────┬──────────────────────┬───────────────────── ┤
│  Feature                │  Controlled          │  Uncontrolled        │
├─────────────────────────┼──────────────────────┼──────────────────────┤
│  Source of truth        │  React state         │  Native DOM/element  │
│  value prop             │  Required            │  Not used            │
│  onChange handler       │  Required            │  Optional            │
│  How to read value      │  From state          │  Via ref             │
│  Re-renders on type     │  Yes (every change)  │  No                  │
│  Real-time validation   │  ✅ Easy             │  ❌ Difficult        │
│  Conditional rendering  │  ✅ Easy             │  ❌ Difficult        │
│  Input transformation   │  ✅ Easy (format)    │  ❌ Difficult        │
│  Disable submit button  │  ✅ Easy             │  ❌ Difficult        │
│  Sync multiple inputs   │  ✅ Easy             │  ❌ Very difficult    │
│  Performance (keypress) │  Slight overhead     │  Marginally faster   │
│  Recommended in RN      │  ✅ YES — default    │  Rarely              │
└─────────────────────────┴──────────────────────┴──────────────────────┘
```

The verdict is clear: in React Native, **controlled components are the correct default choice** for virtually every input. The marginal performance difference is negligible on modern devices, and the gains in control, predictability, and developer experience are significant.
