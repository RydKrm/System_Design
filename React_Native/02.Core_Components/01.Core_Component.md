# Part One: Core Layout & Display Components

## 1. `View` — The Universal Container

If you have ever worked with HTML, you are already familiar with the concept of a `<div>` — a generic, invisible container that exists purely to hold and organize other elements. The `View` component in React Native serves precisely this same purpose, and it is, without any doubt, the most fundamental building block of every React Native interface ever built.

A `View` by itself renders absolutely nothing visible on the screen. It is transparent and dimensionless until you give it a style. Its true power lies in its role as a **layout container** — it uses the **Flexbox layout system** to arrange its children, control spacing, define boundaries, and establish visual structure. Every screen you build will be composed of dozens of nested `View` components, each organizing a specific section of your interface.

The `View` component maps directly to the native `android.view.View` on Android and `UIView` on iOS, meaning it is not a simulated element — it is a genuine, first-class native component.

**When to use it:** Use `View` whenever you need to group components together, create a box with a background color, add borders, apply padding or margin, control the arrangement of children using flexbox, or simply create an invisible structural wrapper.

```
┌─────────────────────────────────────────────┐
│                  View (Container)           │
│  ┌──────────────┐    ┌──────────────────┐   │
│  │  Child View  │    │   Child View     │   │
│  │  (red box)   │    │   (blue box)     │   │
│  └──────────────┘    └──────────────────┘   │
│                                             │
│  flexDirection: 'row' → children side-by-side│
└─────────────────────────────────────────────┘
```

```jsx
import { View, StyleSheet } from 'react-native';

export default function App() {
  return (
    // Outer container — takes full screen
    <View style={styles.container}>

      {/* A simple colored box */}
      <View style={styles.redBox} />

      {/* A row of two boxes sitting side by side */}
      <View style={styles.row}>
        <View style={styles.blueBox} />
        <View style={styles.greenBox} />
      </View>

    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,                  // fills entire screen height
    backgroundColor: '#f5f5f5',
    alignItems: 'center',     // centers children horizontally
    justifyContent: 'center', // centers children vertically
    padding: 20,
  },
  redBox: {
    width: 100,
    height: 100,
    backgroundColor: 'red',
    marginBottom: 20,
    borderRadius: 10,
  },
  row: {
    flexDirection: 'row',     // places children left to right
    gap: 12,
  },
  blueBox: {
    width: 80,
    height: 80,
    backgroundColor: 'steelblue',
    borderRadius: 8,
  },
  greenBox: {
    width: 80,
    height: 80,
    backgroundColor: 'seagreen',
    borderRadius: 8,
  },
});
```

## 2. `Text` — Displaying Words on Screen

The `Text` component is the one and only component in React Native through which you may display textual content. Unlike the web, where you can write plain text almost anywhere inside HTML, React Native enforces a strict rule: **all text must be wrapped inside a `Text` component**. Attempting to write raw text directly inside a `View` will result in an error.

This strictness exists because on mobile platforms, text rendering is a complex, native-level operation. The `Text` component maps to `TextView` on Android and `UILabel` on iOS, and it carries with it all the machinery needed to handle font rendering, line wrapping, ellipsis truncation, accessibility, and right-to-left language support.

`Text` components can be nested inside one another, and this nesting has a very useful property: **styles cascade from parent to child**, just as they do in CSS. If you set a font size on an outer `Text`, the inner `Text` inherits it. If you override the color in the inner `Text`, only that inner portion changes color.

**When to use it:** Use `Text` for headings, body paragraphs, labels, error messages, captions, or any piece of text that needs to appear on screen. Use its `numberOfLines` prop to truncate long content with an ellipsis.

```jsx
import { View, Text, StyleSheet } from 'react-native';

export default function ArticleCard() {
  return (
    <View style={styles.card}>

      {/* Main heading */}
      <Text style={styles.title}>Understanding React Native</Text>

      {/* Subheading */}
      <Text style={styles.subtitle}>A guide for beginners</Text>

      {/* Body paragraph — truncated to 3 lines */}
      <Text style={styles.body} numberOfLines={3}>
        React Native allows developers to build truly native mobile applications
        using JavaScript and React. It bridges the gap between web development
        and mobile development, enabling code reuse and rapid iteration. The
        framework has grown tremendously since its introduction by Meta in 2015.
      </Text>

      {/* Nested Text — demonstrating style inheritance */}
      <Text style={styles.note}>
        Status:{' '}
        <Text style={styles.highlight}>Active Development</Text>
      </Text>

    </View>
  );
}

const styles = StyleSheet.create({
  card: {
    backgroundColor: '#fff',
    padding: 20,
    margin: 16,
    borderRadius: 12,
    shadowColor: '#000',
    shadowOpacity: 0.1,
    shadowRadius: 6,
    elevation: 4,
  },
  title: {
    fontSize: 22,
    fontWeight: '700',
    color: '#1a1a2e',
    marginBottom: 4,
  },
  subtitle: {
    fontSize: 14,
    color: '#666',
    marginBottom: 12,
  },
  body: {
    fontSize: 15,
    lineHeight: 24,     // controls spacing between lines
    color: '#333',
    marginBottom: 12,
  },

    fontSize: 14,

  },
```

## 3. `Image` — Displaying Pictures and Icons

The `Image` component is responsible for rendering images of all kinds — photographs loaded from the internet, icons bundled inside the application, images stored in the device's local file system, and even base64-encoded image strings. It is the single, unified tool for all image display needs in React Native.

One of the most important things to understand about `Image` is the concept of `resizeMode`. Because images come in all shapes and sizes, and the container you place them in rarely matches those dimensions exactly, React Native needs to know how to fit the image within its given space. The `cover` mode scales the image until it fills the container (potentially cropping the edges), `contain` scales it to fit entirely within the container (potentially leaving empty space), and `stretch` distorts it to fill exactly. Choosing the right `resizeMode` is essential for images to look correct.

For remote images (URLs), you **must** specify both `width` and `height` in the style, because React Native cannot know the dimensions of a remote image before it downloads it.

**When to use it:** Use `Image` for product photos, user avatars, banner images, icons, logos, and any static visual asset.

```jsx
import { View, Image, Text, StyleSheet } from 'react-native';

export default function ProfileCard() {
  return (
    <View style={styles.card}>

      {/* Remote image — must specify width and height */}
      <Image
        source={{ uri: 'https://picsum.photos/200' }}
        style={styles.avatar}
        resizeMode="cover"     // fills the circle, crops edges
      />

      {/* Local/bundled image — dimensions are known automatically */}
      {/* <Image source={require('./assets/logo.png')} style={styles.logo} /> */}

      <Text style={styles.name}>Jane Doe</Text>
      <Text style={styles.role}>Senior Engineer</Text>

      {/* A banner image with 'contain' — shows whole image with letterboxing */}
      <Image
        source={{ uri: 'https://picsum.photos/600/200' }}
        style={styles.banner}
        resizeMode="contain"
      />

    </View>
  );
}

const styles = StyleSheet.create({
  card: {
    alignItems: 'center',
    padding: 24,
    backgroundColor: '#fff',
    margin: 16,
    borderRadius: 16,
    elevation: 5,
  },
  avatar: {
    width: 100,
    height: 100,
    borderRadius: 50,          // makes a circle
    marginBottom: 12,
    borderWidth: 3,
    borderColor: '#4A90E2',
  },
  name: {
    fontSize: 20,
    fontWeight: '700',
    color: '#1a1a1a',
  },
  role: {
    fontSize: 14,
    color: '#888',
    marginBottom: 16,
  },
  banner: {
    width: '100%',
    height: 120,
    backgroundColor: '#f0f0f0', // shows while image loads
    borderRadius: 8,
  },
});
```

## 4. `ImageBackground` — An Image That Holds Other Components

There is a common design pattern in mobile applications where content is displayed on top of a background image — a hero section, a login screen with a scenic photograph behind it, or a card with a textured background. On the web, this is trivially accomplished with CSS's `background-image` property. In React Native, however, the `Image` component cannot have children placed on top of it.

This is where `ImageBackground` enters the picture. It is, in essence, an `Image` that accepts children. Internally, `ImageBackground` renders the image first, then positions an absolutely-placed container on top of it into which your children are placed. The `imageStyle` prop allows you to style the image itself (for instance, applying `borderRadius` so the image respects rounded corners), while the regular `style` prop controls the outer container.

**When to use it:** Use `ImageBackground` for hero banners, decorative card backgrounds, login/splash screens with photographic backgrounds, and anywhere you need to layer interface elements on top of an image.

```jsx
import { View, Text, ImageBackground, StyleSheet } from 'react-native';

export default function HeroSection() {
  return (
    <ImageBackground
      source={{ uri: 'https://picsum.photos/400/250' }}
      style={styles.hero}
      imageStyle={styles.heroImage}  // styles the actual <img> element
    >
      {/* This overlay darkens the background so text is readable */}
      <View style={styles.overlay}>
        <Text style={styles.heroTitle}>Welcome to Nepal</Text>
        <Text style={styles.heroSubtitle}>Land of the Himalayas</Text>
      </View>

    </ImageBackground>
  );
}

const styles = StyleSheet.create({
  hero: {
    width: '100%',
    height: 250,
    justifyContent: 'flex-end', // push text to bottom
  },
  heroImage: {
    borderRadius: 16,           // rounds the image corners
  },
  overlay: {
    backgroundColor: 'rgba(0, 0, 0, 0.45)', // semi-transparent dark layer
    padding: 20,
    borderBottomLeftRadius: 16,
    borderBottomRightRadius: 16,
  },
  heroTitle: {
    color: '#fff',
    fontSize: 28,
    fontWeight: '800',
    letterSpacing: 1,
  },
  heroSubtitle: {
    color: 'rgba(255,255,255,0.8)',
    fontSize: 15,
    marginTop: 4,
  },
});
```

## 5. `ScrollView` — A Scrollable Container for Everything

By default, a `View` does not scroll. If you place more content inside a `View` than the screen can display, the overflow is simply clipped and hidden. To make content scrollable, you wrap it in a `ScrollView`. It is the simplest and most direct way to make a screen scrollable — you place all your content inside it, and React Native handles the rest, including scroll physics, bounce effects on iOS, and momentum scrolling.

However, `ScrollView` has one very important characteristic that every developer must understand: **it renders all of its children at once, regardless of whether they are visible on screen**. When a `ScrollView` mounts, every single child component is instantiated, measured, and placed in memory, even if the user would need to scroll for five minutes to reach them. For short content — a settings screen, a profile page, a form — this is perfectly acceptable. For a list of a hundred or a thousand items, this becomes a serious performance problem.

This limitation is precisely the reason `FlatList` and `SectionList` were created, which we will explore next.

**When to use it:** Use `ScrollView` for screens with a modest, known amount of content — forms, settings pages, detail screens, article readers, and any screen where you can comfortably predict that the total content length is manageable.

```jsx
import { ScrollView, View, Text, StyleSheet } from 'react-native';

export default function SettingsScreen() {
  // Imagine these are settings sections
  const sections = [
    { title: 'Account', items: ['Profile', 'Email', 'Password', 'Phone Number'] },
    { title: 'Privacy', items: ['Data Sharing', 'Location', 'Analytics'] },
    { title: 'Notifications', items: ['Push Alerts', 'Email Digest', 'SMS'] },
    { title: 'Appearance', items: ['Theme', 'Font Size', 'Language'] },
    { title: 'Support', items: ['Help Center', 'Contact Us', 'Report a Bug'] },
  ];

  return (
    <ScrollView
      style={styles.container}
      contentContainerStyle={styles.content}  // styles the inner scrollable area
      showsVerticalScrollIndicator={false}     // hides the scrollbar line
      bounces={true}                           // iOS bounce effect at edges
    >
      {sections.map((section) => (
        <View key={section.title} style={styles.section}>
          <Text style={styles.sectionTitle}>{section.title}</Text>
          {section.items.map((item) => (
            <View key={item} style={styles.row}>
              <Text style={styles.rowText}>{item}</Text>
              <Text style={styles.arrow}>›</Text>
            </View>
          ))}
        </View>
      ))}
    </ScrollView>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f2f2f7',
  },
  content: {
    padding: 16,
    paddingBottom: 40,      // extra space at the very bottom
  },
  section: {
    backgroundColor: '#fff',
    borderRadius: 12,
    marginBottom: 20,
    overflow: 'hidden',
  },
  sectionTitle: {
    fontSize: 13,
    fontWeight: '600',
    color: '#888',
    padding: 12,
    paddingBottom: 6,
    textTransform: 'uppercase',
    letterSpacing: 0.8,
  },
  row: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    paddingVertical: 14,
    paddingHorizontal: 16,
    borderTopWidth: 0.5,
    borderTopColor: '#e0e0e0',
  },
  rowText: {
    fontSize: 16,
    color: '#1c1c1e',
  },
  arrow: {
    fontSize: 20,
    color: '#c7c7cc',
  },
});
```

## 6. `FlatList` — The High-Performance List

`FlatList` is the solution to the performance problem that `ScrollView` creates for long lists. Rather than rendering every item at once, `FlatList` employs a technique called **windowing** (also known as virtualization). It only renders the items that are currently visible on screen, plus a small buffer of items just above and below the visible area. As the user scrolls, items that leave the visible area are unmounted from memory, and new items entering the visible area are mounted. The result is that no matter how many thousands of items your list contains, the device only ever has to hold a small fraction of them in memory at any given moment.

`FlatList` requires two essential props: `data` (the array of items) and `renderItem` (a function that receives each item and returns the JSX to render it). It also accepts a `keyExtractor` function which tells it how to derive a unique key for each item — this is important for React's reconciliation algorithm to work correctly.

`FlatList` also provides many built-in conveniences: `ListHeaderComponent` and `ListFooterComponent` for adding content before and after the list, `ItemSeparatorComponent` for drawing dividers between items, `onEndReached` for implementing infinite scroll pagination, and `refreshing`/`onRefresh` for pull-to-refresh functionality.

**When to use it:** Use `FlatList` for any list of unknown length or potentially large size — social media feeds, product catalogues, search results, message threads, contact lists.

```
┌───────────────────────────────┐
│   Visible Area on Screen      │
│  ┌─────────────────────────┐  │
│  │ Item 3 (rendered)       │  │◄── Buffer above
│  ├─────────────────────────┤  │
│  │ Item 4 (visible)        │  │◄── On screen
│  ├─────────────────────────┤  │
│  │ Item 5 (visible)        │  │◄── On screen
│  ├─────────────────────────┤  │
│  │ Item 6 (visible)        │  │◄── On screen
│  ├─────────────────────────┤  │
│  │ Item 7 (rendered)       │  │◄── Buffer below
│  └─────────────────────────┘  │
│ Items 1,2,8,9... NOT in memory│
└───────────────────────────────┘
  Virtualization = only what's
  near the screen stays alive
```

```jsx
import { FlatList, View, Text, Image, StyleSheet } from 'react-native';

// Sample data representing a list of users
const USERS = Array.from({ length: 50 }, (_, i) => ({
  id: String(i + 1),
  name: `User ${i + 1}`,
  role: i % 3 === 0 ? 'Admin' : i % 2 === 0 ? 'Editor' : 'Viewer',
  avatar: `https://i.pravatar.cc/60?img=${i + 1}`,
}));

// This component renders a single row — keep it extracted for performance
function UserRow({ item }) {
  return (
    <View style={styles.row}>
      <Image source={{ uri: item.avatar }} style={styles.avatar} />
      <View style={styles.info}>
        <Text style={styles.name}>{item.name}</Text>
        <Text style={styles.role}>{item.role}</Text>
      </View>
    </View>
  );
}

// A divider drawn between each item
function Separator() {
  return <View style={styles.separator} />;
}

export default function UserList() {
  return (
    <FlatList
      data={USERS}
      keyExtractor={(item) => item.id}     // unique key per item
      renderItem={({ item }) => <UserRow item={item} />}
      ItemSeparatorComponent={Separator}   // drawn between items, not at edges
      ListHeaderComponent={
        <Text style={styles.header}>All Users ({USERS.length})</Text>
      }
      ListFooterComponent={
        <Text style={styles.footer}>— End of List —</Text>
      }
      ListEmptyComponent={
        <Text style={styles.empty}>No users found.</Text>
      }
      onEndReachedThreshold={0.3}          // triggers when 30% from bottom
      onEndReached={() => console.log('Load more items here')}
      showsVerticalScrollIndicator={false}
    />
  );
}

const styles = StyleSheet.create({
  row: {
    flexDirection: 'row',
    alignItems: 'center',
    backgroundColor: '#fff',
    paddingHorizontal: 16,
    paddingVertical: 12,
  },
  avatar: {
    width: 46,

    marginLeft: 76,  // aligns with text, skipping avatar
  },
  header: {
    fontSize: 13,

    textTransform: 'uppercase',
```

## 7. `SectionList` — The Grouped, Sectioned List

`SectionList` is the natural extension of `FlatList` for situations where your data is not a flat array but rather a grouped collection — an alphabetically organized contact list, a menu grouped by category, a chat history grouped by date, or a product list grouped by department. Like `FlatList`, it uses virtualization and only renders visible items. Unlike `FlatList`, it accepts an array of section objects, each with a title and its own array of data.

Each section can have its own header (rendered via `renderSectionHeader`) and optionally its own footer (rendered via `renderSectionFooter`). The global `renderItem` function renders individual items within each section, just as it does in `FlatList`.

**When to use it:** Use `SectionList` whenever your list data has a natural grouping — contacts by letter, orders by date, products by category, messages by day.

## 8. `Pressable` — The Modern, Flexible Touch Handler

`Pressable` is the most modern and recommended touch-handling component in React Native. It was introduced to replace the aging `Touchable*` family of components with a single, unified, and far more flexible API. Unlike its predecessors, `Pressable` exposes the **press state** directly to its children via a render function, allowing you to change the appearance of your component during each phase of a press interaction — when the user's finger first touches down, while it is held, and when it is lifted.

`Pressable` supports four distinct interaction callbacks: `onPressIn` (finger touches the screen), `onPress` (quick tap, fires on finger lift), `onLongPress` (finger held for an extended duration), and `onPressOut` (finger lifts or leaves the area). It also accepts a `hitSlop` prop that invisibly expands the tappable area beyond the component's visual boundaries — extremely useful for small targets like icon buttons.

**When to use it:** Use `Pressable` as your default touch handler for any custom interactive element — custom buttons, cards, list items, icon buttons, and any component where you want precise control over the press state appearance.

```jsx
import { Pressable, View, Text, StyleSheet } from 'react-native';

export default function PressableDemo() {
  return (
    <View style={styles.container}>

      {/* The child is a function that receives the pressed state */}
      <Pressable
        onPress={() => console.log('Pressed!')}
        onLongPress={() => console.log('Long pressed!')}
        hitSlop={10}   // expands tap area by 10px on all sides
        style={({ pressed }) => [
          styles.button,
          pressed && styles.buttonPressed,  // apply extra style when pressed
        ]}
      >
        {/* Children can also be a function for more complex pressed-state changes */}
        {({ pressed }) => (
          <Text style={[styles.label, pressed && styles.labelPressed]}>
            {pressed ? 'Holding...' : 'Press Me'}
          </Text>
        )}
      </Pressable>

      {/* A card that dims when pressed */}
      <Pressable
        onPress={() => console.log('Card tapped')}
        style={({ pressed }) => [
          styles.card,
          { opacity: pressed ? 0.7 : 1 },
        ]}
      >
        <Text style={styles.cardTitle}>Pressable Card</Text>
        <Text style={styles.cardBody}>Tap to select this item</Text>
      </Pressable>

    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', gap: 20 },
  button: {
    backgroundColor: '#4A90E2',
    paddingVertical: 14,
    paddingHorizontal: 32,
    borderRadius: 10,
  },
  buttonPressed: {
    backgroundColor: '#357ABD',
    transform: [{ scale: 0.96 }],
  },
  label: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
  labelPressed: {
    color: 'rgba(255,255,255,0.7)',
  },
  card: {
    width: 300,
    backgroundColor: '#fff',
    padding: 20,
    borderRadius: 12,
    elevation: 3,
  },
  cardTitle: { fontSize: 18, fontWeight: '700', color: '#1a1a1a' },
  cardBody: { fontSize: 14, color: '#888', marginTop: 6 },
```

## 9. `TouchableOpacity` — The Classic Fade-on-Press

`TouchableOpacity` is perhaps the most widely used touch component in the React Native ecosystem, owing to its simplicity and the naturalness of its feedback mechanism. When a user presses a component wrapped in `TouchableOpacity`, the entire component fades to a reduced opacity — by default `0.2` — giving clear visual confirmation that the press was registered. When the finger lifts, the opacity smoothly returns to full.

This opacity fade is handled natively for performance, meaning it does not require a JavaScript re-render to animate. The degree of dimming is controlled by the `activeOpacity` prop, which accepts a value between `0` (completely transparent) and `1` (no change). A value around `0.6` or `0.7` gives a subtler, more refined feel than the default.

While `Pressable` is the newer recommended choice, `TouchableOpacity` remains perfectly valid and is very widely used in existing codebases. Understanding it is essential for any React Native developer.

**When to use it:** Use `TouchableOpacity` for buttons, list items, menu entries, and any pressable element where a simple opacity fade provides sufficient visual feedback.

## 10. `TouchableHighlight` — The Darkening Press Effect

Where `TouchableOpacity` reduces the opacity of a component on press, `TouchableHighlight` works differently: it shows a **colored underlay beneath the component** when pressed. The component does not become transparent — instead, a color bleeds through from behind it, creating a darkening or coloring effect. You control this color through the `underlayColor` prop.

`TouchableHighlight` has one important constraint: it must contain exactly **one child**. It cannot have multiple children directly — if you need multiple elements, you must wrap them in a single `View`.

The underlayColor is typically chosen to be a slightly darker shade of the component's background color, simulating the natural darkening effect of a physical button being pressed. On iOS, list cells with a white background conventionally use a light grey underlay (`#eee`), which is a familiar and expected interaction pattern for iOS users.

**When to use it:** Use `TouchableHighlight` for list items (especially on iOS where the grey-on-white press style is a platform convention), navigation items, and components where you want a darkening or coloring effect rather than a fade.

```jsx
import { TouchableHighlight, View, Text, StyleSheet } from 'react-native';

const MENU_ITEMS = ['Dashboard', 'Profile', 'Messages', 'Settings', 'Logout'];

export default function NavigationMenu() {
  return (
    <View style={styles.menu}>
      {MENU_ITEMS.map((item, index) => (
        <TouchableHighlight
          key={item}
          underlayColor="#e8e8e8"   // grey underlay on press — iOS convention
          onPress={() => console.log(`Navigate to ${item}`)}
          style={[
            styles.menuItem,
            index < MENU_ITEMS.length - 1 && styles.bordered,
            item === 'Logout' && styles.danger,
          ]}
        >
          {/* Must be a single child — wrap in View */}
          <View style={styles.row}>
            <Text style={[styles.itemText, item === 'Logout' && styles.dangerText]}>
              {item}
            </Text>
            <Text style={styles.chevron}>›</Text>
          </View>
        </TouchableHighlight>
      ))}
    </View>
  );
}

const styles = StyleSheet.create({
```

## 11. `TouchableWithoutFeedback` — Silent, Invisible Touch

`TouchableWithoutFeedback` is the most minimal of all the touch components. It wraps a child in a touchable area but provides **absolutely no visual feedback** whatsoever — no opacity change, no color change, no animation of any kind. The press is detected and the callback fires, but the user sees nothing change.

This might seem counterintuitive for a button, and indeed it should almost never be used for actual buttons, as user interface guidelines across all platforms universally agree that interactive elements must provide visual feedback. Without feedback, users do not know if their press was registered, which creates confusion and frustration.

Its legitimate use cases are specific and narrow: dismissing a keyboard when the user taps outside a text input, detecting taps on non-interactive areas for analytics purposes, or wrapping components that already manage their own visual feedback internally.

**When to use it:** Use `TouchableWithoutFeedback` primarily to dismiss the keyboard on outside taps. For almost any other use case, prefer `Pressable` or `TouchableOpacity`.

```jsx
import {
  TouchableWithoutFeedback,
  Keyboard,
  View,
  TextInput,
  Text,
  StyleSheet,
} from 'react-native';

export default function KeyboardDismissScreen() {
  return (
    // Tapping anywhere outside the TextInput dismisses the keyboard
    <TouchableWithoutFeedback onPress={Keyboard.dismiss} accessible={false}>
      <View style={styles.screen}>

        <Text style={styles.title}>Tap outside to dismiss keyboard</Text>

        <TextInput
          style={styles.input}
          placeholder="Type something..."
          placeholderTextColor="#aaa"
        />

        <TextInput
          style={styles.input}
          placeholder="And another field..."
          placeholderTextColor="#aaa"
        />

      </View>
    </TouchableWithoutFeedback>
  );
}

const styles = StyleSheet.create({
  screen: {
    flex: 1,
    backgroundColor: '#f9f9f9',
    padding: 24,
    justifyContent: 'center',
  },
  title: {
    fontSize: 18,
    fontWeight: '600',
    color: '#333',
    marginBottom: 24,
    textAlign: 'center',
  },
  input: {
```

## 12. `TouchableNativeFeedback` — Android's Ripple Effect

`TouchableNativeFeedback` is an **Android-exclusive** component that leverages the platform's native Material Design **ripple effect**. When a user taps a component wrapped in `TouchableNativeFeedback`, an animated ripple — a wave of color expanding outward from the point of touch — spreads across the component. This is the same interaction pattern seen in every native Android application, and using it makes your app feel immediately at home on Android devices.

The ripple is configured using `TouchableNativeFeedback.Ripple(color, borderless)` — the first argument sets the ripple color, and the second boolean controls whether the ripple is clipped to the component's bounds (false) or spreads beyond them in a circle (true, useful for icon buttons).

Because this component does not exist on iOS, you must either use it only in Android-specific code or create a platform-conditional component that uses `TouchableNativeFeedback` on Android and `TouchableOpacity` on iOS.

**When to use it:** Use `TouchableNativeFeedback` on Android for any component that should feel natively Android — Material Design cards, action items, list rows, and FABs (floating action buttons).

```jsx
import {
  TouchableNativeFeedback,
  View,
  Text,
  Platform,
  StyleSheet,
} from 'react-native';

// A smart wrapper that uses the right component per platform
function NativeTouchable({ onPress, style, children }) {
  if (Platform.OS === 'android') {
    return (
      <TouchableNativeFeedback
        onPress={onPress}
        background={TouchableNativeFeedback.Ripple('#00000020', false)}
        useForeground={true}   // ripple on top of content, not behind
      >
        <View style={style}>{children}</View>
      </TouchableNativeFeedback>
    );
  }
  // Fallback for iOS
  const { TouchableOpacity } = require('react-native');
  return (
    <TouchableOpacity onPress={onPress} activeOpacity={0.75}>
      <View style={style}>{children}</View>
    </TouchableOpacity>
  );
}

export default function MaterialButtons() {
  return (
    <View style={styles.container}>

      <NativeTouchable onPress={() => console.log('Like')} style={styles.card}>
        <Text style={styles.cardText}>👍  Like this Post</Text>
      </NativeTouchable>

      <NativeTouchable onPress={() => console.log('Share')} style={styles.card}>
        <Text style={styles.cardText}>🔗  Share with Friends</Text>
      </NativeTouchable>

      {/* Borderless ripple — great for icon buttons */}
      <TouchableNativeFeedback
        onPress={() => console.log('Favorite')}
        background={TouchableNativeFeedback.Ripple('#FF6B6B40', true)}
      >
        <View style={styles.iconBtn}>
          <Text style={styles.iconText}>♥</Text>
        </View>
      </TouchableNativeFeedback>

    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', gap: 16 },
  card: {
    width: 300,
    padding: 18,
    backgroundColor: '#fff',
    borderRadius: 12,
    elevation: 3,
    overflow: 'hidden',  // clips the ripple to the card bounds
  },
  cardText: { fontSize: 16, color: '#333', fontWeight: '500' },
  iconBtn: {
    width: 56,
    height: 56,
    borderRadius: 28,
    backgroundColor: '#fff',
    justifyContent: 'center',
    alignItems: 'center',
    elevation: 2,
  },
  iconText: { fontSize: 24, color: '#FF6B6B' },
});
```

## 13. `Button` — The Simplest Possible Button

The `Button` component is the simplest, most opinionated button in React Native. It renders a platform-native button — a standard Android material button on Android, and the standard iOS-style text button on iOS — and accepts only a small handful of props: `title` (the button text), `onPress` (the press handler), `color` (the button color), and `disabled`.

This simplicity is also its greatest limitation. You cannot style a `Button` with a custom border-radius, custom font, gradient background, or custom layout. What you see is what you get. For any degree of design customization, you will need to build your own button using `Pressable` or `TouchableOpacity`.

**When to use it:** Use `Button` for prototyping, internal tools, simple utility screens, or anywhere that a platform-native look is desired and custom styling is not required.

```jsx
import { View, Button, Alert, StyleSheet } from 'react-native';

export default function ButtonDemo() {
  return (

      <Button
        title="Delete Account"

      {/* Disabled state */}
      <Button
        title="Unavailable Feature"
        disabled={true}
        onPress={() => {}}
      />

    </View>

}

const styles = StyleSheet.create({
```

## 14. `Switch` — The Toggle Control

`Switch` renders the familiar on/off toggle control — the same component you see in every settings screen across both iOS and Android. It is a **controlled component**, meaning it does not maintain its own internal state. You must provide the current value through the `value` prop and update it yourself in the `onValueChange` callback. This is consistent with how React's controlled input model works.

`Switch` also accepts `trackColor` (an object specifying separate colors for the false/off and true/on states), `thumbColor` (the color of the circular thumb), and `ios_backgroundColor` (the background color when the switch is off, specifically for iOS).

**When to use it:** Use `Switch` for binary settings — enabling notifications, toggling dark mode, activating features, accepting terms and conditions, and any other on/off preference.

```jsx
import { useState } from 'react';
import { View, Switch, Text, StyleSheet } from 'react-native';

const SETTINGS = [
  { id: 'notifications', label: 'Push Notifications' },
  { id: 'darkMode', label: 'Dark Mode' },
  { id: 'locationSharing', label: 'Location Sharing' },
  { id: 'analytics', label: 'Usage Analytics' },
  { id: 'biometric', label: 'Biometric Login' },
];

export default function ToggleSettings() {
  // State object tracking each setting's value
  const [values, setValues] = useState({
    notifications: true,
    darkMode: false,
    locationSharing: false,
    analytics: true,
    biometric: false,
  });

  const toggle = (id) => {
    setValues((prev) => ({ ...prev, [id]: !prev[id] }));
  };

  return (
    <View style={styles.container}>
      <Text style={styles.heading}>App Preferences</Text>

      {SETTINGS.map((setting) => (
        <View key={setting.id} style={styles.row}>
          <Text style={styles.label}>{setting.label}</Text>
          <Switch
            value={values[setting.id]}
            onValueChange={() => toggle(setting.id)}
            trackColor={{
              false: '#ddd',       // track color when OFF
              true: '#4CAF50',     // track color when ON
            }}
            thumbColor={values[setting.id] ? '#fff' : '#f0f0f0'}
          />
        </View>
      ))}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f2f2f7', padding: 16 },
  heading: {
    fontSize: 22,
    fontWeight: '700',
    color: '#1c1c1e',
    marginBottom: 20,
    paddingHorizontal: 4,
  },
  row: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    backgroundColor: '#fff',
    paddingHorizontal: 16,
    paddingVertical: 14,
    marginBottom: 1,
    borderRadius: 10,
  },
  label: { fontSize: 16, color: '#1c1c1e' },
});
```

## 15. `TextInput` — Accepting Text from the User

`TextInput` is the component through which users type text into your application. It renders a native text input field and is one of the most complex and feature-rich components in React Native, supporting dozens of props for controlling keyboard type, autocorrection, secure text entry, multiline input, placeholder text, input validation, and much more.

Like `Switch`, `TextInput` is typically used as a **controlled component**: you provide the current value through the `value` prop and update state in the `onChangeText` callback. This gives you complete control over the input's content at all times.

Several of its props deserve special attention. The `keyboardType` prop tells the operating system what kind of keyboard to show — `email-address` opens a keyboard optimized for email entry (with an `@` key), `numeric` shows only numbers, `phone-pad` shows a dial pad. The `secureTextEntry` prop obscures typed characters with dots, essential for password fields. The `multiline` prop allows the input to expand across multiple lines, necessary for message composition or note-taking. The `returnKeyType` prop controls what text the Return/Enter key on the keyboard displays (`done`, `next`, `search`, `go`).

**When to use it:** Use `TextInput` for login forms, search bars, chat composition, note editors, filter fields, any form that requires user-typed input.

```jsx
import { useState } from 'react';
import {
  View,
  TextInput,
  Text,
  TouchableOpacity,
  StyleSheet,
  KeyboardAvoidingView,
  Platform,
} from 'react-native';

export default function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [note, setNote] = useState('');
  const [showPassword, setShowPassword] = useState(false);
  const [emailError, setEmailError] = useState('');

  const validateEmail = (text) => {
    setEmail(text);
    if (text.length > 0 && !text.includes('@')) {
      setEmailError('Please enter a valid email address');
    } else {
      setEmailError('');
    }
  };

  return (
    // KeyboardAvoidingView pushes content up when keyboard appears
    <KeyboardAvoidingView
      style={styles.container}
      behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
    >
      <Text style={styles.title}>Sign In</Text>

      {/* Email field */}
      <View style={styles.fieldWrapper}>
        <Text style={styles.fieldLabel}>Email Address</Text>
        <TextInput
          style={[styles.input, emailError ? styles.inputError : null]}
          value={email}
          onChangeText={validateEmail}
          placeholder="you@example.com"
          placeholderTextColor="#aaa"
          keyboardType="email-address"     // shows @ key on keyboard
          autoCapitalize="none"            // no auto-capitalization
          autoCorrect={false}              // disable autocorrect for email
          returnKeyType="next"             // shows 'Next' on keyboard
        />
        {emailError ? <Text style={styles.errorText}>{emailError}</Text> : null}
      </View>

      {/* Password field */}
      <View style={styles.fieldWrapper}>
        <Text style={styles.fieldLabel}>Password</Text>
        <View style={styles.passwordWrapper}>
          <TextInput
            style={[styles.input, styles.passwordInput]}
            value={password}
            onChangeText={setPassword}
            placeholder="Enter your password"
            placeholderTextColor="#aaa"
            secureTextEntry={!showPassword}  // hides text as dots
            autoCapitalize="none"
            autoCorrect={false}
            returnKeyType="done"
          />
          <TouchableOpacity
            style={styles.eyeBtn}
            onPress={() => setShowPassword(!showPassword)}
          >
            <Text style={styles.eyeIcon}>{showPassword ? '🙈' : '👁️'}</Text>
          </TouchableOpacity>
        </View>
      </View>

      {/* Multiline note field */}
      <View style={styles.fieldWrapper}>
        <Text style={styles.fieldLabel}>Note (optional)</Text>
        <TextInput
          style={[styles.input, styles.multilineInput]}
          value={note}
          onChangeText={setNote}
          placeholder="Add a note about this login..."
          placeholderTextColor="#aaa"
          multiline={true}           // allows multiple lines
          numberOfLines={4}          // initial height on Android
          textAlignVertical="top"    // starts text from top (Android)
          maxLength={200}
        />
        <Text style={styles.charCount}>{note.length}/200</Text>
      </View>

      {/* Submit button */}
      <TouchableOpacity
        style={[styles.submitBtn, (!email || !password) && styles.submitDisabled]}
        onPress={() => console.log('Login:', email, password)}
        disabled={!email || !password}
      >
        <Text style={styles.submitLabel}>Sign In</Text>
      </TouchableOpacity>

    </KeyboardAvoidingView>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f9f9f9',
    padding: 24,
    justifyContent: 'center',
  },
  title: {
    fontSize: 28,
    fontWeight: '800',
    color: '#1a1a2e',
    marginBottom: 28,
  },
  fieldWrapper: { marginBottom: 18 },
  fieldLabel: {
    fontSize: 13,
    fontWeight: '600',
    color: '#555',
    marginBottom: 6,

    letterSpacing: 0.5,
  },
  input: {

    borderColor: '#e0e0e0',

    paddingHorizontal: 14,
    paddingVertical: 12,

  },
  inputError: {
    borderColor: '#e74c3c',
  },
  errorText: {

    marginTop: 4,
    marginLeft: 2,
  },
  passwordWrapper: {

  },

  },
```
