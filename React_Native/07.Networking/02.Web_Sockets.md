### The Problem That WebSockets Solve

To understand WebSockets, you must first understand the traditional way applications communicate over the internet — which is through the **HTTP request-response model**. In this model, the client (your mobile app) always initiates the conversation. It sends a request to the server, the server sends a response back, and the connection closes. This works perfectly well for fetching a page of products or submitting a form. However, consider applications like a live chat messenger, a multiplayer game, a real-time stock ticker, or a collaborative whiteboard. In these scenarios, the server needs to push new data to the client at any moment — not just when the client happens to ask. Trying to simulate this with HTTP leads to a technique called **polling**, where the client sends a request to the server every few seconds asking "is there anything new?" This is enormously wasteful — it floods the server with redundant requests and introduces unnecessary latency.

**WebSockets** solve this problem elegantly by establishing a single, persistent, bidirectional communication channel between the client and the server. Once the WebSocket connection is established, both the client and the server can send messages to each other at any time, freely and independently, without the overhead of establishing a new connection for every message.

### The WebSocket Handshake — How the Connection Is Born

A WebSocket connection begins its life as a perfectly ordinary HTTP request. This is by design — it allows WebSockets to work through existing web infrastructure like proxies and load balancers. The client sends an HTTP request with a special `Upgrade` header, signalling that it wishes to upgrade the connection protocol from HTTP to WebSocket. If the server supports WebSockets and agrees to the upgrade, it responds with a `101 Switching Protocols` status code. From this moment forward, the HTTP connection is transformed into a WebSocket connection, and the two parties can communicate freely.

```

  │                                          │
  │  HTTP GET /chat                          │
  │  Upgrade: websocket                      │
  │  Connection: Upgrade                     │
```

### WebSocket in React Native

React Native exposes the browser-standard `WebSocket` API directly in JavaScript, which means if you have ever worked with WebSockets in a web browser environment, the API will feel immediately familiar. There is no additional library required for basic WebSocket usage — it is built right in. The `WebSocket` object has four event handlers: `onopen` (called when the connection is successfully established), `onmessage` (called whenever a message arrives from the server), `onerror` (called when an error occurs), and `onclose` (called when the connection is closed, either by the client or the server).

```
WebSocket Object Lifecycle
┌─────────────────────────────────────────────────────┐
│                                                     │
│   new WebSocket("wss://server.com/chat")            │
│             │                                       │
│             ▼                                       │
│       [ CONNECTING ]  ◄── Performing HTTP Upgrade   │
│             │                                       │
│      onopen fires                                   │
│             │                                       │
│             ▼                                       │
│         [ OPEN ]      ◄── Fully connected           │
│        ╱       ╲                                    │
│  send()         onmessage                           │
│  (client→server) (server→client)                   │
│        ╲       ╱                                    │
│             │                                       │
│    close() or error                                 │
│             │                                       │
│             ▼                                       │
│       [ CLOSING ]     ◄── Teardown in progress      │
│             │                                       │
│      onclose fires                                  │
│             │                                       │
│             ▼                                       │
│        [ CLOSED ]     ◄── Connection terminated     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### A Complete Real-World Example: Live Chat Application

The following example demonstrates a fully functional real-time chat screen. It connects to a public WebSocket echo server for demonstration purposes (an echo server simply bounces back every message you send to it), and showcases all key aspects of WebSocket usage in React Native — connection management, sending messages, receiving messages, handling reconnection, and proper cleanup.

javascript

```javascript
import React, { useState, useEffect, useRef, useCallback } from 'react';
import {
  View,
  Text,
  TextInput,
  FlatList,
  StyleSheet,
  TouchableOpacity,
  KeyboardAvoidingView,
  Platform,
} from 'react-native';

// ─── Constants ──────────────────────────────────────────────────────────────
// wss:// is the secure version of WebSocket (like https:// for HTTP).
// Always prefer wss:// in production applications.
const WS_URL = 'wss://echo.websocket.org';

const ChatScreen = () => {
  const [messages, setMessages] = useState([]);
  const [inputText, setInputText] = useState('');
  const [status, setStatus] = useState('Disconnected'); // UI status label
  const wsRef = useRef(null);        // Persistent ref to the WebSocket instance
  const reconnectTimer = useRef(null); // Ref to manage reconnection timeout
  const flatListRef = useRef(null);

  // ─── Helper: Add a message to the chat list ────────────────────────────────
  const addMessage = useCallback((text, fromServer = false) => {
    setMessages((prev) => [
      ...prev,
      {
        id: Date.now().toString(),
        text,
        fromServer,
        time: new Date().toLocaleTimeString(),
      },
    ]);
  }, []);

  // ─── Core: Connect to WebSocket ────────────────────────────────────────────
  // This function creates a new WebSocket instance and wires up all
  // four event handlers. It is called on mount and on every reconnect attempt.
  const connect = useCallback(() => {
    // Don't create a duplicate connection if one is already open or connecting.
    if (
      wsRef.current &&
      (wsRef.current.readyState === WebSocket.OPEN ||
        wsRef.current.readyState === WebSocket.CONNECTING)
    ) {
      return;
    }

    setStatus('Connecting...');
    const ws = new WebSocket(WS_URL);

    // ── onopen ──────────────────────────────────────────────────────────────
    // Fired when the HTTP upgrade handshake completes and the WebSocket
    // connection is fully established and ready to use.
    ws.onopen = () => {
      setStatus('Connected 🟢');
      addMessage('✅ Connected to server', true);
    };

    // ── onmessage ───────────────────────────────────────────────────────────
    // Fired every time the server pushes a message. The message payload
    // arrives as `event.data`. It can be a plain string, a JSON string,
    // or even binary data (ArrayBuffer / Blob).
    ws.onmessage = (event) => {
      try {
        // Attempt to parse as JSON first (real servers often send JSON)
        const parsed = JSON.parse(event.data);
        addMessage(`Server: ${parsed.text || event.data}`, true);
      } catch {
        // If it's not JSON, treat it as a plain string
        addMessage(`Server: ${event.data}`, true);
      }
    };

    // ── onerror ─────────────────────────────────────────────────────────────
    // Fired when a connection error occurs. Note that an error event is
    // always followed by a close event, so reconnection logic lives in onclose.
    ws.onerror = (error) => {
      console.warn('WebSocket error:', error.message);
      setStatus('Error ⚠️');
    };

    // ── onclose ─────────────────────────────────────────────────────────────
    // Fired when the connection is closed — either intentionally (ws.close())
    // or due to a network error. The `event.code` and `event.reason` fields
    // provide context about why the connection was closed.
    // Code 1000 means "Normal Closure" (intentional).
    ws.onclose = (event) => {
      setStatus('Disconnected 🔴');
      addMessage(`❌ Disconnected (code: ${event.code})`, true);

      // ── Automatic Reconnection ───────────────────────────────────────────
      // If the closure was NOT intentional (not code 1000), schedule a
      // reconnection attempt after 3 seconds. This is a simple linear
      // backoff strategy. Production apps often use exponential backoff.
      if (event.code !== 1000) {
        addMessage('⏳ Reconnecting in 3 seconds...', true);
        reconnectTimer.current = setTimeout(() => {
          connect();
        }, 3000);
      }
    };

    wsRef.current = ws;
  }, [addMessage]);

  // ─── Lifecycle: Connect on mount, disconnect on unmount ───────────────────
  useEffect(() => {
    connect();

    // Cleanup function — this runs when the component is unmounted.
    // Always close the WebSocket and clear any pending timers to avoid
    // memory leaks and dangling connections.
    return () => {
      if (reconnectTimer.current) {
        clearTimeout(reconnectTimer.current);
      }
      if (wsRef.current) {
        // Code 1000 = Normal Closure. Passing this prevents auto-reconnect.
        wsRef.current.close(1000, 'Component unmounted');
      }
    };
  }, [connect]);

  // ─── Sending a message ────────────────────────────────────────────────────
  // Before calling ws.send(), always verify that the connection is OPEN.
  // Calling send() on a CONNECTING, CLOSING, or CLOSED socket will throw.
  const sendMessage = useCallback(() => {
    const trimmed = inputText.trim();
    if (!trimmed) return;

    if (wsRef.current && wsRef.current.readyState === WebSocket.OPEN) {
      // We send JSON-encoded data — this is the most common pattern
      // in real applications, as it allows structured data exchange.
      const payload = JSON.stringify({ text: trimmed, timestamp: Date.now() });
      wsRef.current.send(payload);
      addMessage(`You: ${trimmed}`, false);
      setInputText('');
    } else {
      addMessage('⚠️ Cannot send — not connected.', true);
    }
  }, [inputText, addMessage]);

  // ─── Render ───────────────────────────────────────────────────────────────
  return (
    <KeyboardAvoidingView
      style={styles.container}
      behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
    >
      {/* Status Bar */}
      <View style={styles.statusBar}>
        <Text style={styles.statusText}>Status: {status}</Text>
        <TouchableOpacity onPress={connect} style={styles.reconnectBtn}>
          <Text style={styles.reconnectText}>Reconnect</Text>
        </TouchableOpacity>
      </View>

      {/* Message List */}
      <FlatList
        ref={flatListRef}
        data={messages}
        keyExtractor={(item) => item.id}
        onContentSizeChange={() =>
          flatListRef.current?.scrollToEnd({ animated: true })
        }
        renderItem={({ item }) => (
          <View
            style={[
              styles.bubble,
              item.fromServer ? styles.serverBubble : styles.clientBubble,
            ]}
          >
            <Text style={styles.bubbleText}>{item.text}</Text>
            <Text style={styles.timeText}>{item.time}</Text>
          </View>
        )}
      />

      {/* Input Area */}
      <View style={styles.inputRow}>
        <TextInput
          style={styles.input}
          value={inputText}
          onChangeText={setInputText}
          placeholder="Type a message..."
          placeholderTextColor="#999"
          onSubmitEditing={sendMessage}
          returnKeyType="send"
        />
        <TouchableOpacity style={styles.sendBtn} onPress={sendMessage}>
          <Text style={styles.sendText}>Send ➤</Text>
        </TouchableOpacity>
      </View>
    </KeyboardAvoidingView>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#0D1117' },
  statusBar: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    padding: 12,
    backgroundColor: '#161B22',
    borderBottomWidth: 1,
    borderBottomColor: '#30363D',
  },
  statusText: { color: '#8B949E', fontSize: 13 },
  reconnectBtn: {
    backgroundColor: '#238636',
    paddingHorizontal: 12,
    paddingVertical: 6,
    borderRadius: 6,
  },
  reconnectText: { color: '#FFF', fontSize: 12, fontWeight: '600' },
  bubble: {
    margin: 8,
    padding: 12,
    borderRadius: 12,
    maxWidth: '80%',
  },
  serverBubble: {
    backgroundColor: '#21262D',
    alignSelf: 'flex-start',
    borderBottomLeftRadius: 2,
  },
  clientBubble: {
    backgroundColor: '#1F6FEB',
    alignSelf: 'flex-end',
    borderBottomRightRadius: 2,
  },
  bubbleText: { color: '#E6EDF3', fontSize: 14 },
  timeText: { color: '#8B949E', fontSize: 10, marginTop: 4, textAlign: 'right' },
  inputRow: {
    flexDirection: 'row',
    padding: 10,
    backgroundColor: '#161B22',
    borderTopWidth: 1,
    borderTopColor: '#30363D',
  },
  input: {
    flex: 1,
    backgroundColor: '#21262D',
    color: '#E6EDF3',
    borderRadius: 8,
    paddingHorizontal: 14,
    paddingVertical: 10,
    marginRight: 8,
    fontSize: 14,
  },
  sendBtn: {
    backgroundColor: '#1F6FEB',
    paddingHorizontal: 16,
    borderRadius: 8,
    justifyContent: 'center',
  },
  sendText: { color: '#FFF', fontWeight: 'bold', fontSize: 14 },
});

export default ChatScreen;
```

### Understanding the readyState Property

One of the most important aspects of working with WebSockets responsibly is checking the `readyState` property before performing operations. The `readyState` can be one of four numeric constants: `0` (CONNECTING), `1` (OPEN), `2` (CLOSING), or `3` (CLOSED). You must only call `ws.send()` when `readyState` is `1` (OPEN). Attempting to send a message in any other state will throw an error and potentially crash your application.

```
readyState Values at a Glance
┌───────────┬────────┬─────────────────────────────────────┐
│  Constant │ Value  │ Meaning                             │
├───────────┼────────┼─────────────────────────────────────┤
│ CONNECTING│   0    │ Handshake in progress               │
│ OPEN      │   1    │ Connection live — safe to send()    │
│ CLOSING   │   2    │ Close handshake in progress         │
│ CLOSED    │   3    │ Connection terminated               │
└───────────┴────────┴─────────────────────────────────────┘

Check before sending:
if (ws.readyState === WebSocket.OPEN) {
   ws.send(data);  ✅ Safe
}
```

### Combining Connectivity Status with WebSockets

A truly robust production application combines both concepts together. The NetInfo library can watch for the device going offline and pause WebSocket operations accordingly, then trigger a reconnection attempt the moment the network returns.

```
App Architecture: NetInfo + WebSocket Combined
┌─────────────────────────────────────────────────┐
│                   React Native App              │
│                                                 │
│  ┌──────────────┐       ┌──────────────────┐    │
│  │   NetInfo    │──────►│  WebSocket Mgr   │    │
│  │  Listener   │       │  (connect/       │    │
│  │             │       │   reconnect/     │    │
│  │ isOnline?   │       │   disconnect)    │    │
│  └──────────────┘       └────────┬─────────┘    │
│                                  │              │
│                                  ▼              │
│                       ┌──────────────────┐      │
│                       │  WebSocket Conn  │      │
│                       │  wss://...       │      │
│                       └────────┬─────────┘      │
└────────────────────────────────┼────────────────┘
                                 │
                    ┌────────────▼────────────┐
                    │     Remote Server       │
                    │  (Chat / Game / Feed)   │
                    └─────────────────────────┘

Flow:
  Network drops → NetInfo fires → WebSocket closes
  Network returns → NetInfo fires → WebSocket reconnects
```

javascript

```javascript
// Tying it all together — NetInfo + WebSocket auto-reconnect

useEffect(() => {
  const unsubscribe = NetInfo.addEventListener((state) => {
    if (state.isConnected && state.isInternetReachable) {
      // Network is back — reconnect the WebSocket
      connect();
    } else {
      // Network is gone — close the socket cleanly to avoid zombie connections
      if (wsRef.current && wsRef.current.readyState === WebSocket.OPEN) {
        wsRef.current.close(1001, 'Network lost');
      }
    }
  });

  return () => unsubscribe();
}, [connect]);
```
