# React Native Networking: A Deep Dive
### What Is Connectivity Status and Why Does It Matter?

Imagine you are building a mobile application that fetches a list of products from a remote server. Everything works beautifully when the user has a strong internet connection. But what happens when the user steps into an elevator, travels through a tunnel, or simply runs out of mobile data? Without proper connectivity awareness, your application will either crash silently, throw cryptic errors, or leave the user staring at an infinite loading spinner — none of which is acceptable in a production-grade application.

This is precisely the problem that **Connectivity Status** monitoring solves. In React Native, the `@react-native-community/netinfo` library provides a robust and reliable way to observe the state of the device's network connection in real time. It tells you not only whether the device is connected to the internet, but also _how_ it is connected — whether through WiFi, cellular data (2G, 3G, 4G, 5G), Ethernet, Bluetooth, or some other means.

### How It Works Under the Hood

The NetInfo library acts as a bridge between your JavaScript code and the native APIs of the underlying operating system. On Android, it communicates with the `ConnectivityManager` system service, which is a core Android API that tracks all network interfaces. On iOS, it uses the `SCNetworkReachability` API from Apple's SystemConfiguration framework. These native systems continuously emit events whenever the network state changes — for example, when WiFi is disconnected or when the device switches from WiFi to cellular data. The NetInfo library listens to these native events, translates them into a JavaScript-friendly format, and delivers them to your React Native application through the React Native Bridge.


### The Shape of Network State

When you query or subscribe to network state changes, the library returns an object that describes the current connectivity in rich detail. The most important fields are `isConnected`, which is a simple boolean telling you whether any network connection is available at all; `type`, which describes the kind of connection (such as `wifi`, `cellular`, `ethernet`, or `none`); and `isInternetReachable`, which goes one step further and tells you whether the internet is actually reachable — because a device can be connected to a WiFi network that has no internet access.

```
NetInfo State Object
┌──────────────────────────────────────────┐
│  {                                       │
│    type: "wifi"           ◄─ Connection  │
│    isConnected: true      ◄─ Has network │
│    isInternetReachable: true ◄─ Has web  │
│    details: {                            │
│      isConnectionExpensive: false,       │
│      ssid: "HomeNetwork",                │
│      strength: 85,                       │
│      ipAddress: "192.168.1.5"            │
│    }                                     │
│  }                                       │
└──────────────────────────────────────────┘
```

### A Real-World Example

Let us now look at a complete, practical example that demonstrates connectivity monitoring in action. In this example, we build a component that fetches data from an API but gracefully handles offline scenarios by showing a banner and queuing the request for when connectivity is restored.

javascript

```javascript
// First, install the library:
// npm install @react-native-community/netinfo

import React, { useState, useEffect, useCallback } from 'react';
import {
  View,
  Text,
  FlatList,
  StyleSheet,
  ActivityIndicator,
  TouchableOpacity,
} from 'react-native';
import NetInfo from '@react-native-community/netinfo';

const ProductListScreen = () => {
  const [products, setProducts] = useState([]);
  const [isOnline, setIsOnline] = useState(true);
  const [connectionType, setConnectionType] = useState('unknown');
  const [isLoading, setIsLoading] = useState(false);
  const [pendingFetch, setPendingFetch] = useState(false);

  // ─── Step 1: Subscribe to network changes ──────────────────────────────────
  // NetInfo.addEventListener returns an unsubscribe function.
  // We call it inside useEffect so that we clean up the listener
  // when the component unmounts, preventing memory leaks.
  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener((state) => {
      const online = state.isConnected && state.isInternetReachable;
      setIsOnline(online);
      setConnectionType(state.type);

      // ─── Step 2: Execute pending requests when connectivity returns ─────────
      // If there was a failed/pending fetch and we just came back online,
      // automatically retry the fetch without user intervention.
      if (online && pendingFetch) {
        fetchProducts();
        setPendingFetch(false);
      }
    });

    // Cleanup: remove the listener when the component is unmounted
    return () => unsubscribe();
  }, [pendingFetch]);

  // ─── Step 3: Fetch products, with offline awareness ────────────────────────
  const fetchProducts = useCallback(async () => {
    // Before making a network call, check connectivity one more time.
    // This is called a "one-time fetch" — it returns the current state
    // without subscribing to future changes.
    const state = await NetInfo.fetch();

    if (!state.isConnected) {
      // Device is offline. Mark the fetch as pending so it retries
      // automatically when connectivity is restored.
      setPendingFetch(true);
      return;
    }

    try {
      setIsLoading(true);
      const response = await fetch('https://fakestoreapi.com/products');
      const data = await response.json();
      setProducts(data);
    } catch (error) {
      console.error('Fetch error:', error);
    } finally {
      setIsLoading(false);
    }
  }, []);

  // Initial data load
  useEffect(() => {
    fetchProducts();
  }, []);

  return (
    <View style={styles.container}>

      {/* ─── Connectivity Banner ─────────────────────────────────────────── */}
      {/* This banner slides into view only when the device goes offline.     */}
      {!isOnline && (
        <View style={styles.offlineBanner}>
          <Text style={styles.offlineText}>
            📡 You are offline. Will retry when connected.
          </Text>
        </View>
      )}

      {/* ─── Connection Type Badge ───────────────────────────────────────── */}
      <View style={styles.badge}>
        <Text style={styles.badgeText}>
          Network: {connectionType.toUpperCase()} {isOnline ? '🟢' : '🔴'}
        </Text>
      </View>

      {/* ─── Loading / Content ───────────────────────────────────────────── */}
      {isLoading ? (
        <ActivityIndicator size="large" color="#6200EE" />
      ) : (
        <FlatList
          data={products}
          keyExtractor={(item) => item.id.toString()}
          renderItem={({ item }) => (
            <View style={styles.productCard}>
              <Text style={styles.productTitle}>{item.title}</Text>
              <Text style={styles.productPrice}>${item.price}</Text>
            </View>
          )}
        />
      )}

      {/* ─── Manual Retry Button ─────────────────────────────────────────── */}
      <TouchableOpacity style={styles.retryBtn} onPress={fetchProducts}>
        <Text style={styles.retryText}>↺ Retry</Text>
      </TouchableOpacity>

    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#F8F8F8' },
  offlineBanner: {
    backgroundColor: '#B00020',
    padding: 10,
    alignItems: 'center',
  },
  offlineText: { color: '#FFF', fontWeight: '600' },
  badge: {
    backgroundColor: '#E8E8E8',
    padding: 8,
    alignItems: 'center',
  },
  badgeText: { fontSize: 13, color: '#333' },
  productCard: {
    margin: 10,
    padding: 14,
    backgroundColor: '#FFF',
    borderRadius: 8,
    elevation: 2,
  },
  productTitle: { fontSize: 14, color: '#333', marginBottom: 4 },
  productPrice: { fontSize: 16, fontWeight: 'bold', color: '#6200EE' },
  retryBtn: {
    margin: 16,
    backgroundColor: '#6200EE',
    padding: 14,
    borderRadius: 8,
    alignItems: 'center',
  },
  retryText: { color: '#FFF', fontWeight: 'bold', fontSize: 16 },
});

export default ProductListScreen;
```

In the example above, notice the elegant interplay between `NetInfo.addEventListener` and `NetInfo.fetch`. The `addEventListener` function runs a callback _every time_ the network state changes, making it perfect for reactive UI updates like showing or hiding the offline banner. The `NetInfo.fetch` function, on the other hand, is a one-time snapshot — it checks the current network state at a single point in time, which is ideal for imperative checks right before making an API call.
