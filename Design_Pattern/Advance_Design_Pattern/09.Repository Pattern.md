# Repository Pattern for Building Enterprise ERP Systems in Go

## Introduction: The Problem of Data Access Sprawl

When you build an Enterprise Resource Planning system, your business logic needs to interact with data persistence. Your `OrderService` needs to save orders to a database. Your `CustomerService` needs to retrieve customer information. Your `InventoryManager` needs to update stock levels. The natural, straightforward approach is to embed database queries directly in your business logic. Inside `OrderService.CreateOrder()`, you write SQL to insert into the orders table. Inside `CustomerService.GetCustomer()`, you write a SELECT statement.

This approach feels simple at first. But as your ERP system grows, this direct coupling between business logic and data access creates serious problems. Your business logic becomes contaminated with database concerns—SQL queries, connection management, transaction handling. Testing becomes difficult because your services cannot function without a real database. Changing database technology requires modifying business logic throughout your codebase. Optimizing queries means touching service code that should be focused purely on business rules.

Your codebase becomes a tangled mess where domain logic and persistence logic are inseparably intertwined. A simple business operation like "place an order" involves dozens of lines of SQL scattered throughout your service methods. When you need to switch from PostgreSQL to MongoDB, or add caching, or implement sharding, you must modify service code in hundreds of places.

The Repository pattern solves this problem by introducing an abstraction layer between your domain logic and data access logic. A repository acts as an in-memory collection of domain objects, hiding the details of how those objects are persisted. Your business logic interacts with repositories using domain terminology—`GetCustomerByID()`, `SaveOrder()`, `FindProductsBySKU()`—without knowing whether data comes from SQL, NoSQL, cache, or external APIs.

Let us explore what the Repository pattern is, why it matters profoundly for ERP systems, and how to implement it effectively in Go.

## Understanding the Repository Pattern: Core Concepts

### What is a Repository?

A repository is a mechanism for encapsulating storage, retrieval, and search behavior which emulates a collection of objects. From the perspective of your domain layer, a repository is simply a collection. You add items to it, remove items from it, and query for items, just as you would with any in-memory collection.

The key difference is that repositories persist these objects somewhere—a database, file system, external API—but your domain code does not know or care about these details. The repository handles all the messy infrastructure concerns: SQL generation, connection pooling, transaction management, error handling, caching, and serialization.

```go
package repository

import (
    "fmt"
    "time"
)

// ❌ WITHOUT REPOSITORY: Business logic coupled to database

type OrderServiceWithoutRepository struct {
    db *DatabaseConnection
}

func (s *OrderServiceWithoutRepository) CreateOrder(customerID string, items []Item) error {
    // Business logic mixed with SQL
    conn := s.db.GetConnection()
    defer conn.Close()
    
    tx, _ := conn.Begin()
    
    // SQL directly in business logic - tight coupling!
    orderID := generateOrderID()
    _, err := tx.Exec(
        "INSERT INTO orders (id, customer_id, total, status, created_at) VALUES ($1, $2, $3, $4, $5)",
        orderID, customerID, calculateTotal(items), "pending", time.Now(),
    )
    
    if err != nil {
        tx.Rollback()
        return err
    }
    
    // More SQL for order items
    for _, item := range items {
        _, err = tx.Exec(
            "INSERT INTO order_items (order_id, product_id, quantity, price) VALUES ($1, $2, $3, $4)",
            orderID, item.ProductID, item.Quantity, item.Price,
        )
        if err != nil {
            tx.Rollback()
            return err
        }
    }
    
    tx.Commit()
    return nil
}

// ✅ WITH REPOSITORY: Business logic decoupled from database

// Domain model
type Order struct {

    FindByStatus(status string) ([]*Order, error)

// Business service uses repository
type OrderService struct {
    orderRepo OrderRepository
}

func NewOrderService(repo OrderRepository) *OrderService {
    return &OrderService{orderRepo: repo}
}

func (s *OrderService) CreateOrder(customerID string, items []Item) (*Order, error) {
    // Pure business logic - no SQL!
    orderItems := make([]OrderItem, 0, len(items))
    total := 0.0
    
    for _, item := range items {
        orderItems = append(orderItems, OrderItem{
            ID:        generateItemID(),
            ProductID: item.ProductID,
            Quantity:  item.Quantity,
            Price:     item.Price,
        })
        total += float64(item.Quantity) * item.Price
    }
    
    order := &Order{
        ID:         generateOrderID(),
        CustomerID: customerID,
        Items:      orderItems,
        Total:      total,
        Status:     "pending",
        CreatedAt:  time.Now(),
        UpdatedAt:  time.Now(),
    }
    
    // Repository handles all persistence details
    if err := s.orderRepo.Save(order); err != nil {
        return nil, fmt.Errorf("failed to save order: %w", err)
    }
    
    fmt.Printf("[Order Service] Order created: %s for customer %s ($%.2f)\n",
        order.ID, customerID, total)
    
    return order, nil
}

func (s *OrderService) GetCustomerOrders(customerID string) ([]*Order, error) {
    // Clean, readable domain logic
    orders, err := s.orderRepo.FindByCustomerID(customerID)
    if err != nil {
        return nil, fmt.Errorf("failed to retrieve orders: %w", err)
    }
    
    fmt.Printf("[Order Service] Found %d orders for customer %s\n", len(orders), customerID)
    return orders, nil
}

func (s *OrderService) CancelOrder(orderID string) error {
    order, err := s.orderRepo.FindByID(orderID)
    if err != nil {
        return fmt.Errorf("order not found: %w", err)
    }
    
    // Business rule validation
    if order.Status == "shipped" {
        return fmt.Errorf("cannot cancel shipped order")
    }
    
    order.Status = "cancelled"
    order.UpdatedAt = time.Now()
    
    if err := s.orderRepo.Save(order); err != nil {
        return fmt.Errorf("failed to cancel order: %w", err)
    }
    
    fmt.Printf("[Order Service] Order cancelled: %s\n", orderID)
    return nil
}

    return fmt.Sprintf("ORD-%d", time.Now().Unix())

}

    total := 0.0

}
```

### Pattern 1: Basic Repository Implementation

```go
    // Basic CRUD

    FindByID(id string) (*Customer, error)
    FindAll() ([]*Customer, error)
    Delete(id string) error
    
    // Query methods
    FindByEmail(email string) (*Customer, error)
    FindByName(firstName, lastName string) ([]*Customer, error)
    FindByCity(city string) ([]*Customer, error)
    
    // Aggregate queries
    Count() (int, error)
    Exists(id string) (bool, error)

// In-memory implementation (for testing or prototyping)
type InMemoryCustomerRepository struct {
    customers map[string]*Customer
    mu        sync.RWMutex
}

func NewInMemoryCustomerRepository() *InMemoryCustomerRepository {
    return &InMemoryCustomerRepository{
        customers: make(map[string]*Customer),
    }
}

func (repo *InMemoryCustomerRepository) Save(customer *Customer) error {
    repo.mu.Lock()
    defer repo.mu.Unlock()
    
    // Set timestamps
    if customer.CreatedAt.IsZero() {
        customer.CreatedAt = time.Now()
    }
    customer.UpdatedAt = time.Now()
    
    // Store copy to prevent external modifications
    stored := *customer
    repo.customers[customer.ID] = &stored
    
    fmt.Printf("[In-Memory Repo] Saved customer: %s %s (%s)\n",
        customer.FirstName, customer.LastName, customer.ID)
    
    return nil
}

func (repo *InMemoryCustomerRepository) FindByID(id string) (*Customer, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    customer, exists := repo.customers[id]
    if !exists {
        return nil, fmt.Errorf("customer not found: %s", id)
    }
    
    // Return copy to prevent external modifications
    result := *customer
    return &result, nil
}

func (repo *InMemoryCustomerRepository) FindAll() ([]*Customer, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    result := make([]*Customer, 0, len(repo.customers))
    for _, customer := range repo.customers {
        copied := *customer
        result = append(result, &copied)
    }
    
    fmt.Printf("[In-Memory Repo] Retrieved %d customers\n", len(result))
    return result, nil
}

func (repo *InMemoryCustomerRepository) Delete(id string) error {
    repo.mu.Lock()
    defer repo.mu.Unlock()
    
    if _, exists := repo.customers[id]; !exists {
        return fmt.Errorf("customer not found: %s", id)
    }
    
    delete(repo.customers, id)
    fmt.Printf("[In-Memory Repo] Deleted customer: %s\n", id)
    return nil
}

func (repo *InMemoryCustomerRepository) FindByEmail(email string) (*Customer, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    for _, customer := range repo.customers {
        if customer.Email == email {
            result := *customer
            return &result, nil
        }
    }
    
    return nil, fmt.Errorf("customer not found with email: %s", email)
}

func (repo *InMemoryCustomerRepository) FindByName(firstName, lastName string) ([]*Customer, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    result := make([]*Customer, 0)
    
    for _, customer := range repo.customers {
        if customer.FirstName == firstName && customer.LastName == lastName {
            copied := *customer
            result = append(result, &copied)
        }
    }
    
    return result, nil
}

func (repo *InMemoryCustomerRepository) FindByCity(city string) ([]*Customer, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    result := make([]*Customer, 0)
    
    for _, customer := range repo.customers {
        if customer.Address.City == city {
            copied := *customer
            result = append(result, &copied)
        }
    }
    
    fmt.Printf("[In-Memory Repo] Found %d customers in %s\n", len(result), city)
    return result, nil
}

func (repo *InMemoryCustomerRepository) Count() (int, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    return len(repo.customers), nil
}

func (repo *InMemoryCustomerRepository) Exists(id string) (bool, error) {
    repo.mu.RLock()
    defer repo.mu.RUnlock()
    
    _, exists := repo.customers[id]
    return exists, nil
}
```

### Pattern 2: Repository with Specification Pattern

For complex queries, combine Repository with Specification pattern to build flexible query criteria.

```go
package specification

import (
    "fmt"
    "time"
)

// Product domain model
type Product struct {
    ID          string
    Name        string
    SKU         string
    Category    string
    Price       float64
    Stock       int
    IsActive    bool
    CreatedAt   time.Time
}

// Specification interface for composable queries
type ProductSpecification interface {
    IsSatisfiedBy(product *Product) bool
}

// Concrete specifications
type ActiveProductsSpec struct{}

func (s *ActiveProductsSpec) IsSatisfiedBy(product *Product) bool {
    return product.IsActive
}

type CategorySpec struct {
    Category string
}

func (s *CategorySpec) IsSatisfiedBy(product *Product) bool {
    return product.Category == s.Category
}

type PriceRangeSpec struct {
    MinPrice float64
    MaxPrice float64
}

func (s *PriceRangeSpec) IsSatisfiedBy(product *Product) bool {
    return product.Price >= s.MinPrice && product.Price <= s.MaxPrice
}

type InStockSpec struct{}

func (s *InStockSpec) IsSatisfiedBy(product *Product) bool {
    return product.Stock > 0
}

// Composite specifications (AND, OR, NOT)
type AndSpec struct {
    Specs []ProductSpecification
}

func (s *AndSpec) IsSatisfiedBy(product *Product) bool {
    for _, spec := range s.Specs {
        if !spec.IsSatisfiedBy(product) {
            return false
        }
    }
    return true
}

type OrSpec struct {
    Specs []ProductSpecification
}

func (s *OrSpec) IsSatisfiedBy(product *Product) bool {
    for _, spec := range s.Specs {
        if spec.IsSatisfiedBy(product) {
            return true
        }
    }
    return false
}

type NotSpec struct {
    Spec ProductSpecification
}

func (s *NotSpec) IsSatisfiedBy(product *Product) bool {
    return !s.Spec.IsSatisfiedBy(product)
}

// Repository with specification support
type ProductRepository interface {
    Save(product *Product) error
    FindByID(id string) (*Product, error)
    FindAll() ([]*Product, error)
    
    // Find by specification
    FindBySpec(spec ProductSpecification) ([]*Product, error)
}

type InMemoryProductRepository struct {
    products map[string]*Product
}

func NewInMemoryProductRepository() *InMemoryProductRepository {
    return &InMemoryProductRepository{
        products: make(map[string]*Product),
    }
}

func (repo *InMemoryProductRepository) Save(product *Product) error {
    repo.products[product.ID] = product
    fmt.Printf("[Product Repo] Saved product: %s\n", product.Name)
    return nil
}

func (repo *InMemoryProductRepository) FindByID(id string) (*Product, error) {
    product, exists := repo.products[id]
    if !exists {
        return nil, fmt.Errorf("product not found: %s", id)
    }
    return product, nil
}

func (repo *InMemoryProductRepository) FindAll() ([]*Product, error) {
    result := make([]*Product, 0, len(repo.products))
    for _, product := range repo.products {
        result = append(result, product)
    }
    return result, nil
}

func (repo *InMemoryProductRepository) FindBySpec(spec ProductSpecification) ([]*Product, error) {
    result := make([]*Product, 0)
    
    for _, product := range repo.products {
        if spec.IsSatisfiedBy(product) {
            result = append(result, product)
        }
    }
    
    fmt.Printf("[Product Repo] Found %d products matching specification\n", len(result))
    return result, nil
}

// Product service using specifications
type ProductService struct {
    productRepo ProductRepository
}

func NewProductService(repo ProductRepository) *ProductService {
    return &ProductService{productRepo: repo}
}

func (s *ProductService) FindAffordableElectronics(maxPrice float64) ([]*Product, error) {
    // Build composite specification
    spec := &AndSpec{
        Specs: []ProductSpecification{
            &ActiveProductsSpec{},
            &CategorySpec{Category: "Electronics"},
            &PriceRangeSpec{MinPrice: 0, MaxPrice: maxPrice},
            &InStockSpec{},
        },
    }
    
    products, err := s.productRepo.FindBySpec(spec)
    if err != nil {
        return nil, err
    }
    
    fmt.Printf("[Product Service] Found %d affordable electronics\n", len(products))
    return products, nil
}

func (s *ProductService) FindPopularCategories() ([]*Product, error) {
    // Find active products in Electronics OR Books
    spec := &AndSpec{
        Specs: []ProductSpecification{
            &ActiveProductsSpec{},
            &OrSpec{
                Specs: []ProductSpecification{
                    &CategorySpec{Category: "Electronics"},
                    &CategorySpec{Category: "Books"},
                },
            },
        },
    }
    
    return s.productRepo.FindBySpec(spec)
}

func DemonstrateSpecificationPattern() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         REPOSITORY WITH SPECIFICATION PATTERN              ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")

    fmt.Println("\n=== Finding Popular Categories (Electronics OR Books) ===")
    popular, _ := service.FindPopularCategories()
```

### Pattern 3: Unit of Work with Repository

Unit of Work pattern coordinates multiple repositories in a single transaction.

```go
package unitofwork

import (
    "fmt"
    "time"
)

// Domain models
type Order struct {
    ID         string
    CustomerID string
    Total      float64
    Status     string
}

type OrderItem struct {
    ID        string
    OrderID   string
    ProductID string
    Quantity  int
}

type InventoryRecord struct {
    ProductID string
    Quantity  int
    Reserved  int
}

// Repositories
type OrderRepository interface {
    Save(order *Order) error
    FindByID(id string) (*Order, error)
}

type OrderItemRepository interface {
    SaveAll(items []*OrderItem) error
    FindByOrderID(orderID string) ([]*OrderItem, error)
}

type InventoryRepository interface {
    FindByProductID(productID string) (*InventoryRecord, error)
    Update(record *InventoryRecord) error
}

// Unit of Work interface
type UnitOfWork interface {
    OrderRepo() OrderRepository
    OrderItemRepo() OrderItemRepository
    InventoryRepo() InventoryRepository
    Commit() error
    Rollback() error
}

// Implementation
type TransactionalUnitOfWork struct {
    orderRepo     OrderRepository
    orderItemRepo OrderItemRepository
    inventoryRepo InventoryRepository
    committed     bool
    changes       []func() error
}

func NewUnitOfWork(
    orderRepo OrderRepository,
    orderItemRepo OrderItemRepository,
    inventoryRepo InventoryRepository,
) *TransactionalUnitOfWork {
    return &TransactionalUnitOfWork{
        orderRepo:     orderRepo,
        orderItemRepo: orderItemRepo,
        inventoryRepo: inventoryRepo,
        changes:       make([]func() error, 0),
    }
}

func (uow *TransactionalUnitOfWork) OrderRepo() OrderRepository {
    return uow.orderRepo
}

func (uow *TransactionalUnitOfWork) OrderItemRepo() OrderItemRepository {
    return uow.orderItemRepo
}

func (uow *TransactionalUnitOfWork) InventoryRepo() InventoryRepository {
    return uow.inventoryRepo
}

func (uow *TransactionalUnitOfWork) Commit() error {
    if uow.committed {
        return fmt.Errorf("already committed")
    }
    
    fmt.Println("[Unit of Work] Committing transaction...")
    
    // Execute all changes
    for i, change := range uow.changes {
        if err := change(); err != nil {
            fmt.Printf("[Unit of Work] Change %d failed: %v\n", i+1, err)
            return err
        }
    }
    
    uow.committed = true
    fmt.Println("[Unit of Work] Transaction committed successfully")
    return nil
}

func (uow *TransactionalUnitOfWork) Rollback() error {
    fmt.Println("[Unit of Work] Rolling back transaction...")
    uow.changes = make([]func() error, 0)
    return nil
}

// Service using Unit of Work
type OrderPlacementService struct{}

func (s *OrderPlacementService) PlaceOrder(
    uow UnitOfWork,
    customerID string,
    items []OrderItem,
) error {
    fmt.Printf("\n[Order Placement] Processing order for customer %s\n", customerID)
    
    // Step 1: Check inventory for all items
    for _, item := range items {
        inventory, err := uow.InventoryRepo().FindByProductID(item.ProductID)
        if err != nil {
            return fmt.Errorf("product not found: %s", item.ProductID)
        }
        
        if inventory.Quantity-inventory.Reserved < item.Quantity {
            return fmt.Errorf("insufficient inventory for product %s", item.ProductID)
        }
    }
    
    // Step 2: Create order
    order := &Order{
        ID:         generateOrderID(),
        CustomerID: customerID,
        Total:      calculateOrderTotal(items),
        Status:     "pending",
    }
    
    if err := uow.OrderRepo().Save(order); err != nil {
        return fmt.Errorf("failed to save order: %w", err)
    }
    
    fmt.Printf("[Order Placement] Order created: %s\n", order.ID)
    
    // Step 3: Save order items
    orderItems := make([]*OrderItem, len(items))
    for i := range items {
        items[i].OrderID = order.ID
        orderItems[i] = &items[i]
    }
    
    if err := uow.OrderItemRepo().SaveAll(orderItems); err != nil {
        return fmt.Errorf("failed to save order items: %w", err)
    }
    
    fmt.Printf("[Order Placement] Saved %d order items\n", len(orderItems))
    
    // Step 4: Reserve inventory
    for _, item := range items {
        inventory, _ := uow.InventoryRepo().FindByProductID(item.ProductID)
        inventory.Reserved += item.Quantity
        
        if err := uow.InventoryRepo().Update(inventory); err != nil {
            return fmt.Errorf("failed to reserve inventory: %w", err)
        }
        
        fmt.Printf("[Order Placement] Reserved %d units of product %s\n",
            item.Quantity, item.ProductID)
    }
    
    // Step 5: Commit entire transaction
    if err := uow.Commit(); err != nil {
        uow.Rollback()
        return fmt.Errorf("transaction failed: %w", err)
    }
    
    fmt.Printf("[Order Placement] Order %s placed successfully\n", order.ID)
    return nil
}

func generateOrderID() string {
    return fmt.Sprintf("ORD-%d", time.Now().Unix())
}

func calculateOrderTotal(items []OrderItem) float64 {
    // Simplified total calculation
    return float64(len(items)) * 29.99
}

func DemonstrateUnitOfWork() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         UNIT OF WORK PATTERN                               ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝")
    
    fmt.Println("\nBenefits:")
    fmt.Println("  ✓ Multiple repositories in single transaction")
    fmt.Println("  ✓ All-or-nothing semantics")
    fmt.Println("  ✓ Automatic rollback on failure")
    fmt.Println("  ✓ Consistency across multiple aggregates")
}
```

## Best Practices for Repository Pattern

### Practice 1: Keep Repositories Focused on Persistence

Repositories should only handle persistence concerns, not business logic.

```go
package practices

import "fmt"

// ❌ BAD: Business logic in repository
type BadOrderRepository struct{}

func (r *BadOrderRepository) CreateOrderWithValidation(customerID string, items []string) error {
    // Business validation in repository - WRONG!
    if len(items) == 0 {
        return fmt.Errorf("order must have items")
    }
    
    // Price calculation in repository - WRONG!
    total := len(items) * 29.99
    
    // Save order
    fmt.Printf("Saving order with total: %.2f\n", total)
    return nil
}

// ✅ GOOD: Repository only handles persistence
type GoodOrderRepository struct{}

func (r *GoodOrderRepository) Save(order *Order) error {
    // Only persistence logic
    fmt.Printf("Saving order: %s\n", order.ID)
    return nil
}

// Business logic stays in service
type GoodOrderService struct {
    repo *GoodOrderRepository
}

func (s *GoodOrderService) CreateOrder(customerID string, items []string) error {
    // Business validation in service - CORRECT!
    if len(items) == 0 {
        return fmt.Errorf("order must have items")
    }
    
    // Business logic in service - CORRECT!
    order := &Order{
        ID:         "ORD-001",
        CustomerID: customerID,
        Total:      float64(len(items)) * 29.99,
    }
    
    // Repository only handles saving
    return s.repo.Save(order)
}
```

### Practice 2: Return Domain Objects, Not Data Transfer Objects

Repositories should work with domain models, not DTOs.

```go
package dtos

// ✅ GOOD: Repository returns domain objects
type CustomerRepository interface {
    FindByID(id string) (*Customer, error) // Returns domain model
}

// ❌ BAD: Repository returns DTOs
type BadCustomerRepository interface {
    FindByID(id string) (*CustomerDTO, error) // Returns DTO - wrong layer!
}

type CustomerDTO struct {
    ID   string
    Name string
}
```

### Practice 3: Use Interfaces, Not Concrete Types

Always program against repository interfaces, not implementations.

```go
package interfaces

// ✅ GOOD: Depend on interface
type OrderService struct {
    orderRepo OrderRepository // Interface
}

// ❌ BAD: Depend on concrete type
type BadOrderService struct {
    orderRepo *PostgresOrderRepository // Concrete implementation - tight coupling!
}
```

## Conclusion: Repository Pattern for Maintainable ERP Systems

The Repository pattern is one of the most valuable patterns for building maintainable Enterprise Resource Planning systems. By creating an abstraction layer between your domain logic and data access logic, you achieve true separation of concerns. Your business services focus purely on business rules, while repositories handle the messy details of persistence.

The benefits are substantial: your code becomes testable without databases, you can swap storage implementations transparently, you centralize data access logic for easier optimization, and your domain layer remains clean and expressive. The repository interface serves as a contract between layers, making your architecture explicit and understandable.

In Go, with its emphasis on interfaces and composition, the Repository pattern feels natural and idiomatic. You define clear contracts through interfaces, provide multiple implementations for different contexts (in-memory for testing, SQL for production, cache for performance), and inject dependencies through constructors.

As you build your Go-based ERP system, embrace the Repository pattern for all your data access needs. Define repository interfaces in your domain layer using domain terminology. Implement concrete repositories in your infrastructure layer with database-specific logic. Test your business services with mock repositories. The clean separation and testability you gain will pay dividends as your system grows in complexity.

Remember: repositories are collections of domain objects from the perspective of your business logic. They hide infrastructure concerns behind simple, expressive interfaces. This abstraction is not overhead—it is the foundation of a maintainable, evolvable architecture.
