# Event-Driven Architecture for Building Enterprise ERP Systems in Go

## Introduction: The Problem with Tight Coupling in Distributed Systems

When you build an Enterprise Resource Planning system, you inevitably create multiple interconnected subsystems. Your order processing module needs to notify the inventory system when stock is reserved. Your payment system needs to inform accounting when transactions complete. Your shipping module needs to update customer service when packages are dispatched. The natural, straightforward approach is direct coupling: `OrderService` calls `InventoryService.ReserveStock()` directly. `PaymentProcessor` calls `AccountingService.RecordTransaction()` explicitly. `ShippingModule` invokes `CustomerService.NotifyShipment()` synchronously.

This direct coupling feels simple initially. You write `inventoryService.ReserveStock(items)` right where you need it, and everything works. But as your ERP system grows, this tight coupling becomes increasingly problematic. Your modules become interdependent in complex ways. Order processing cannot function without inventory, payment, shipping, and notification services all being available simultaneously. A failure in any downstream service causes the entire operation to fail. Adding new functionality requires modifying existing services. Deploying changes requires coordinating multiple teams. Your system becomes a fragile, monolithic tangle despite being divided into modules.

Moreover, this synchronous coupling creates performance bottlenecks. When a customer places an order, the request must wait while you reserve inventory, process payment, update accounting, notify warehouse, send confirmation email, and update analytics—all in sequence. Each step adds latency. If any step is slow, the entire operation is slow. Your system cannot scale because everything must scale together.

Event-Driven Architecture (EDA) solves these problems through asynchronous, loosely-coupled communication via events. Instead of services calling each other directly, they publish events to an event bus. Interested services subscribe to relevant events and react independently. When an order is placed, the order service publishes an `OrderPlaced` event. The inventory service subscribes to this event and reserves stock. The payment service processes payment. The email service sends confirmation. Each service operates independently, asynchronously, and in parallel.

Let us explore what Event-Driven Architecture is, why it transforms ERP systems, and how to implement it effectively in Go.

## Understanding Event-Driven Architecture: Core Concepts

### What is Event-Driven Architecture?

Event-Driven Architecture is an architectural pattern where system components communicate by producing and consuming events. An event represents a significant occurrence or state change—something that happened that other parts of the system might care about. Components publish events without knowing who (if anyone) will consume them. Other components subscribe to events they care about without knowing who published them.

This inversion of dependencies—publishers don't know consumers, consumers don't know publishers—enables loose coupling at an architectural level.

```go
package events

import (
    "fmt"
    "time"
)

// ❌ TRADITIONAL SYNCHRONOUS COUPLING

type OrderServiceSync struct {
    inventoryService  *InventoryService
    paymentService    *PaymentService
    emailService      *EmailService
    shippingService   *ShippingService
    analyticsService  *AnalyticsService
}

func (s *OrderServiceSync) PlaceOrder(orderID, customerID string, items []Item) error {
    fmt.Println("[Order Service] Placing order synchronously...")
    
    // Tight coupling - must call each service directly
    // All must be available and respond
    
    // Step 1: Reserve inventory (blocks until complete)
    if err := s.inventoryService.ReserveStock(items); err != nil {
        return fmt.Errorf("inventory reservation failed: %w", err)
    }
    
    // Step 2: Process payment (blocks until complete)
    if err := s.paymentService.ProcessPayment(orderID, 100.00); err != nil {
        s.inventoryService.ReleaseStock(items) // Manual rollback
        return fmt.Errorf("payment failed: %w", err)
    }
    
    // Step 3: Send confirmation (blocks until complete)
    if err := s.emailService.SendConfirmation(customerID, orderID); err != nil {
        // Email failed but payment succeeded - what now?
        fmt.Printf("Warning: email failed: %v\n", err)
    }
    
    // Step 4: Notify shipping (blocks until complete)
    if err := s.shippingService.CreateShipment(orderID); err != nil {
        fmt.Printf("Warning: shipment creation failed: %v\n", err)
    }
    
    // Step 5: Update analytics (blocks until complete)
    if err := s.analyticsService.RecordOrder(orderID); err != nil {
        fmt.Printf("Warning: analytics update failed: %v\n", err)
    }
    
    fmt.Println("[Order Service] Order placed (took 5+ seconds)")
    return nil
}

func (bus *InMemoryEventBus) Publish(event interface{}) error {
    eventType := getEventType(event)
    
    fmt.Printf("[Event Bus] Publishing event: %s\n", eventType)
    
    handlers, exists := bus.handlers[eventType]
    if !exists {
        fmt.Println("[Event Bus] No subscribers for this event")
        return nil
    }
    
    // Notify all subscribers asynchronously
    for i, handler := range handlers {
        go func(idx int, h EventHandler, evt interface{}) {
            if err := h(evt); err != nil {
                fmt.Printf("[Event Bus] Handler %d failed: %v\n", idx, err)
            }
        }(i, handler, event)
    }
    
    fmt.Printf("[Event Bus] Notified %d subscribers\n", len(handlers))
    return nil
}

    fmt.Printf("[Event Bus] New subscriber for %s events\n", eventType)
    return nil
}
```

### Component 2: Event Bus - The Communication Channel

The event bus is the infrastructure that routes events from publishers to subscribers.

```go
package eventbus

import (
    "fmt"
    "sync"
    "time"
)

// Event Bus interface
type EventBus interface {
    Publish(event Event) error
    Subscribe(eventType string, handler EventHandler) Subscription
    Unsubscribe(subscription Subscription) error
}

type Event interface {
    GetEventType() string
    GetEventID() string
    GetTimestamp() time.Time
}

type EventHandler func(event Event) error

type Subscription interface {
    Unsubscribe() error
}

// In-memory event bus implementation
type InMemoryBus struct {
    handlers   map[string][]EventHandler
    mu         sync.RWMutex
    middleware []Middleware
    errorHandler ErrorHandler
}

type Middleware func(Event, EventHandler) EventHandler
type ErrorHandler func(Event, error)

func NewInMemoryBus() *InMemoryBus {
    return &InMemoryBus{
        handlers:   make(map[string][]EventHandler),
        middleware: make([]Middleware, 0),
        errorHandler: func(event Event, err error) {
            fmt.Printf("[Event Bus] Error handling event %s: %v\n", 
                event.GetEventType(), err)
        },
    }
}

func (bus *InMemoryBus) Publish(event Event) error {
    eventType := event.GetEventType()
    
    fmt.Printf("[Event Bus] Publishing: %s (ID: %s)\n", 
        eventType, event.GetEventID())
    
    bus.mu.RLock()
    handlers := bus.handlers[eventType]
    bus.mu.RUnlock()
    
    if len(handlers) == 0 {
        fmt.Printf("[Event Bus] No subscribers for %s\n", eventType)
        return nil
    }
    
    // Publish to all handlers asynchronously
    var wg sync.WaitGroup
    for _, handler := range handlers {
        wg.Add(1)
        
        // Apply middleware chain
        wrappedHandler := handler
        for _, mw := range bus.middleware {
            wrappedHandler = mw(event, wrappedHandler)
        }
        
        // Execute handler in goroutine
        go func(h EventHandler) {
            defer wg.Done()
            
            if err := h(event); err != nil {
                bus.errorHandler(event, err)
            }
        }(wrappedHandler)
    }
    
    fmt.Printf("[Event Bus] Dispatched to %d subscribers\n", len(handlers))
    return nil
}

func (bus *InMemoryBus) Subscribe(eventType string, handler EventHandler) Subscription {
    bus.mu.Lock()
    defer bus.mu.Unlock()
    
    bus.handlers[eventType] = append(bus.handlers[eventType], handler)
    
    fmt.Printf("[Event Bus] New subscription for %s (total: %d)\n", 
        eventType, len(bus.handlers[eventType]))
    
    return &inMemorySubscription{
        bus:       bus,
        eventType: eventType,
        handler:   handler,
    }
}

func (bus *InMemoryBus) Unsubscribe(subscription Subscription) error {
    return subscription.Unsubscribe()
}

// Add middleware
func (bus *InMemoryBus) Use(middleware Middleware) {
    bus.middleware = append(bus.middleware, middleware)
}

// Set error handler
func (bus *InMemoryBus) SetErrorHandler(handler ErrorHandler) {
    bus.errorHandler = handler
}

type inMemorySubscription struct {
    bus       *InMemoryBus
    eventType string
    handler   EventHandler
}

func (sub *inMemorySubscription) Unsubscribe() error {
    sub.bus.mu.Lock()
    defer sub.bus.mu.Unlock()
    
    handlers := sub.bus.handlers[sub.eventType]
    for i, h := range handlers {
        // Compare function pointers
        if &h == &sub.handler {
            // Remove handler
            sub.bus.handlers[sub.eventType] = append(handlers[:i], handlers[i+1:]...)
            fmt.Printf("[Event Bus] Unsubscribed from %s\n", sub.eventType)
            return nil
        }
    }
    
    return fmt.Errorf("subscription not found")
}

// Middleware examples

// Logging middleware
func LoggingMiddleware(event Event, next EventHandler) EventHandler {
    return func(e Event) error {
        start := time.Now()
        fmt.Printf("[Middleware] Before handling %s\n", e.GetEventType())
        
        err := next(e)
        
        duration := time.Since(start)
        fmt.Printf("[Middleware] After handling %s (took %v)\n", 
            e.GetEventType(), duration)
        
        return err
    }
}

// Retry middleware
func RetryMiddleware(maxRetries int) Middleware {
    return func(event Event, next EventHandler) EventHandler {
        return func(e Event) error {
            var err error
            
            for attempt := 1; attempt <= maxRetries; attempt++ {
                err = next(e)
                if err == nil {
                    return nil
                }
                
                fmt.Printf("[Retry Middleware] Attempt %d failed: %v\n", attempt, err)
                
                if attempt < maxRetries {
                    time.Sleep(time.Duration(attempt) * 100 * time.Millisecond)
                }
            }
            
            return fmt.Errorf("failed after %d attempts: %w", maxRetries, err)
        }
    }
}

// Timeout middleware
func TimeoutMiddleware(timeout time.Duration) Middleware {
    return func(event Event, next EventHandler) EventHandler {
        return func(e Event) error {
            done := make(chan error, 1)
            
            go func() {
                done <- next(e)
            }()
            
            select {
            case err := <-done:
                return err
            case <-time.After(timeout):
                return fmt.Errorf("handler timeout after %v", timeout)
            }
        }
    }
}

func DemonstrateEventBus() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         EVENT BUS WITH MIDDLEWARE                          ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    bus := NewInMemoryBus()
    
    // Add middleware
    bus.Use(LoggingMiddleware)
    bus.Use(RetryMiddleware(3))
    bus.Use(TimeoutMiddleware(5 * time.Second))
    
    fmt.Println("✓ Event bus created with middleware chain")
    fmt.Println("✓ Logging, retry, and timeout enabled")
}
```

### Component 3: Event Handlers - The Consumers

Event handlers are functions that react to events. They should be idempotent and handle failures gracefully.

```go
    GetTimestamp() time.Time

        sentEmails: make(map[string]bool),

func (h *EmailNotificationHandler) Handle(event Event) error {

        orderEvent.OrderID)
```

### Pattern 1: Event Sourcing Integration

Event-Driven Architecture pairs naturally with Event Sourcing, where events are both the source of truth and the means of communication.

```go

import (

type OrderAggregate struct {

    EventID     string

    EventType   string

            PaymentID: paymentID,
            Amount:    amount,
        },
        Version:   agg.Version + 1,
        Timestamp: time.Now(),
    }
    
    agg.apply(event)
    agg.UncommittedEvents = append(agg.UncommittedEvents, event)
    return nil
}

func (agg *OrderAggregate) MarkAsShipped(shipmentID, carrier string) error {
    if agg.Status != "paid" {
        return fmt.Errorf("order must be paid before shipping")
    }
    
    event := DomainEvent{
        EventID:     generateEventID(),
        AggregateID: agg.ID,
        EventType:   "OrderShipped",
        Data: OrderShippedEventData{
            ShipmentID: shipmentID,
            Carrier:    carrier,
        },
        Version:   agg.Version + 1,
        Timestamp: time.Now(),
    }
    
    agg.apply(event)
    agg.UncommittedEvents = append(agg.UncommittedEvents, event)
    return nil

// Apply event to aggregate state
func (agg *OrderAggregate) apply(event DomainEvent) {
    switch event.EventType {
    case "OrderCreated":
        data := event.Data.(OrderCreatedEventData)
        agg.CustomerID = data.CustomerID
        agg.Items = data.Items
        agg.Total = data.Total
        agg.Status = "pending"
        
    case "OrderPaid":
        agg.Status = "paid"
        
    case "OrderShipped":
        agg.Status = "shipped"
    }
    
    agg.Version = event.Version
}

// Repository saves events and publishes to event bus
type OrderRepository struct {
    eventStore EventStore
    eventBus   EventBus
}

func (repo *OrderRepository) Save(aggregate *OrderAggregate) error {
    // Save events to event store
    if err := repo.eventStore.Append(aggregate.UncommittedEvents); err != nil {
        return err
    }
    
    // Publish events to event bus for other services
    for _, event := range aggregate.UncommittedEvents {
        if err := repo.eventBus.Publish(event); err != nil {
            fmt.Printf("Failed to publish event: %v\n", err)
        }
    }
    
    // Clear uncommitted events
    aggregate.UncommittedEvents = make([]DomainEvent, 0)
    
    return nil
}

type EventStore interface {
    Append(events []DomainEvent) error
}

type EventBus interface {
    Publish(event interface{}) error
}

}
```

### Pattern 2: Saga Pattern for Distributed Transactions

Sagas coordinate long-running business processes across multiple services using compensating events.

```go
package saga

import (
    "fmt"
    "time"
)

// Saga orchestrates distributed transactions
type OrderSaga struct {
    orderID       string
    state         string
    steps         []SagaStep
    currentStep   int
    compensations []func() error
}

type SagaStep struct {
    Name        string
    Execute     func() error
    Compensate  func() error
}

func NewOrderSaga(orderID string) *OrderSaga {
    return &OrderSaga{
        orderID:       orderID,
        state:         "initiated",
        steps:         make([]SagaStep, 0),
        compensations: make([]func() error, 0),
    }
}

func (saga *OrderSaga) AddStep(step SagaStep) {
    saga.steps = append(saga.steps, step)
}

func (saga *OrderSaga) Execute() error {
    fmt.Printf("[Saga] Starting order saga for %s\n", saga.orderID)
    
    for i, step := range saga.steps {
        saga.currentStep = i
        
        fmt.Printf("[Saga] Executing step %d: %s\n", i+1, step.Name)
        
        if err := step.Execute(); err != nil {
            fmt.Printf("[Saga] Step %d failed: %v\n", i+1, err)
            fmt.Println("[Saga] Starting compensation...")
            
            // Execute compensating transactions in reverse order
            return saga.compensate()
        }
        
        // Store compensation for this step
        saga.compensations = append(saga.compensations, step.Compensate)
        
        fmt.Printf("[Saga] Step %d completed successfully\n", i+1)
    }
    
    saga.state = "completed"
    fmt.Println("[Saga] Saga completed successfully")
    return nil
}

func (saga *OrderSaga) compensate() error {
    fmt.Println("[Saga] Executing compensating transactions...")
    
    // Execute compensations in reverse order
    for i := len(saga.compensations) - 1; i >= 0; i-- {
        compensation := saga.compensations[i]
        
        fmt.Printf("[Saga] Compensating step %d\n", i+1)
        
        if err := compensation(); err != nil {
            fmt.Printf("[Saga] Compensation %d failed: %v\n", i+1, err)
            // Log but continue with other compensations
        }
    }
    
    saga.state = "compensated"
    fmt.Println("[Saga] Compensation complete")
    return fmt.Errorf("saga failed and was compensated")
}

// Example: Order processing saga
func CreateOrderSagaExample(orderID string) *OrderSaga {
    saga := NewOrderSaga(orderID)
    
    // Step 1: Reserve inventory
    saga.AddStep(SagaStep{
        Name: "Reserve Inventory",
        Execute: func() error {
            fmt.Println("  → Reserving inventory...")
            time.Sleep(50 * time.Millisecond)
            return nil // Success
        },
        Compensate: func() error {
            fmt.Println("  ← Releasing inventory...")
            time.Sleep(30 * time.Millisecond)
            return nil
        },
    })
    
    // Step 2: Process payment
    saga.AddStep(SagaStep{
        Name: "Process Payment",
        Execute: func() error {
            fmt.Println("  → Processing payment...")
            time.Sleep(100 * time.Millisecond)
            // Simulate failure
            // return fmt.Errorf("payment declined")
            return nil // Success
        },
        Compensate: func() error {
            fmt.Println("  ← Refunding payment...")
            time.Sleep(50 * time.Millisecond)
            return nil
        },
    })
    
    // Step 3: Create shipment
    saga.AddStep(SagaStep{
        Name: "Create Shipment",
        Execute: func() error {
            fmt.Println("  → Creating shipment...")
            time.Sleep(50 * time.Millisecond)
            return nil // Success
        },
        Compensate: func() error {
            fmt.Println("  ← Cancelling shipment...")
            time.Sleep(30 * time.Millisecond)
            return nil
        },
    })
    
    // Step 4: Send confirmation
    saga.AddStep(SagaStep{
        Name: "Send Confirmation",
        Execute: func() error {
            fmt.Println("  → Sending confirmation email...")
            time.Sleep(30 * time.Millisecond)
            return nil // Success
        },
        Compensate: func() error {
            fmt.Println("  ← Sending cancellation email...")
            time.Sleep(30 * time.Millisecond)
            return nil
        },
    })
    
    return saga
}

func DemonstrateSagaPattern() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         SAGA PATTERN FOR DISTRIBUTED TRANSACTIONS          ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    fmt.Println("=== Successful Saga ===")
    successSaga := CreateOrderSagaExample("ORD-001")
    successSaga.Execute()
    
    fmt.Println("\n\n✓ Saga coordinates multi-step transactions")
    fmt.Println("✓ Automatic compensation on failure")
    fmt.Println("✓ Maintains consistency across services")
    fmt.Println("✓ No distributed transactions needed")
}
```

### Pattern 3: Event Replay and Time Travel

Event-driven systems can replay events for debugging, auditing, or rebuilding state.

```go
package replay

import (
    "fmt"
    "time"
)

// Event store with replay capability
type ReplayableEventStore struct {
    events []StoredEvent
}

type StoredEvent struct {
    EventID     string
    EventType   string
    AggregateID string
    Data        interface{}
    Timestamp   time.Time
    Version     int
}

func NewReplayableEventStore() *ReplayableEventStore {
    return &ReplayableEventStore{
        events: make([]StoredEvent, 0),
    }
}

func (store *ReplayableEventStore) Append(event StoredEvent) {
    store.events = append(store.events, event)
}

// Replay all events
func (store *ReplayableEventStore) ReplayAll(handler func(StoredEvent) error) error {
    fmt.Printf("[Replay] Replaying %d events\n", len(store.events))
    
    for i, event := range store.events {
        fmt.Printf("[Replay] Event %d/%d: %s\n", i+1, len(store.events), event.EventType)
        
        if err := handler(event); err != nil {
            return fmt.Errorf("replay failed at event %d: %w", i+1, err)
        }
    }
    
    fmt.Println("[Replay] Replay complete")
    return nil
}

// Replay events up to a specific point in time
func (store *ReplayableEventStore) ReplayUntil(timestamp time.Time, handler func(StoredEvent) error) error {
    count := 0
    
    for _, event := range store.events {
        if event.Timestamp.After(timestamp) {
            break
        }
        
        if err := handler(event); err != nil {
            return fmt.Errorf("replay failed: %w", err)
        }
        
        count++
    }
    
    fmt.Printf("[Replay] Replayed %d events until %s\n", 
        count, timestamp.Format("2006-01-02 15:04:05"))
    
    return nil
}

// Replay events for specific aggregate
func (store *ReplayableEventStore) ReplayAggregate(aggregateID string, handler func(StoredEvent) error) error {
    count := 0
    
    for _, event := range store.events {
        if event.AggregateID == aggregateID {
            if err := handler(event); err != nil {
                return fmt.Errorf("replay failed: %w", err)
            }
            count++
        }
    }
    
    fmt.Printf("[Replay] Replayed %d events for aggregate %s\n", count, aggregateID)
    return nil
}

// Rebuild aggregate state from events
func (store *ReplayableEventStore) RebuildAggregate(aggregateID string) (*AggregateState, error) {
    state := &AggregateState{
        ID:      aggregateID,
        Balance: 0,
        Status:  "unknown",
    }
    
    handler := func(event StoredEvent) error {
        switch event.EventType {
        case "AccountOpened":
            state.Status = "open"
            state.Balance = event.Data.(float64)
            
        case "MoneyDeposited":
            state.Balance += event.Data.(float64)
            
        case "MoneyWithdrawn":
            state.Balance -= event.Data.(float64)
            
        case "AccountClosed":
            state.Status = "closed"
        }
        
        return nil
    }
    
    if err := store.ReplayAggregate(aggregateID, handler); err != nil {
        return nil, err
    }
    
    return state, nil
}

type AggregateState struct {
    ID      string
    Balance float64
    Status  string
}

func DemonstrateEventReplay() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         EVENT REPLAY AND TIME TRAVEL                       ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    store := NewReplayableEventStore()
    
    // Simulate events
    baseTime := time.Now().Add(-24 * time.Hour)
    
    store.Append(StoredEvent{
        EventID:     "evt-1",
        EventType:   "AccountOpened",
        AggregateID: "ACC-001",
        Data:        1000.0,
        Timestamp:   baseTime,
    })
    
    store.Append(StoredEvent{
        EventID:     "evt-2",
        EventType:   "MoneyDeposited",
        AggregateID: "ACC-001",
        Data:        500.0,
        Timestamp:   baseTime.Add(6 * time.Hour),
    })
    
    store.Append(StoredEvent{
        EventID:     "evt-3",
        EventType:   "MoneyWithdrawn",
        AggregateID: "ACC-001",
        Data:        200.0,
        Timestamp:   baseTime.Add(12 * time.Hour),
    })
    
    // Rebuild current state
    fmt.Println("=== Rebuilding Current State ===")
    state, _ := store.RebuildAggregate("ACC-001")
    fmt.Printf("Account Status: %s\n", state.Status)
    fmt.Printf("Current Balance: $%.2f\n", state.Balance)
    
    // Time travel - state at specific point
    fmt.Println("\n=== Time Travel to 8 Hours Ago ===")
    pointInTime := time.Now().Add(-16 * time.Hour)
    
    pastState := &AggregateState{ID: "ACC-001"}
    store.ReplayUntil(pointInTime, func(event StoredEvent) error {
        if event.AggregateID == "ACC-001" {
            switch event.EventType {
            case "AccountOpened":
                pastState.Balance = event.Data.(float64)
            case "MoneyDeposited":
                pastState.Balance += event.Data.(float64)
            }
        }
        return nil
    })
    
    fmt.Printf("Balance at %s: $%.2f\n", 
        pointInTime.Format("2006-01-02 15:04:05"), pastState.Balance)
    
    fmt.Println("\n✓ Events can be replayed for debugging")
    fmt.Println("✓ State can be rebuilt at any point in time")
    fmt.Println("✓ Complete audit trail available")
}
```

### Practice 2: Handle Duplicate Events Idempotently

Events may be delivered more than once. Handlers must be idempotent.

```go
package idempotency

import "fmt"

// Idempotent handler using event ID tracking
type IdempotentHandler struct {
    processedEvents map[string]bool
}

func (h *IdempotentHandler) Handle(eventID string, process func()) {
    // Check if already processed
    if h.processedEvents[eventID] {
        fmt.Printf("Event %s already processed (idempotent)\n", eventID)
        return
    }
    
    // Process event
    process()
    
    // Mark as processed
    h.processedEvents[eventID] = true
}
```

### Practice 3: Use Dead Letter Queues

Failed events should go to dead letter queue for manual intervention.

```go
package dlq

import "fmt"

type DeadLetterQueue struct {
    failedEvents []FailedEvent
}

type FailedEvent struct {
    Event      interface{}
    Error      error
    Attempts   int
    Timestamp  time.Time
}

func (dlq *DeadLetterQueue) Add(event interface{}, err error, attempts int) {
    dlq.failedEvents = append(dlq.failedEvents, FailedEvent{
        Event:     event,
        Error:     err,
        Attempts:  attempts,
        Timestamp: time.Now(),
    })
    
    fmt.Printf("[DLQ] Added failed event (attempts: %d): %v\n", attempts, err)
}
```

## Conclusion: Event-Driven Architecture for Scalable ERP Systems

Event-Driven Architecture transforms how we build Enterprise Resource Planning systems by enabling loose coupling, scalability, and resilience. Rather than services calling each other directly in tight synchronous chains, they communicate asynchronously through events. This architectural shift eliminates single points of failure, enables independent scaling, and makes systems naturally resilient to partial failures.

The benefits are substantial: services can evolve independently without breaking others, new functionality can be added by subscribing to existing events, systems can process operations in parallel for better performance, and the event stream provides a complete audit trail of everything that happened. Each service focuses on its core responsibility, reacting to events it cares about while ignoring others.

In Go, with its excellent concurrency primitives, channels, and goroutines, implementing Event-Driven Architecture feels natural. Events flow through channels, handlers process events concurrently, and the language's simplicity keeps the architecture comprehensible even as complexity grows.

As you build your Go-based ERP system, embrace Event-Driven Architecture for inter-service communication. Define clear, self-contained events that capture business facts. Implement an event bus that reliably delivers events to subscribers. Write idempotent handlers that process events resilably. The loose coupling and scalability you gain will be essential as your ERP system grows to serve your business.

Remember: events are facts about the past that services can react to independently. This simple principle, applied consistently, creates systems that are more maintainable, scalable, and resilient than traditional synchronous architectures.
