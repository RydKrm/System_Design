# Event Sourcing Pattern for Building Enterprise ERP Systems in Go

## Introduction: The Problem with State-Based Persistence

When you build a traditional Enterprise Resource Planning system, you store the current state of your entities in a database. Your `orders` table contains the current status of each order. Your `inventory` table shows current stock levels. Your `accounts` table displays current balances. This seems natural and intuitive—after all, you care about the present state of your business, not ancient history.

However, this state-based approach discards valuable information. When an order status changes from "pending" to "shipped," you update a single field in the database. You lose the information about when the change happened, who made it, why it was made, and what the previous state was. When inventory decreases, you decrement a number. You cannot reconstruct how that inventory was used over time. When an account balance changes, you overwrite the old value. You cannot audit the sequence of transactions that led to the current balance.

This loss of history creates serious problems in enterprise systems. Auditing becomes impossible without complex logging infrastructure. Debugging production issues requires guesswork because you cannot see what actually happened. Regulatory compliance demands detailed audit trails that state-based systems struggle to provide. Business intelligence tools must work with snapshots rather than the full temporal richness of your data.

Event Sourcing offers a fundamentally different approach: instead of storing current state, store the sequence of events that led to that state. Instead of updating an order's status, append an `OrderShipped` event. Instead of decrementing inventory, append an `ItemsSold` event. Instead of updating a balance, append a `MoneyDeposited` event. The current state is derived by replaying these events from the beginning.

This shift from state to events transforms your architecture in profound ways. Let us explore how Event Sourcing works, why it matters for ERP systems, and how to implement it effectively in Go.

## Understanding Event Sourcing: The Core Concepts

### What is an Event?

In Event Sourcing, an event is an immutable fact representing something that happened in the past. Events are expressed in past tense because they describe completed actions. They are never deleted or modified—only appended to the event stream.

```go
package events

import (
    "fmt"
    "time"
)

// Events are immutable facts about what happened

// ✅ GOOD: Events in past tense, describing facts
type OrderPlacedEvent struct {
    OrderID       string
    CustomerID    string
    Items         []OrderItem
    TotalAmount   float64
    PlacedAt      time.Time
}

type OrderShippedEvent struct {
    OrderID       string
    TrackingNumber string
    Carrier       string
    ShippedAt     time.Time
}

type PaymentReceivedEvent struct {
    OrderID       string
    Amount        float64
    PaymentMethod string
    TransactionID string
    ReceivedAt    time.Time
}

type OrderCancelledEvent struct {

    CancelledAt   time.Time
}

type OrderItem struct {
    ProductID string

// Events contain all necessary data
// Events are self-contained - no need to look elsewhere to understand them
func (e OrderPlacedEvent) String() string {
    return fmt.Sprintf("Order %s placed by customer %s for $%.2f at %s",
        e.OrderID, e.CustomerID, e.TotalAmount, e.PlacedAt.Format(time.RFC3339))
```

### Traditional State vs Event Sourcing

Let us contrast how traditional state-based persistence differs from Event Sourcing.

```go
package comparison

import (
    "fmt"
    "time"
)

// ❌ TRADITIONAL STATE-BASED PERSISTENCE

type OrderState struct {
    ID            string
    CustomerID    string
    Items         []Item
    Total         float64
    Status        string // Current status only
    ShippedAt     *time.Time
    DeliveredAt   *time.Time
    UpdatedAt     time.Time
}

type Item struct {
    ProductID string
    Quantity  int
    Price     float64
}

// State-based operations overwrite previous state
func UpdateOrderState(order *OrderState, newStatus string) {
    // Lost information: when did it change? who changed it? why?
    order.Status = newStatus
    order.UpdatedAt = time.Now()
    
    fmt.Printf("[State] Updated order %s to status: %s\n", order.ID, newStatus)
    fmt.Println("  ✗ Previous status lost")
    fmt.Println("  ✗ History of transitions lost")
    fmt.Println("  ✗ Cannot audit state changes")
}

// ✅ EVENT SOURCING APPROACH

type Event struct {
    ID            string
    AggregateID   string
    AggregateType string
    EventType     string
    Data          interface{}
    Metadata      EventMetadata
    Version       int
    Timestamp     time.Time
}

type EventMetadata struct {
    UserID        string
    CorrelationID string
    CausationID   string
    IPAddress     string
}

// Events are appended, never modified
type EventStore struct {
    events []Event
}

func (es *EventStore) AppendEvent(event Event) {
    event.Timestamp = time.Now()
    event.Version = len(es.events) + 1
    es.events = append(es.events, event)
    
    fmt.Printf("[Event Store] Appended event: %s (version %d)\n", event.EventType, event.Version)
    fmt.Println("  ✓ Complete history preserved")
    fmt.Println("  ✓ Can replay to any point in time")
    fmt.Println("  ✓ Full audit trail maintained")
}

func (es *EventStore) GetEvents(aggregateID string) []Event {
    result := make([]Event, 0)
    for _, event := range es.events {
        if event.AggregateID == aggregateID {
            result = append(result, event)
        }
    }
    return result
}

// Rebuild current state by replaying events
func (es *EventStore) ReconstructState(aggregateID string) *OrderState {
    events := es.GetEvents(aggregateID)
    
    // Start with empty state
    order := &OrderState{
        ID: aggregateID,
    }
    
    // Replay each event to rebuild state
    for _, event := range events {
        applyEventToState(order, event)
    }
    
    return order
}

func applyEventToState(order *OrderState, event Event) {
    switch event.EventType {
    case "OrderPlaced":
        data := event.Data.(OrderPlacedData)
        order.CustomerID = data.CustomerID
        order.Items = data.Items
        order.Total = data.Total
        order.Status = "Placed"
        
    case "PaymentReceived":
        order.Status = "Paid"
        
    case "OrderShipped":
        order.Status = "Shipped"
        shipped := event.Timestamp
        order.ShippedAt = &shipped
        
    case "OrderDelivered":
        order.Status = "Delivered"
        delivered := event.Timestamp
        order.DeliveredAt = &delivered
    }
}

type OrderPlacedData struct {
    CustomerID string
    Items      []Item
    Total      float64
}

func DemonstrateStateVsEvents() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         STATE-BASED vs EVENT SOURCING                      ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    fmt.Println("=== Traditional State-Based ===")
    order := &OrderState{ID: "ORD-001", Status: "Placed"}
    UpdateOrderState(order, "Shipped")
    UpdateOrderState(order, "Delivered")
    fmt.Printf("Final state: %s (history lost)\n", order.Status)
    
    fmt.Println("\n=== Event Sourcing ===")
    eventStore := &EventStore{events: make([]Event, 0)}
    
    eventStore.AppendEvent(Event{
        AggregateID:   "ORD-001",
        AggregateType: "Order",
        EventType:     "OrderPlaced",
        Data: OrderPlacedData{
            CustomerID: "CUST-123",
            Items:      []Item{{ProductID: "PROD-A", Quantity: 1, Price: 29.99}},
            Total:      29.99,
        },
    })
    
    eventStore.AppendEvent(Event{
        AggregateID:   "ORD-001",
        AggregateType: "Order",
        EventType:     "PaymentReceived",
    })
    
    eventStore.AppendEvent(Event{
        AggregateID:   "ORD-001",
        AggregateType: "Order",
        EventType:     "OrderShipped",
    })
    
    // Reconstruct current state from events
    reconstructed := eventStore.ReconstructState("ORD-001")
    fmt.Printf("\nReconstructed state: %s\n", reconstructed.Status)
    fmt.Printf("Event count: %d (complete history available)\n", len(eventStore.GetEvents("ORD-001")))
}
```

## Core Components of Event Sourcing

### Component 1: Aggregates and Domain Events

In Event Sourcing, aggregates are the entities that emit events. An aggregate processes commands and generates events based on its business logic.

```go
package aggregate

import (
    "fmt"
    "time"
)

// Aggregate represents a domain entity that produces events
type BankAccountAggregate struct {
    // Identity
    AccountID     string
    AccountNumber string
    
    // Current state (rebuilt from events)
    Balance       float64
    Status        string
    OwnerID       string
    
    // Event tracking
    Version       int
    UncommittedEvents []DomainEvent
}

// DomainEvent represents something that happened
type DomainEvent struct {
    EventID       string
    AggregateID   string
    EventType     string
    Data          interface{}
    Version       int
    Timestamp     time.Time
}

// Events for bank account
type AccountOpenedEvent struct {
    AccountID     string
    AccountNumber string
    OwnerID       string
    InitialDeposit float64
}

type MoneyDepositedEvent struct {
    Amount        float64
    DepositedBy   string
    Description   string
}

type MoneyWithdrawnEvent struct {
    Amount        float64
    WithdrawnBy   string
    Description   string
}

type AccountClosedEvent struct {
    Reason        string
    ClosedBy      string
    FinalBalance  float64
}

// Constructor creates aggregate from events
func NewBankAccountFromEvents(events []DomainEvent) *BankAccountAggregate {
    account := &BankAccountAggregate{
        UncommittedEvents: make([]DomainEvent, 0),
    }
    
    // Replay all events to rebuild state
    for _, event := range events {
        account.applyEvent(event, false)
    }
    
    return account
}

// Commands are intentions to change state
func (a *BankAccountAggregate) OpenAccount(accountID, accountNumber, ownerID string, initialDeposit float64) error {
    // Business rule validation
    if initialDeposit < 0 {
        return fmt.Errorf("initial deposit cannot be negative")
    }
    
    // Create event
    event := DomainEvent{
        EventID:     generateEventID(),
        AggregateID: accountID,
        EventType:   "AccountOpened",
        Data: AccountOpenedEvent{
            AccountID:      accountID,
            AccountNumber:  accountNumber,
            OwnerID:        ownerID,
            InitialDeposit: initialDeposit,
        },
        Version:   a.Version + 1,
        Timestamp: time.Now(),
    }
    
    // Apply event to aggregate
    a.applyEvent(event, true)
    
    return nil
}

func (a *BankAccountAggregate) Deposit(amount float64, depositedBy, description string) error {
    // Business rule validation
    if a.Status == "Closed" {
        return fmt.Errorf("cannot deposit to closed account")
    }
    
    if amount <= 0 {
        return fmt.Errorf("deposit amount must be positive")
    }
    
    // Create event
    event := DomainEvent{
        EventID:     generateEventID(),
        AggregateID: a.AccountID,
        EventType:   "MoneyDeposited",
        Data: MoneyDepositedEvent{
            Amount:      amount,
            DepositedBy: depositedBy,
            Description: description,
        },
        Version:   a.Version + 1,
        Timestamp: time.Now(),
    }
    
    a.applyEvent(event, true)
    
    fmt.Printf("[Account %s] Deposited $%.2f - New balance: $%.2f\n", 
        a.AccountNumber, amount, a.Balance)
    
    return nil
}

func (a *BankAccountAggregate) Withdraw(amount float64, withdrawnBy, description string) error {
    // Business rule validation
    if a.Status == "Closed" {
        return fmt.Errorf("cannot withdraw from closed account")
    }
    
    if amount <= 0 {
        return fmt.Errorf("withdrawal amount must be positive")
    }
    
    if a.Balance < amount {
        return fmt.Errorf("insufficient funds: balance=$%.2f, requested=$%.2f", 
            a.Balance, amount)
    }
    
    // Create event
    event := DomainEvent{
        EventID:     generateEventID(),
        AggregateID: a.AccountID,
        EventType:   "MoneyWithdrawn",
        Data: MoneyWithdrawnEvent{
            Amount:      amount,
            WithdrawnBy: withdrawnBy,
            Description: description,
        },
        Version:   a.Version + 1,
        Timestamp: time.Now(),
    }
    
    a.applyEvent(event, true)
    
    fmt.Printf("[Account %s] Withdrew $%.2f - New balance: $%.2f\n", 
        a.AccountNumber, amount, a.Balance)
    
    return nil
}

func (a *BankAccountAggregate) CloseAccount(reason, closedBy string) error {
    if a.Status == "Closed" {
        return fmt.Errorf("account already closed")
    }
    
    event := DomainEvent{
        EventID:     generateEventID(),
        AggregateID: a.AccountID,
        EventType:   "AccountClosed",
        Data: AccountClosedEvent{
            Reason:       reason,
            ClosedBy:     closedBy,
            FinalBalance: a.Balance,
        },
        Version:   a.Version + 1,
        Timestamp: time.Now(),
    }
    
    a.applyEvent(event, true)
    
    fmt.Printf("[Account %s] Account closed. Final balance: $%.2f\n", 
        a.AccountNumber, a.Balance)
    
    return nil
}

// Apply event to aggregate state
func (a *BankAccountAggregate) applyEvent(event DomainEvent, isNew bool) {
    switch event.EventType {
    case "AccountOpened":
        data := event.Data.(AccountOpenedEvent)
        a.AccountID = data.AccountID
        a.AccountNumber = data.AccountNumber
        a.OwnerID = data.OwnerID
        a.Balance = data.InitialDeposit
        a.Status = "Open"
        
    case "MoneyDeposited":
        data := event.Data.(MoneyDepositedEvent)
        a.Balance += data.Amount
        
    case "MoneyWithdrawn":
        data := event.Data.(MoneyWithdrawnEvent)
        a.Balance -= data.Amount
        
    case "AccountClosed":
        a.Status = "Closed"
    }
    
    a.Version = event.Version
    
    // Track uncommitted events (new events not yet persisted)
    if isNew {
        a.UncommittedEvents = append(a.UncommittedEvents, event)
    }
}

// Get events that haven't been persisted yet
func (a *BankAccountAggregate) GetUncommittedEvents() []DomainEvent {
    return a.UncommittedEvents
}

// Clear uncommitted events after persistence
func (a *BankAccountAggregate) MarkEventsAsCommitted() {
    a.UncommittedEvents = make([]DomainEvent, 0)
}

func generateEventID() string {
    return fmt.Sprintf("evt-%d", time.Now().UnixNano())
}

func DemonstrateAggregate() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         AGGREGATE AND DOMAIN EVENTS                        ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    // Create new account
    account := &BankAccountAggregate{
        UncommittedEvents: make([]DomainEvent, 0),
    }
    
    // Execute commands - each generates events
    account.OpenAccount("ACC-001", "1234567890", "CUST-123", 1000.00)
    account.Deposit(500.00, "CUST-123", "Paycheck")
    account.Withdraw(200.00, "CUST-123", "Groceries")
    account.Deposit(100.00, "CUST-123", "Refund")
    
    fmt.Printf("\nCurrent State:\n")
    fmt.Printf("  Account: %s\n", account.AccountNumber)
    fmt.Printf("  Balance: $%.2f\n", account.Balance)
    fmt.Printf("  Status: %s\n", account.Status)
    fmt.Printf("  Version: %d\n", account.Version)
    fmt.Printf("  Uncommitted Events: %d\n", len(account.GetUncommittedEvents()))
    
    fmt.Println("\n✓ Commands generate events")
    fmt.Println("✓ Events update aggregate state")
    fmt.Println("✓ Business rules enforced before event creation")
    fmt.Println("✓ State is derived from event history")
}
```

### Component 2: Event Store

The Event Store is the database optimized for appending and reading events. It is the source of truth in Event Sourcing.

```go
package eventstore

import (
    "fmt"
    "sync"
    "time"
)

// Event Store is the central repository for all events
type EventStore struct {
    events     []StoredEvent
    mu         sync.RWMutex
    snapshots  map[string]Snapshot
}

type StoredEvent struct {
    EventID       string
    AggregateID   string
    AggregateType string
    EventType     string
    EventData     interface{}
    Metadata      map[string]string
    Version       int
    Timestamp     time.Time
}

type Snapshot struct {
    AggregateID   string
    AggregateType string
    State         interface{}
    Version       int
    Timestamp     time.Time
}

func NewEventStore() *EventStore {
    return &EventStore{
        events:    make([]StoredEvent, 0),
        snapshots: make(map[string]Snapshot),
    }
}

// Append events to the store
func (es *EventStore) AppendEvents(aggregateID string, events []StoredEvent, expectedVersion int) error {
    es.mu.Lock()
    defer es.mu.Unlock()
    
    // Optimistic concurrency check
    currentVersion := es.getCurrentVersion(aggregateID)
    if currentVersion != expectedVersion {
        return fmt.Errorf("concurrency conflict: expected version %d, but current is %d", 
            expectedVersion, currentVersion)
    }
    
    // Append events
    for _, event := range events {
        es.events = append(es.events, event)
        fmt.Printf("[Event Store] Appended: %s (v%d) for aggregate %s\n", 
            event.EventType, event.Version, event.AggregateID)
    }
    
    return nil
}

// Get all events for an aggregate
func (es *EventStore) GetEvents(aggregateID string, fromVersion int) ([]StoredEvent, error) {
    es.mu.RLock()
    defer es.mu.RUnlock()
    
    result := make([]StoredEvent, 0)
    
    for _, event := range es.events {
        if event.AggregateID == aggregateID && event.Version > fromVersion {
            result = append(result, event)
        }
    }
    
    fmt.Printf("[Event Store] Retrieved %d events for aggregate %s (from version %d)\n", 
        len(result), aggregateID, fromVersion)
    
    return result, nil
}

// Get events within a time range
func (es *EventStore) GetEventsByTimeRange(aggregateID string, start, end time.Time) ([]StoredEvent, error) {
    es.mu.RLock()
    defer es.mu.RUnlock()
    
    result := make([]StoredEvent, 0)
    
    for _, event := range es.events {
        if event.AggregateID == aggregateID && 
           event.Timestamp.After(start) && 
           event.Timestamp.Before(end) {
            result = append(result, event)
        }
    }
    
    return result, nil
}

// Get all events of a specific type
func (es *EventStore) GetEventsByType(eventType string) ([]StoredEvent, error) {
    es.mu.RLock()
    defer es.mu.RUnlock()
    
    result := make([]StoredEvent, 0)
    
    for _, event := range es.events {
        if event.EventType == eventType {
            result = append(result, event)
        }
    }
    
    return result, nil
}

// Save snapshot for performance optimization
func (es *EventStore) SaveSnapshot(snapshot Snapshot) error {
    es.mu.Lock()
    defer es.mu.Unlock()
    
    es.snapshots[snapshot.AggregateID] = snapshot
    fmt.Printf("[Event Store] Saved snapshot for %s at version %d\n", 
        snapshot.AggregateID, snapshot.Version)
    
    return nil
}

// Get latest snapshot
func (es *EventStore) GetSnapshot(aggregateID string) (*Snapshot, error) {
    es.mu.RLock()
    defer es.mu.RUnlock()
    
    snapshot, exists := es.snapshots[aggregateID]
    if !exists {
        return nil, fmt.Errorf("no snapshot found for aggregate %s", aggregateID)
    }
    
    return &snapshot, nil
}

// Helper: Get current version of aggregate
func (es *EventStore) getCurrentVersion(aggregateID string) int {
    version := 0
    for _, event := range es.events {
        if event.AggregateID == aggregateID && event.Version > version {
            version = event.Version
        }
    }
    return version
}

// Get total event count
func (es *EventStore) GetTotalEventCount() int {
    es.mu.RLock()
    defer es.mu.RUnlock()
    return len(es.events)
}

// Get event statistics
func (es *EventStore) GetStatistics() map[string]int {
    es.mu.RLock()
    defer es.mu.RUnlock()
    
    stats := make(map[string]int)
    
    for _, event := range es.events {
        stats[event.EventType]++
    }
    
    return stats
}

func DemonstrateEventStore() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         EVENT STORE                                        ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    store := NewEventStore()
    
    // Append events
    events := []StoredEvent{
        {
            EventID:       "evt-1",
            AggregateID:   "ACC-001",
            AggregateType: "BankAccount",
            EventType:     "AccountOpened",
            EventData:     map[string]interface{}{"initialDeposit": 1000.00},
            Version:       1,
            Timestamp:     time.Now(),
        },
        {
            EventID:       "evt-2",
            AggregateID:   "ACC-001",
            AggregateType: "BankAccount",
            EventType:     "MoneyDeposited",
            EventData:     map[string]interface{}{"amount": 500.00},
            Version:       2,
            Timestamp:     time.Now(),
        },
        {
            EventID:       "evt-3",
            AggregateID:   "ACC-001",
            AggregateType: "BankAccount",
            EventType:     "MoneyWithdrawn",
            EventData:     map[string]interface{}{"amount": 200.00},
            Version:       3,
            Timestamp:     time.Now(),
        },
    }
    
    store.AppendEvents("ACC-001", events, 0)
    
    // Retrieve events
    fmt.Println("\n=== Retrieving Events ===")
    retrieved, _ := store.GetEvents("ACC-001", 0)
    fmt.Printf("Total events retrieved: %d\n", len(retrieved))
    
    // Statistics
    fmt.Println("\n=== Event Statistics ===")
    stats := store.GetStatistics()
    for eventType, count := range stats {
        fmt.Printf("  %s: %d events\n", eventType, count)
    }
    
    fmt.Println("\n✓ Events stored immutably")
    fmt.Println("✓ Events retrieved in order")
    fmt.Println("✓ Optimistic concurrency supported")
    fmt.Println("✓ Snapshots for performance")
}
```

### Component 3: Projections and Read Models

Projections transform events into read models optimized for queries.

```go
package projections

import (
    "fmt"
    "time"
)

// Projection transforms events into read models

// Read Model: Account Balance View
type AccountBalanceView struct {
    AccountID     string
    AccountNumber string
    OwnerID       string
    Balance       float64
    Status        string
    LastUpdated   time.Time
}

// Read Model: Transaction History View
type TransactionHistoryView struct {
    AccountID     string
    Transactions  []Transaction
}

type Transaction struct {
    TransactionID string
    Type          string // "Deposit" or "Withdrawal"
    Amount        float64
    Description   string
    Balance       float64 // Balance after transaction
    Timestamp     time.Time
}

// Read Model: Account Summary View (for dashboard)
type AccountSummaryView struct {
    AccountID         string
    AccountNumber     string
    CurrentBalance    float64
    TotalDeposits     float64
    TotalWithdrawals  float64
    TransactionCount  int
    AccountAge        time.Duration
    Status            string
}

// Projector builds read models from events
type AccountProjector struct {
    balanceViews      map[string]*AccountBalanceView
    transactionViews  map[string]*TransactionHistoryView
    summaryViews      map[string]*AccountSummaryView
}

func NewAccountProjector() *AccountProjector {
    return &AccountProjector{
        balanceViews:     make(map[string]*AccountBalanceView),
        transactionViews: make(map[string]*TransactionHistoryView),
        summaryViews:     make(map[string]*AccountSummaryView),
    }
}

// Project events to update read models
func (p *AccountProjector) ProjectEvent(event StoredEvent) {
    fmt.Printf("[Projector] Processing event: %s for %s\n", 
        event.EventType, event.AggregateID)
    
    switch event.EventType {
    case "AccountOpened":
        p.onAccountOpened(event)
    case "MoneyDeposited":
        p.onMoneyDeposited(event)
    case "MoneyWithdrawn":
        p.onMoneyWithdrawn(event)
    case "AccountClosed":
        p.onAccountClosed(event)
    }
}

func (p *AccountProjector) onAccountOpened(event StoredEvent) {
```

## Advanced Event Sourcing Patterns

### Pattern 1: Snapshots for Performance

As event streams grow long, replaying thousands of events becomes slow. Snapshots capture aggregate state at a point in time, allowing faster reconstruction.

```go
package snapshots

import (
    "fmt"
    "time"
)

// Snapshot captures aggregate state at a specific version
type AggregateSnapshot struct {
    AggregateID   string
    AggregateType string
    Version       int
    State         interface{}
    Timestamp     time.Time
}

// Account state for snapshot
type AccountSnapshotState struct {
    AccountNumber string
    OwnerID       string
    Balance       float64
    Status        string
}

// Snapshot strategy
type SnapshotStrategy interface {
    ShouldTakeSnapshot(version int) bool
}

// Take snapshot every N events
type FixedIntervalStrategy struct {
    Interval int
}

func (s *FixedIntervalStrategy) ShouldTakeSnapshot(version int) bool {
    return version > 0 && version%s.Interval == 0
}

// Aggregate with snapshot support
type SnapshotableAccount struct {
    AccountID     string
    AccountNumber string
    Balance       float64
    Status        string
    Version       int
}

func (a *SnapshotableAccount) CreateSnapshot() AggregateSnapshot {
    return AggregateSnapshot{
        AggregateID:   a.AccountID,
        AggregateType: "BankAccount",
        Version:       a.Version,
        State: AccountSnapshotState{
            AccountNumber: a.AccountNumber,
            OwnerID:       "", // Would be populated from actual state
            Balance:       a.Balance,
            Status:        a.Status,
        },
        Timestamp: time.Now(),
    }
}

func (a *SnapshotableAccount) RestoreFromSnapshot(snapshot AggregateSnapshot) {
    state := snapshot.State.(AccountSnapshotState)
    a.AccountNumber = state.AccountNumber
    a.Balance = state.Balance
    a.Status = state.Status
    a.Version = snapshot.Version
}

// Repository with snapshot support
type AccountRepository struct {
    eventStore    *EventStore
    snapshotStore *SnapshotStore
    strategy      SnapshotStrategy
}

func NewAccountRepository(es *EventStore, ss *SnapshotStore, strategy SnapshotStrategy) *AccountRepository {
    return &AccountRepository{
        eventStore:    es,
        snapshotStore: ss,
        strategy:      strategy,
    }
}

func (repo *AccountRepository) Load(accountID string) (*SnapshotableAccount, error) {
    account := &SnapshotableAccount{AccountID: accountID}
    
    // Try to load from snapshot first
    snapshot, err := repo.snapshotStore.GetLatest(accountID)
    if err == nil {
        fmt.Printf("[Repository] Loaded snapshot at version %d\n", snapshot.Version)
        account.RestoreFromSnapshot(snapshot)
        
        // Load events after snapshot
        events, _ := repo.eventStore.GetEvents(accountID, snapshot.Version)
        fmt.Printf("[Repository] Replaying %d events after snapshot\n", len(events))
        
        for _, event := range events {
            // Apply events to bring account to current state
            account.Version = event.Version
        }
    } else {
        // No snapshot - load all events
        events, _ := repo.eventStore.GetEvents(accountID, 0)
        fmt.Printf("[Repository] No snapshot found, replaying all %d events\n", len(events))
        
        for _, event := range events {
            // Apply events
            account.Version = event.Version
        }
    }
    
    return account, nil
}

func (repo *AccountRepository) Save(account *SnapshotableAccount, events []StoredEvent) error {
    // Append events
    err := repo.eventStore.AppendEvents(account.AccountID, events, account.Version-len(events))
    if err != nil {
        return err
    }
    
    // Check if we should take a snapshot
    if repo.strategy.ShouldTakeSnapshot(account.Version) {
        snapshot := account.CreateSnapshot()
        repo.snapshotStore.Save(snapshot)
        fmt.Printf("[Repository] Created snapshot at version %d\n", account.Version)
    }
    
    return nil
}

type SnapshotStore struct {
    snapshots map[string]AggregateSnapshot
}

func NewSnapshotStore() *SnapshotStore {
    return &SnapshotStore{
        snapshots: make(map[string]AggregateSnapshot),
    }
}

func (ss *SnapshotStore) Save(snapshot AggregateSnapshot) error {
    ss.snapshots[snapshot.AggregateID] = snapshot
    return nil
}

func (ss *SnapshotStore) GetLatest(aggregateID string) (AggregateSnapshot, error) {
    snapshot, exists := ss.snapshots[aggregateID]
    if !exists {
        return AggregateSnapshot{}, fmt.Errorf("no snapshot found")
    }
    return snapshot, nil
}

type EventStore struct {
    events []StoredEvent
}

func NewEventStore() *EventStore {
    return &EventStore{events: make([]StoredEvent, 0)}
}

func (es *EventStore) AppendEvents(aggregateID string, events []StoredEvent, expectedVersion int) error {
    es.events = append(es.events, events...)
    return nil
}

func (es *EventStore) GetEvents(aggregateID string, fromVersion int) ([]StoredEvent, error) {
    result := make([]StoredEvent, 0)
    for _, event := range es.events {
        if event.AggregateID == aggregateID && event.Version > fromVersion {
            result = append(result, event)
        }
    }
    return result, nil
}

func DemonstrateSnapshots() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         SNAPSHOTS FOR PERFORMANCE                          ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    fmt.Println("Without snapshots:")
    fmt.Println("  - Account with 1000 events → replay all 1000 events")
    fmt.Println("  - Slow reconstruction")
    
    fmt.Println("\nWith snapshots (every 100 events):")
    fmt.Println("  - Account with 1000 events → load snapshot at event 900")
    fmt.Println("  - Replay only 100 events")
    fmt.Println("  - 10x faster reconstruction")
    
    fmt.Println("\n✓ Snapshots are optimization, not source of truth")
    fmt.Println("✓ Can always rebuild from events if snapshot corrupted")
    fmt.Println("✓ Strategy determines snapshot frequency")
}
```

### Pattern 2: Event Upcasting for Schema Evolution

As your system evolves, event schemas change. Event Upcasting allows old events to be transformed to new formats.

```go
package upcasting

import (
    "fmt"
    "time"
)

// Version 1: Original event
type OrderPlacedV1 struct {
    OrderID    string
    CustomerID string
    Total      float64
}

// Version 2: Added items detail
type OrderPlacedV2 struct {
    OrderID    string
    CustomerID string
    Items      []Item
    Total      float64
}

type Item struct {
    ProductID string
    Quantity  int
    Price     float64
}

// Version 3: Added timestamp and currency
type OrderPlacedV3 struct {
    OrderID    string
    CustomerID string
    Items      []Item
    Total      float64
    Currency   string
    PlacedAt   time.Time
}

// Upcaster transforms old events to new schema
type EventUpcaster interface {
    Upcast(event interface{}) (interface{}, error)
}

type OrderPlacedUpcaster struct{}

func (u *OrderPlacedUpcaster) Upcast(event interface{}) (interface{}, error) {
    switch v := event.(type) {
    case OrderPlacedV1:
        // V1 → V2: Create default items
        return OrderPlacedV2{
            OrderID:    v.OrderID,
            CustomerID: v.CustomerID,
            Items: []Item{
                {ProductID: "UNKNOWN", Quantity: 1, Price: v.Total},
            },
            Total: v.Total,
        }, nil
        
    case OrderPlacedV2:
        // V2 → V3: Add defaults for new fields
        return OrderPlacedV3{
            OrderID:    v.OrderID,
            CustomerID: v.CustomerID,
            Items:      v.Items,
            Total:      v.Total,
            Currency:   "USD",        // Default currency
            PlacedAt:   time.Now(),   // Approximate timestamp
        }, nil
        
    case OrderPlacedV3:
        // Already latest version
        return v, nil
        
    default:
        return nil, fmt.Errorf("unknown event type")
    }
}

func DemonstrateUpcasting() {
    fmt.Println("╔════════════════════════════════════════════════════════════╗")
    fmt.Println("║         EVENT UPCASTING                                    ║")
    fmt.Println("╚════════════════════════════════════════════════════════════╝\n")
    
    upcaster := &OrderPlacedUpcaster{}
    
    // Old event from version 1
    oldEvent := OrderPlacedV1{
        OrderID:    "ORD-001",
        CustomerID: "CUST-123",
        Total:      99.99,
    }
    
    fmt.Println("Original event (V1):")
    fmt.Printf("  %+v\n", oldEvent)
    
    // Upcast to V2
    v2Event, _ := upcaster.Upcast(oldEvent)
    fmt.Println("\nAfter upcasting to V2:")
    fmt.Printf("  %+v\n", v2Event)
    
    // Upcast to V3
    v3Event, _ := upcaster.Upcast(v2Event)
    fmt.Println("\nAfter upcasting to V3:")
    fmt.Printf("  %+v\n", v3Event)
    
    fmt.Println("\n✓ Old events transformed to new schema")
    fmt.Println("✓ System works with latest event version")
    fmt.Println("✓ No need to migrate historical data")
}
```

## Benefits and Trade-offs of Event Sourcing

### Key Benefits

**Complete Audit Trail**: Every change is recorded as an immutable event. You have a complete history of what happened, when, and why. This is invaluable for compliance, debugging, and business intelligence.

**Temporal Queries**: You can answer questions like "what was the state at 3pm yesterday?" by replaying events up to that point. Time-travel debugging becomes possible.

**Event Replay**: If you discover a bug in your business logic, you can fix it and replay all events to correct the state. If you need a new read model, build it by replaying existing events.

**Natural Audit Log**: Events are naturally suited for audit requirements. Each event captures who did what, when, and why.

**Event-Driven Architecture**: Events can trigger notifications, update projections, and integrate with external systems naturally.

### Key Trade-offs

**Complexity**: Event Sourcing is more complex than traditional CRUD. You must manage event stores, handle event versioning, build projections, and deal with eventual consistency.

**Eventual Consistency**: Read models are eventually consistent with write models. Your UI must handle this gracefully.

**Learning Curve**: Team members need to understand new concepts: aggregates, events, projections, snapshots. This takes time and training.

**Event Schema Evolution**: As your domain evolves, event schemas change. You need strategies like upcasting to handle old events.

**Storage Growth**: Events are never deleted, so storage grows continuously. You need strategies for archiving old events.

## When to Use Event Sourcing

### Ideal Scenarios

**Complex Domains with Rich Behavior**: When your domain has complex workflows, multiple state transitions, and intricate business rules, Event Sourcing's explicit event modeling clarifies the domain.

**Audit Requirements**: When regulatory compliance demands detailed audit trails of all changes, Event Sourcing provides this naturally.

**Temporal Analysis**: When you need to analyze historical data, understand trends over time, or perform "what-if" analysis on past states.

**Event-Driven Systems**: When your architecture is already event-driven with event buses, message queues, and reactive components.

**Multiple Views of Same Data**: When different users need different denormalized views of the same underlying data.

### Poor Fit Scenarios

**Simple CRUD Applications**: If your application is primarily simple data entry and retrieval without complex business logic, Event Sourcing adds unnecessary complexity.

**Immediate Consistency Required**: If your business absolutely requires immediate consistency between writes and reads, Event Sourcing's eventual consistency is problematic.

**Small Team Without Expertise**: If your team lacks distributed systems experience and time to learn, Event Sourcing may overwhelm them.

**Frequent Schema Changes Early On**: If your domain is still being discovered and event schemas change frequently, the overhead of event versioning becomes burdensome.

## Conclusion: Event Sourcing as a Powerful Tool for Complex Systems

Event Sourcing represents a fundamental shift in how we think about data persistence. Rather than storing current state and discarding history, we store the sequence of events that tell the complete story of what happened. This inversion unlocks powerful capabilities: complete audit trails, temporal queries, event replay, and natural integration with event-driven architectures.

However, Event Sourcing is not a universal solution. It adds significant complexity and requires careful design of aggregates, events, projections, and event stores. The eventual consistency between write and read models demands thoughtful UI design. Event schema evolution requires planning and tooling.

In Go, with its excellent concurrency support, clear separation of concerns through interfaces, and strong typing, Event Sourcing can be implemented cleanly and idiomatically. Your aggregates become clear expressions of domain logic. Your events become immutable facts. Your projections become optimized read models.

Apply Event Sourcing judiciously—in domains where its benefits clearly outweigh its costs. When you have complex business logic, audit requirements, temporal analysis needs, or event-driven architecture, Event Sourcing provides a solid foundation. For simpler domains, traditional approaches often suffice.

As you build your Go-based ERP system, consider Event Sourcing for those critical aggregates where understanding what happened is as important as knowing the current state. Your accounting ledger, order processing, inventory movements, and audit logs are natural candidates. The complete history Event Sourcing provides becomes invaluable as your business grows and regulatory scrutiny increases.
