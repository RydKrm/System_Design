### Blur as a Design Tool — Focus, Depth, and Atmosphere

Blur in digital interfaces serves three distinct purposes, each governed by different principles and implemented with different techniques. The first purpose is **depth simulation** — blurring background content to create the illusion that the foreground element is in focus while the background is spatially farther away, imitating the depth of field effect of camera lenses. The second purpose is **content subordination** — blurring content that is present but not currently relevant, allowing it to remain visible (preserving context) while communicating that interaction with it is temporarily unavailable. The third purpose is **atmospheric texture** — using blur to create soft, glowing, painterly background effects that give depth and richness to a composition without the complexity of photographic imagery.

In CSS, blur is most commonly applied through three properties: `filter: blur()` applies a Gaussian blur to an element and all its contents; `backdrop-filter: blur()` applies a blur to the content visible behind an element (the foundation of glassmorphism, discussed earlier); and the CSS `blur()` function within gradient definitions creates soft, diffuse color fields when used with radial gradients.

### Performance Considerations of Blur

Blur is among the most computationally expensive CSS effects, particularly `backdrop-filter`, which requires the browser to composite and blur the layers beneath an element on every frame. On underpowered devices or when many blurred elements appear simultaneously, this can cause significant performance degradation, manifesting as dropped frames, janky animations, and excessive battery drain on mobile devices.

Several strategies mitigate this cost. The first is to ensure that blurred elements are promoted to their own GPU compositing layer using `will-change: transform` or `transform: translateZ(0)` — this allows the GPU to handle the blur in hardware rather than through software rendering. The second is to use `@media (prefers-reduced-motion)` to disable or reduce blur effects for users who have requested reduced visual complexity (which often correlates with lower-powered devices). The third is to limit the number of simultaneously blurred elements — a single modal with a blurred backdrop is far less costly than a page of fifteen simultaneously blurred cards.
