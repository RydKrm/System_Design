### What Is a Spacing System?

A spacing system is a defined vocabulary of distance values — a curated set of numbers for padding, margin, and gap — that is used consistently throughout an entire design. It is the spacing equivalent of a type scale: just as a type scale limits you to a small set of pre-approved font sizes (rather than choosing any size freely), a spacing system limits you to a small set of pre-approved spacing values.

The spacing system and the grid system are intimately related. The spacing system provides the values; the grid provides the structural framework within which those values are applied. Together, they ensure that every dimension in a design is intentional, systematic, and harmonious.

### Spacing Scale Design — How to Build One

A well-designed spacing scale has several key properties. First, it is derived from a single base unit (4px or 8px). Second, the steps between values increase as the values increase — small spacings have small steps between them, large spacings have larger steps. Third, the scale covers the full range from micro-spacing (4px between icon and label) to macro-spacing (120px+ for hero section padding).

The most sophisticated spacing scales use a **modular** approach — each step is a consistent multiple or ratio of the previous step. The most common ratios are 2x (doubling: 4, 8, 16, 32, 64, 128) or approximately 1.5x (roughly: 4, 8, 12, 20, 32, 48, 80, 128). The 1.5x approach produces a more gradual, usable scale; the 2x approach produces a cleaner, more mathematical scale with fewer intermediate values.

```
SPACING SCALE TYPES:

  LINEAR (equal steps — simple but limiting):
  4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 44, 48...
  Problem: Too many values, hard to make decisions.
  Fine for micro-spacing but unwieldy at large values.

  EXPONENTIAL / DOUBLING (clean math, large gaps at top):
  4, 8, 16, 32, 64, 128
  Problem: Jumps between large values are too big.
  Gap between 32 and 64 is 32px — too coarse.

  MODULAR (best of both — used by Tailwind, Material):
  ┌──────────────────────────────────────────────────────────┐
  │  Step  Value   Name          Primary Use                 │
  ├──────────────────────────────────────────────────────────┤
  │   1     4px    Hairline      Icon gap, inline micro      │

  │   3    12px    Compact       Badge/chip padding          │
  │   4    16px    Default       Standard padding, margin    │
  │   5    20px    Comfortable   Medium component gap        │
  │   6    24px    Relaxed       Card padding, form spacing  │
  │   7    32px    Loose         Section padding, large gap  │
  │   8    40px    Generous      Between major components    │
  │   9    48px    Spacious      Section breaks, targets     │
  │  10    64px    Open          Page section spacing        │
  │  11    80px    Airy          Large section spacing       │
  │  12    96px    Expansive     Desktop section breaks      │
  │  13   120px    Grand         Hero sections               │
  │  14   160px    Monumental    Maximum desktop padding     │
  └──────────────────────────────────────────────────────────┘

  SPACING DECISION FRAMEWORK:

  Question: "How much space should go here?"

  Between two items IN the same component
  (icon + label, tag + text, list item content):
  → Use 4px or 8px (micro-spacing)

  Between components WITHIN a card or container
  (title + description, image + caption):
  → Use 8px or 16px

  INSIDE a card or container (card padding):
  → Use 16px or 24px

  Between cards or components on a page:
  → Use 24px or 32px

  Between major page sections (mobile):
  → Use 48px or 64px

  Between major page sections (desktop):
  → Use 80px or 96px

  Hero sections and large dramatic gaps (desktop):
  → Use 120px or 160px


SPACING AND VISUAL HIERARCHY — The "Proximity Ladder":

  The amount of space between elements communicates
  how closely related they are.

  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │  SECTION TITLE                                           │
  │  ↕ 8px (title very close to its content → related)      │
  │  Section description paragraph text goes here.          │
  │  More description text continues on this line.          │
  │                                                          │
  │  ↕ 48px (large gap → completely new section)            │
  │                                                          │
  │  NEXT SECTION TITLE                                      │
  │  ↕ 8px                                                   │
  │  This section's content begins here.                     │
  │                                                          │
  └──────────────────────────────────────────────────────────┘

  The large gap (48px) between sections communicates:
  "These belong to different groups."

  The small gap (8px) between title and content says:
  "This title belongs to this content."

  This is the Gestalt Proximity principle implemented
  systematically through the spacing scale.
```

### Spacing Tokens — Making the System Live in Code

The spacing system only delivers its full value when it is implemented as **design tokens** — named variables that map human-readable names to specific pixel values, and are shared between design tools (Figma, Sketch) and the codebase (CSS, iOS, Android). When a designer uses the `space-4` token in their Figma file and a developer uses the `space-4` token in their CSS file, both are referring to the same 16px value, and the design-to-code handoff is perfectly precise.

Without tokens, a designer specifies "24px padding" in their design file, and a developer either types 24px directly in CSS (creating a magic number that is not connected to the system) or types 20px because they misread the spec (creating an off-system value). Over time, codebases without spacing tokens accumulate hundreds of slightly different spacing values — 13px, 14px, 15px, 16px, 17px, 18px all appearing in different places where the design system intended only 16px.
