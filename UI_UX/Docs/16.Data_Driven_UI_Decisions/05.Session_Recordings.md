## Session Recordings — The Most Honest Research You Will Ever Do

### Watching Real Users, In Real Time, With Real Intent

Session recordings are video-like recordings of individual user sessions — complete replays of exactly what a user did on your product, including every mouse movement, every scroll, every click, every keystroke (with sensitive data masked), every moment of hesitation, every rage click, and every moment of confusion made visible through back-and-forth navigation. They are generated by the same JavaScript libraries that produce heatmaps, and they represent the most intimate form of behavioral data available — not a statistical aggregate of many users' behavior, but the complete, unedited record of one person's experience with your product.

The value of session recordings is precisely their specificity. Where heatmaps show you that users tend to hesitate before the pricing field in a form, a session recording shows you the exact nature of that hesitation — the cursor hovering over the field label, the user scrolling up to re-read the value proposition, the user tabbing back to a previous field to re-read a confirmation, the user typing and deleting a response twice. The heatmap quantifies the problem; the recording humanizes it. And humanizing a problem — seeing it as a specific human being's specific struggle — is often what triggers the creative insight that a purely quantitative view cannot.

SESSION RECORDING ANALYSIS — What to Look For
══════════════════════════════════════════════════════════════════════

  THE BEHAVIORAL SIGNALS IN A SESSION RECORDING:
  ─────────────────────────────────────────────────────────────────

  SIGNAL: U-TURNS (user navigates forward then immediately back)
  ┌─────────────────────────────────────────────────────────────────┐
  │  Page A → Page B → [immediately] → Page A                    │
  │                                                                 │
  │  Meaning: Page B failed to meet the expectation set by       │
  │  the link on Page A. The user expected something different.  │
  │                                                                 │
  │  Fix: Align the link label on Page A more precisely with     │
  │  the content of Page B. Or improve Page B's immediate        │
  │  value communication.                                         │
  └─────────────────────────────────────────────────────────────────┘

  SIGNAL: RAGE CLICKS (multiple rapid clicks in same location)
  ┌─────────────────────────────────────────────────────────────────┐
  │  Click. Click. Click. Click. Click. [same element, <2 seconds] │
  │                                                                 │
  │  Meaning: User expects this element to be interactive         │
  │  but it is not responding. Either:                            │
  │  (a) The element is not interactive but looks like it is     │
  │  (b) The element IS interactive but is broken/slow          │
  │  (c) The element has very small touch target (mobile)        │
  │                                                                 │
  │  Rage clicks are among the highest-signal behavioral events  │
  │  — they indicate guaranteed user frustration. Prioritize.    │
  └─────────────────────────────────────────────────────────────────┘

  SIGNAL: DEAD ENDS (user reaches a page with nowhere to go)
  ┌─────────────────────────────────────────────────────────────────┐
  │  User arrives at page → scrolls down to bottom → stops →     │
  │  no further action → tab closes.                              │
  │                                                                 │
  │  Meaning: No clear next action was presented after the        │
  │  content ended. User wanted to continue but could not find   │
  │  a path forward.                                              │
  │                                                                 │
  │  Fix: Add a clear CTA or "next step" prompt at the end       │
  │  of every content page. Never let a page end without a path. │
  └─────────────────────────────────────────────────────────────────┘

  SIGNAL: FORM FIELD HESITATION
  ┌─────────────────────────────────────────────────────────────────┐
  │  User reaches field → cursor hovers → moves away → returns   │
  │  → types slowly → deletes → retypes → tabs to next field     │
  │                                                                 │
  │  Meaning: The field label is ambiguous, the user is unsure   │
  │  what is being asked, or they are anxious about providing    │
  │  the information (phone number, company size, card details). │
  │                                                                 │
  │  Fix: Add helper text explaining why the field is needed.    │
  │  Or remove the field if the hesitation data is widespread.   │
  └─────────────────────────────────────────────────────────────────┘

  SIGNAL: SCROLL-HUNTING (rapid up-down scrolling)
  ┌─────────────────────────────────────────────────────────────────┐
  │  User scrolls rapidly up and down repeatedly in same area     │
  │                                                                 │
  │  Meaning: User is looking for specific information they      │
  │  cannot find. They have scanned the page once, not found     │
  │  what they needed, and are searching again more carefully.   │
  │                                                                 │
  │  Fix: The information they need is either absent, or present │
  │  but visually deprioritized. Elevate it in the hierarchy.   │
  └─────────────────────────────────────────────────────────────────┘

  SESSION RECORDING WORKFLOW:
  ┌─────────────────────────────────────────────────────────────────┐
  │                                                                 │
  │  1. Identify high-drop-off pages from funnel analysis         │
  │  2. Filter recordings to sessions that dropped off at         │
  │     those specific pages                                      │
  │  3. Watch 15–25 recordings from this filtered segment        │
  │  4. Document every behavioral signal observed                 │
  │  5. Look for patterns that appear in 3+ recordings           │
  │  6. Each repeated pattern = a design hypothesis to test      │
  │                                                                 │
  │  TIME INVESTMENT: 15–25 recordings × 3–5 min average = 1–2  │
  │  hours. Insight yield: typically 3–8 actionable hypotheses.  │
  │                                                                 │
  └─────────────────────────────────────────────────────────────────┘
