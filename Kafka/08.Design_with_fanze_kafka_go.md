# Kafka Producer & Consumer with franz-go

### Every line explained — github.com/twmb/franz-go

---

## Preface — Why franz-go Is Different

Before reading a single line of code, it is worth understanding what makes franz-go different from confluent-kafka-go, because the difference explains why the API looks the way it does.

confluent-kafka-go is a thin Go wrapper around `librdkafka`, a C library. It works perfectly, but it requires you to install C headers (`librdkafka-dev`) on every machine that compiles your code — your laptop, your CI server, your Docker build container. This is called a CGO dependency and it adds operational friction.

franz-go is written entirely in pure Go. Not a single line of C. This means you can compile it with `CGO_ENABLED=0`, ship a fully static binary, and run it on any Linux container without any native library dependencies whatsoever. This is a significant operational advantage in Docker and Kubernetes environments.

The second difference is the API design philosophy. franz-go uses a **single client** for both producing and consuming, whereas confluent uses two separate client types. franz-go is also the only Go Kafka library that defaults to **cooperative incremental rebalancing** — the modern rebalancing protocol that avoids the stop-the-world pause during rebalances that we discussed in the consumer group lifecycle guide.

The third difference is the package structure:

- `github.com/twmb/franz-go/pkg/kgo` — the main producer/consumer client
- `github.com/twmb/franz-go/pkg/kadm` — the admin client for creating topics

## Part 1 — Shared Config (`config/config.go`)

```go
package config

import "time"

// Config holds connection settings shared across admin, producer,
// and consumer. Centralising these means one change propagates
// everywhere — you never chase a broker address across five files.
type Config struct {
    // Brokers is a slice of "host:port" seed broker addresses.
    // franz-go calls these "seed brokers" — you only need ONE to
    // start. The library performs the bootstrap metadata exchange
    // with the seed and discovers all other brokers automatically.
    // List 2–3 in production so a single broker restart does not
    // block your service from starting.
    Brokers []string

    // GroupID is the consumer group identity — the same concept
    // explained in depth in the previous guides.
    // Same GroupID = instances share partitions (work-splitting).
    // Different GroupID = each service gets all messages (broadcast).
    GroupID string

    // ClientID is a human-readable label for this process.
    // Appears in Kafka broker logs and Kafka UI dashboards.
    // Does not affect routing or group behaviour.
    ClientID string

    // SessionTimeout is how long the group coordinator waits
    // without a heartbeat before declaring this consumer dead
    // and triggering a rebalance. 45 seconds is the standard.
    SessionTimeout time.Duration

    // HeartbeatInterval is how often the client sends a heartbeat
    // to the coordinator to prove it is alive. Must be less than
    // SessionTimeout / 3. 3 seconds is the standard.
    HeartbeatInterval time.Duration
}

// DefaultConfig returns sensible defaults for development.
// In production, read Brokers from an environment variable and
// tune timeouts based on your SLA requirements.
func DefaultConfig(brokers []string, groupID, clientID string) Config {
    return Config{
        Brokers:           brokers,
        GroupID:           groupID,
        ClientID:          clientID,
        SessionTimeout:    45 * time.Second,
        HeartbeatInterval: 3 * time.Second,
    }
}

// Topic name constants. Always use constants, never raw strings.
// A typo in a string constant is caught at compile time.
// A typo in a raw string literal fails silently at runtime.
const (
    TopicOrderPlaced       = "order.placed"
    TopicInventoryReserved = "inventory.reserved"
    TopicInventoryFailed   = "inventory.failed"
    TopicNotification      = "notification.send"
    TopicUserRegistered    = "user.registered"
)
```

## Part 2 — Creating Topics with kadm (`topic/admin.go`)

```go
package topic

import (
    "context"
    "fmt"
    "log"
    "time"

    "github.com/twmb/franz-go/pkg/kadm"
    "github.com/twmb/franz-go/pkg/kgo"
)

// TopicSpec describes a topic to create. We define our own struct
// so that calling code does not need to import kadm directly —
// only this package needs that dependency.
type TopicSpec struct {
    Name              string
    NumPartitions     int32
    ReplicationFactor int16
    Configs           map[string]*string // pointer because kadm uses *string
}

// AdminClient wraps kadm.Client to add structured logging.
// kadm is franz-go's separate admin package — it provides topic
// management, consumer group inspection, and offset administration.
// It is built on top of a kgo.Client, not a separate connection.
type AdminClient struct {
    client *kadm.Client
}

// NewAdminClient creates an admin client connected to Kafka.
// Under the hood, kadm wraps a kgo.Client with admin-specific
// methods. You never call kgo directly for admin operations —
// kadm handles the raw Kafka admin protocol for you.
func NewAdminClient(brokers []string) (*AdminClient, error) {
    // kgo.NewClient creates the underlying connection.
    // For admin-only use, we pass just the seed brokers — no
    // consumer group, no topic subscriptions.
    kgoClient, err := kgo.NewClient(
        // kgo.SeedBrokers takes a variadic list of "host:port" strings.
        // It does NOT take a comma-separated string — this is a key
        // difference from confluent-kafka-go which used "b1,b2,b3".
        // Here we pass a slice using the "..." spread operator.
        kgo.SeedBrokers(brokers...),
    )
    if err != nil {
        return nil, fmt.Errorf("failed to create kgo client for admin: %w", err)
    }

    // kadm.NewClient wraps the kgo.Client to expose admin operations.
    // The kgo.Client is still the underlying TCP connection — kadm
    // just adds a typed API layer on top of raw Kafka protocol requests.
    adminClient := kadm.NewClient(kgoClient)

    log.Printf("[Admin] Connected to Kafka: %v", brokers)
    return &AdminClient{client: adminClient}, nil
}

// StringPtr is a small helper because kadm topic config values
// are *string (pointer to string), not string. Kafka distinguishes
// between "not set" (nil pointer) and "set to empty string" (pointer
// to ""). We need the helper to create pointers from literals cleanly.
func StringPtr(s string) *string { return &s }

// EnsureTopics creates topics that do not yet exist.
// Safe to call on every service startup — existing topics are
// left unchanged. This is the "ensure" / idempotent create pattern.
func (a *AdminClient) EnsureTopics(specs []TopicSpec) error {
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()

    // Build kadm's TopicConfig slice from our own spec structs.
    topicConfigs := make([]kadm.CreateTopicConfig, len(specs))
    for i, spec := range specs {
        topicConfigs[i] = kadm.CreateTopicConfig{
            // Topic is the name of the topic to create.
            Topic: spec.Name,

            // NumPartitions declares the partition count.
            // This directly answers the question from the previous
            // guide: declare this explicitly, never rely on the
            // broker's num.partitions default (which is 1).
            // -1 means "use broker default" — never use -1 in production.
            NumPartitions: spec.NumPartitions,

            // ReplicationFactor: how many brokers keep a copy.
            // -1 means "use broker default".
            // 1 for development, 3 for production.
            ReplicationFactor: spec.ReplicationFactor,

            // Configs are per-topic overrides of broker defaults.
            // Examples:
            //   "retention.ms":       "604800000"  → keep 7 days
            //   "cleanup.policy":     "compact"    → log compaction
            //   "min.insync.replicas":"2"          → durability
            Configs: spec.Configs,
        }
    }

    // CreateTopics sends the creation request to the cluster.
    // kadm returns a TopicsDetails map — one entry per topic —
    // each containing whether it succeeded or what error occurred.
    resp, err := a.client.CreateTopics(ctx, topicConfigs...)
    if err != nil {
        return fmt.Errorf("CreateTopics request failed: %w", err)
    }

    // Inspect each topic's result individually.
    // Some may succeed while others fail — we must check each one.
    for _, detail := range resp {
        if detail.Err != nil {
            // kerr.TopicAlreadyExists means the topic is already there.
            // This is NOT a failure — it is exactly what we want.
            // We check the error message string because kadm surfaces
            // Kafka error codes through the detail.Err field.
            if detail.Err.Error() == "TOPIC_ALREADY_EXISTS" {
                log.Printf("[Admin] Topic already exists (ok): %s",
                    detail.Topic)
                continue
            }
            return fmt.Errorf("failed to create topic %s: %w",
                detail.Topic, detail.Err)
        }
        log.Printf("[Admin] ✅ Topic created: %s (partitions=%d rf=%d)",
            detail.Topic, detail.NumPartitions, detail.ReplicationFactor)
    }
    return nil
}

// DescribeTopics prints partition leader and replica information.
// Useful for verifying that topics were created with the correct
// partition count and that all replicas are in-sync.
func (a *AdminClient) DescribeTopics(topicNames ...string) error {
    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()

    // ListTopics returns metadata about the named topics.
    // Pass no arguments to list all topics in the cluster.
    details, err := a.client.ListTopics(ctx, topicNames...)
    if err != nil {
        return fmt.Errorf("ListTopics failed: %w", err)
    }

    for _, topic := range details.Sorted() {
        log.Printf("[Admin] Topic: %s  Partitions: %d",
            topic.Topic, len(topic.Partitions))
        for _, p := range topic.Partitions.Sorted() {
            log.Printf("[Admin]   P%d | Leader: %d | Replicas: %v | ISR: %v",
                p.Partition, p.Leader, p.Replicas, p.ISR)
        }
    }
    return nil
}

// Close releases the admin client's resources.
// Always defer this after calling NewAdminClient.
func (a *AdminClient) Close() {
    a.client.Close()
    log.Println("[Admin] Admin client closed")
}
```

## Part 3 — The Producer (`producer/producer.go`)

```go
package producer

import (
    "context"
    "encoding/json"
    "fmt"
    "log"
    "sync/atomic"
    "time"

    "github.com/google/uuid"
    "github.com/twmb/franz-go/pkg/kgo"
)

// Producer wraps kgo.Client for producing only.
// In franz-go, the SAME kgo.Client can produce AND consume.
// We wrap it anyway so service code stays clean — it does not
// need to know about kgo directly. The wrapper also tracks
// delivery statistics using atomic counters.
type Producer struct {
    client      *kgo.Client
    serviceName string
    delivered   atomic.Int64 // count of successfully delivered messages
    failed      atomic.Int64 // count of permanently failed messages
}

// NewProducer creates a kgo.Client configured for producing only.
// Every kgo option is passed as a functional option to kgo.NewClient.
// This is the franz-go way: instead of a ConfigMap string dictionary,
// you pass typed option functions. Typos in option names are compile
// errors, not runtime surprises.
func NewProducer(brokers []string, serviceName string) (*Producer, error) {
    client, err := kgo.NewClient(
        // ── CONNECTION ────────────────────────────────────────────
        // kgo.SeedBrokers accepts a variadic list of "host:port" strings.
        // Unlike confluent's comma-separated "b1:9092,b2:9092" string,
        // franz-go takes separate arguments: SeedBrokers("b1:9092", "b2:9092")
        // or a spread slice: SeedBrokers(brokers...)
        kgo.SeedBrokers(brokers...),

        // kgo.ClientID sets the human-readable label for this client.
        // Appears in broker logs, Kafka UI, and monitoring dashboards.
        // Helps you answer "which service is writing to this topic?"
        // during an incident without guessing.
        kgo.ClientID(serviceName+"-producer"),

        // ── DURABILITY ────────────────────────────────────────────
        // kgo.RequiredAcks controls how many replicas must acknowledge
        // a write before the producer considers it safe.
        //
        // kgo.AllISRAcks() → waits for ALL in-sync replicas to confirm.
        //   This is the safest mode. Even if the leader crashes immediately
        //   after acknowledging, at least one follower has the message.
        //   Combined with min.insync.replicas=2 on the broker, this gives
        //   you a zero-data-loss guarantee.
        //
        // kgo.LeaderAck() → waits only for the leader to confirm.
        //   Faster but less safe. If the leader crashes before replication
        //   completes, the message is lost.
        //
        // kgo.NoAck() → fire and forget. Producer does not wait for
        //   any confirmation. Fastest, zero durability guarantee.
        //
        // For microservices handling orders, payments, or any business
        // event where losing a message is unacceptable: AllISRAcks().
        kgo.RequiredAcks(kgo.AllISRAcks()),

        // ── IDEMPOTENCE ──────────────────────────────────────────
        // franz-go enables idempotent production BY DEFAULT.
        // You do NOT need to set it — it is already on.
        //
        // What does idempotence mean?
        // Each message gets a sequence number. If a network failure
        // causes the producer to retry, the broker uses the sequence
        // number to detect that it already stored this message and
        // silently discards the duplicate. Without idempotence, a
        // network blip can result in the same order being stored twice.
        //
        // To explicitly DISABLE it (rarely the right choice):
        // kgo.DisableIdempotentWrite()
        //
        // We leave it at the default ON. No option needed.

        // ── RETRIES ───────────────────────────────────────────────
        // By default, franz-go retries records FOREVER until they are
        // delivered or until RecordDeliveryTimeout expires.
        // This is safer than a fixed retry count — transient failures
        // (leader elections, network blips) resolve on their own and
        // the record gets delivered without manual intervention.
        //
        // kgo.RecordDeliveryTimeout sets the maximum total time a
        // record can spend trying to be delivered. After this timeout,
        // the record is failed permanently.
        // 30 seconds is generous for transient failures.
        kgo.RecordDeliveryTimeout(30 * time.Second),

        // kgo.RetryBackoffFn controls how long the producer waits
        // between retry attempts. The default is an exponential
        // backoff starting at 100ms, capped at 1 second. This is
        // sensible and you rarely need to override it.
        // Shown here for documentation purposes.
        kgo.RetryBackoffFn(func(attempt int) time.Duration {
            // Exponential backoff: 100ms, 200ms, 400ms, 800ms, 1s, 1s...
            d := 100 * time.Millisecond * time.Duration(1<<attempt)
            if d > time.Second {
                d = time.Second
            }
            return d
        }),

        // ── BATCHING (LATENCY vs THROUGHPUT) ─────────────────────
        // kgo.ProducerLinger sets how long the producer accumulates
        // records before sending a batch.
        //
        // 0 (default) → send immediately when a record is produced.
        //   Best latency. In high-throughput scenarios this is still
        //   fine because records accumulate naturally while previous
        //   batches are in flight. But for low-throughput services,
        //   each message becomes its own network round trip.
        //
        // 5ms → wait up to 5ms for more records to accumulate.
        //   In 5ms a busy service can accumulate hundreds of records
        //   and send them in one network round trip. Trades a small
        //   predictable latency increase for much higher throughput.
        //   The right choice for most production services.
        kgo.ProducerLinger(5 * time.Millisecond),

        // kgo.ProducerBatchMaxBytes sets the maximum batch size in bytes.
        // The producer sends a batch when either the linger expires OR
        // the batch reaches this size — whichever comes first.
        // 1MB is the franz-go default and a good production value.
        kgo.ProducerBatchMaxBytes(1_000_000), // 1 MB

        // ── COMPRESSION ──────────────────────────────────────────
        // kgo.ProducerBatchCompression sets the compression algorithm.
        // You pass one or more codecs in preference order. franz-go
        // tries the first one; if the broker does not support it,
        // it falls back to the next.
        //
        // kgo.SnappyCompression() → fast, good ratio, pure Go impl.
        // kgo.Lz4Compression()   → very fast, excellent ratio.
        // kgo.ZstdCompression()  → best ratio, moderate CPU.
        // kgo.GzipCompression()  → best ratio, slowest CPU.
        // kgo.NoCompression()    → no compression (for Event Hubs etc.)
        //
        // For JSON payloads, Snappy typically achieves 3–5× compression
        // with negligible CPU overhead. It is the sensible default.
        kgo.ProducerBatchCompression(kgo.SnappyCompression()),

        // ── NETWORK ───────────────────────────────────────────────
        // kgo.DialTLSConfig sets TLS for encrypted connections.
        // Omitted here for development. In production:
        // kgo.DialTLSConfig(tlsConfig)
        //
        // kgo.SASL sets authentication credentials.
        // Omitted here for development. In production:
        // kgo.SASL(scram.Auth{...}.AsSASLMechanism())
    )
    if err != nil {
        return nil, fmt.Errorf("[%s] failed to create producer: %w",
            serviceName, err)
    }

    // Ping verifies the connection is actually reachable.
    // Without Ping, errors only surface on the first Produce call,
    // which may be seconds or minutes after startup. Failing fast
    // here means your service crashes at startup with a clear error
    // rather than silently dropping messages later.
    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()
    if err := client.Ping(ctx); err != nil {
        client.Close()
        return nil, fmt.Errorf("[%s] Kafka unreachable: %w", serviceName, err)
    }

    log.Printf("[%s] Producer connected to %v", serviceName, brokers)
    return &Producer{client: client, serviceName: serviceName}, nil
}

// BaseEvent is the envelope every event must carry.
// Embedding this struct in every event type ensures that
// every message has a unique ID, a type label, a timestamp,
// and a schema version — the minimum metadata any consumer needs.
type BaseEvent struct {
    EventID   string    `json:"event_id"`
    EventType string    `json:"event_type"`
    Timestamp time.Time `json:"timestamp"`
    Version   string    `json:"version"`
}

// Publish sends one message to a Kafka topic asynchronously.
//
// Parameters:
//   ctx     — cancellation context. If cancelled before delivery,
//             the record is failed.
//   topic   — the topic name (use constants from config package).
//   key     — routing key. hash(key) % numPartitions = partition.
//             Use an entity ID (orderID, userID) for ordering.
//             Use "" for no key (round-robin, no ordering).
//   payload — any JSON-serialisable Go value.
func (p *Producer) Publish(ctx context.Context,
    topic, key string, payload interface{}) error {

    // Serialise payload to JSON bytes.
    // Kafka is byte-agnostic — it stores whatever bytes you give it.
    // JSON is the simplest format for multi-language systems.
    // In production, consider Avro with Schema Registry for
    // schema enforcement and better compression ratios.
    data, err := json.Marshal(payload)
    if err != nil {
        return fmt.Errorf("[%s] marshal error for %s: %w",
            p.serviceName, topic, err)
    }

    // Build the kgo.Record — franz-go's equivalent of confluent's
    // kafka.Message. Every field maps directly to a Kafka message field.
    record := &kgo.Record{
        // Topic names the target topic.
        Topic: topic,

        // Key is the routing key in bytes.
        // When non-nil, franz-go hashes it to pick the partition:
        //   partition = hash(Key) % numPartitions
        // Same key always → same partition → ordered delivery.
        // When nil, records are distributed round-robin.
        Key: []byte(key),

        // Value is the message payload — the JSON bytes.
        Value: data,

        // Timestamp sets the producer-side event time.
        // This is the time the event OCCURRED, which is important
        // for time-based analysis. The broker records its own arrival
        // time separately. Using producer timestamp (the default in
        // Kafka) gives you the most accurate event ordering.
        Timestamp: time.Now(),

        // Headers are optional envelope metadata readable without
        // deserialising the Value. Useful for tracing, routing,
        // and filtering in consumer handlers.
        Headers: []kgo.RecordHeader{
            // Which service produced this? Visible in Kafka UI
            // without deserialising the payload.
            {Key: "source-service", Value: []byte(p.serviceName)},

            // Distributed trace ID for correlating this event
            // with the HTTP request that triggered it.
            // In production use OpenTelemetry's trace propagation.
            {Key: "trace-id", Value: []byte(uuid.New().String())},
        },
    }

    // Produce sends the record asynchronously.
    // The third argument is a callback function — called when the
    // record is either successfully delivered or permanently failed.
    //
    // This is a KEY difference from confluent-kafka-go:
    //   confluent: Produce(msg, nil) + separate drainDeliveryReports goroutine
    //   franz-go:  Produce(ctx, record, callback) — callback IS the delivery report
    //
    // The callback runs in a franz-go background goroutine.
    // Do not do heavy work inside it — just log and update counters.
    // For synchronous confirmation, use ProduceSync instead (see below).
    p.client.Produce(ctx, record, func(r *kgo.Record, err error) {
        if err != nil {
            // The record was not delivered after all retries and
            // the RecordDeliveryTimeout expired. It is permanently lost.
            // In production: write to dead-letter queue, alert on-call.
            p.failed.Add(1)
            log.Printf("[%s] ❌ DELIVERY FAILED topic=%s key=%s: %v",
                p.serviceName, r.Topic, string(r.Key), err)
        } else {
            // The broker (and all in-sync replicas, since acks=all)
            // confirmed receipt. The record is safely stored.
            // Partition and Offset tell you exactly where in the
            // Kafka log this record landed — useful for debugging.
            p.delivered.Add(1)
            log.Printf("[%s] ✅ DELIVERED topic=%s key=%s "+
                "partition=%d offset=%d",
                p.serviceName, r.Topic, string(r.Key),
                r.Partition, r.Offset)
        }
    })

    log.Printf("[%s] ✉️  Queued → topic=%s key=%s size=%d bytes",
        p.serviceName, topic, key, len(data))
    return nil
}

// PublishSync sends a record and WAITS for delivery confirmation
// before returning. Use this when you need to know the outcome
// before proceeding — for example, inside a database transaction
// where you cannot commit until the event is confirmed.
//
// This is the equivalent of Produce + waiting for the callback.
// Under the hood, ProduceSync calls Produce and blocks on a channel
// until the callback fires.
func (p *Producer) PublishSync(ctx context.Context,
    topic, key string, payload interface{}) error {

    data, err := json.Marshal(payload)
    if err != nil {
        return fmt.Errorf("marshal error: %w", err)
    }

    record := &kgo.Record{
        Topic:     topic,
        Key:       []byte(key),
        Value:     data,
        Timestamp: time.Now(),
        Headers: []kgo.RecordHeader{
            {Key: "source-service", Value: []byte(p.serviceName)},
        },
    }

    // ProduceSync blocks until the record is delivered or fails.
    // It returns a kgo.ProduceResults — one per record (you can
    // pass multiple records to ProduceSync in one call).
    results := p.client.ProduceSync(ctx, record)

    // Check if delivery succeeded for our single record.
    // results.FirstErr() returns the first non-nil error,
    // or nil if everything succeeded.
    if err := results.FirstErr(); err != nil {
        return fmt.Errorf("[%s] sync delivery failed for %s: %w",
            p.serviceName, topic, err)
    }

    // The record in results[0].Record now has Partition and Offset
    // filled in by the broker — you know exactly where it landed.
    delivered := results[0].Record
    log.Printf("[%s] ✅ SYNC DELIVERED topic=%s key=%s "+
        "partition=%d offset=%d",
        p.serviceName, delivered.Topic, string(delivered.Key),
        delivered.Partition, delivered.Offset)
    return nil
}

// ─── Typed event helpers ──────────────────────────────────────────

// OrderItem represents a single line item in an order.
type OrderItem struct {
    ProductID string  `json:"product_id"`
    Name      string  `json:"name"`
    Quantity  int     `json:"quantity"`
    Price     float64 `json:"price"`
}

// OrderPlacedEvent is the full payload for the "order.placed" topic.
type OrderPlacedEvent struct {
    BaseEvent
    OrderID    string      `json:"order_id"`
    UserID     string      `json:"user_id"`
    Items      []OrderItem `json:"items"`
    TotalPrice float64     `json:"total_price"`
}

// PublishOrderPlaced is a typed wrapper around Publish.
// It constructs the envelope, generates a unique EventID, and
// calls Publish — so service code never constructs event structs
// manually. This makes it impossible to forget required fields.
func (p *Producer) PublishOrderPlaced(ctx context.Context,
    orderID, userID string, items []OrderItem, total float64) error {

    event := OrderPlacedEvent{
        BaseEvent: BaseEvent{
            // uuid.New().String() generates a random UUID v4.
            // Each event gets a globally unique ID — consumers use
            // this to detect duplicate deliveries (at-least-once
            // delivery means the same message can arrive twice after
            // a crash+restart cycle).
            EventID:   uuid.New().String(),
            EventType: "ORDER_PLACED",
            Timestamp: time.Now(),
            Version:   "1.0",
        },
        OrderID:    orderID,
        UserID:     userID,
        Items:      items,
        TotalPrice: total,
    }

    // Use orderID as the key. This ensures ALL events for order-8821
    // — placed, confirmed, shipped, delivered — land in the same
    // partition and are read in the exact order they were written.
    return p.Publish(ctx, "order.placed", orderID, event)
}

// Stats returns delivery counters — useful for health checks and metrics.
func (p *Producer) Stats() (delivered, failed int64) {
    return p.delivered.Load(), p.failed.Load()
}

// Flush waits for all buffered records to be sent and their
// callbacks to complete. Call this before Close() to ensure
// no messages are silently discarded during shutdown.
func (p *Producer) Flush(ctx context.Context) error {
    return p.client.Flush(ctx)
}

// Close flushes and closes the producer client.
// Always defer this in main() after calling NewProducer.
func (p *Producer) Close() {
    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()
    if err := p.client.Flush(ctx); err != nil {
        log.Printf("[%s] Flush error on close: %v", p.serviceName, err)
    }
    p.client.Close()
    d, f := p.Stats()
    log.Printf("[%s] Producer closed. Delivered: %d  Failed: %d",
        p.serviceName, d, f)
}
```

## Part 4 — Group Consumer — Kafka Assigns Partitions (`consumer/group.go`)

```go
package consumer

import (
    "context"
    "encoding/json"
    "fmt"
    "log"
    "sync"
    "time"

    "github.com/twmb/franz-go/pkg/kgo"
)

// MessageHandler is the function type each service implements.
// Return nil → commit the offset (message processed successfully).
// Return error → do NOT commit (message will be redelivered).
type MessageHandler func(topic, key string, value []byte) error

// GroupConsumer reads from topics using Kafka's group coordination.
// Kafka assigns partitions based on group_id and group membership.
// This is the correct choice for all microservice consumers.
//
// franz-go's key advantage here: it defaults to COOPERATIVE
// INCREMENTAL REBALANCING (KIP-429). This means during a rebalance,
// only the partitions that need to MOVE are briefly paused. All
// other partitions keep flowing. There is no stop-the-world pause
// like in traditional Kafka consumers. This is a major improvement
// over confluent-kafka-go's default eager rebalancing.
type GroupConsumer struct {
    client      *kgo.Client
    serviceName string
    handlers    map[string]MessageHandler
    mu          sync.RWMutex // protects handlers map
}

// NewGroupConsumer creates a kgo.Client configured for group consumption.
func NewGroupConsumer(brokers []string, groupID,
    serviceName string, topics ...string) (*GroupConsumer, error) {

    gc := &GroupConsumer{
        serviceName: serviceName,
        handlers:    make(map[string]MessageHandler),
    }

    client, err := kgo.NewClient(
        // ── CONNECTION ────────────────────────────────────────────
        kgo.SeedBrokers(brokers...),
        kgo.ClientID(serviceName+"-consumer"),

        // ── GROUP IDENTITY ────────────────────────────────────────
        // kgo.ConsumerGroup sets the group.id.
        // THIS is the fundamental choice that controls whether
        // consumers share work or each get all messages:
        //
        //   kgo.ConsumerGroup("inv-svc")  → all instances sharing this
        //   group_id form one team. Kafka splits partitions among them.
        //   Each partition goes to exactly ONE instance. No duplicates.
        //   Maximum parallelism = number of partitions.
        //
        //   If each service uses a DIFFERENT group_id string, each
        //   gets an independent full copy of all messages, with its
        //   own offset bookmark that never affects the others.
        kgo.ConsumerGroup(groupID),

        // ── TOPICS TO SUBSCRIBE ───────────────────────────────────
        // kgo.ConsumeTopics registers which topics this consumer
        // wants to read from. Kafka will assign partitions from ALL
        // listed topics to this consumer as part of the group
        // balancing algorithm.
        //
        // You are NOT picking specific partitions here — you are
        // expressing interest in topics. Kafka decides which
        // partitions you get. This is the Subscribe() pattern.
        kgo.ConsumeTopics(topics...),

        // ── OFFSET RESET ──────────────────────────────────────────
        // kgo.ConsumeResetOffset controls what happens when the
        // consumer group has NO committed offset for a partition.
        // (First time reading a topic, or after retention expires.)
        //
        // kgo.NewOffset().AtStart() → read from the oldest available
        //   message. Equivalent to auto.offset.reset=earliest.
        //   Use when you need to process historical events.
        //
        // kgo.NewOffset().AtEnd()   → read only new messages from now.
        //   Equivalent to auto.offset.reset=latest.
        //   Use when old messages are irrelevant or harmful to process.
        //
        // Remember: this setting is ONLY consulted when NO committed
        // offset exists. If the group already has a committed offset,
        // a new instance joining the group inherits it and resumes
        // from there — this setting is not consulted again.
        // (This directly answers your Q3 from the previous guide.)
        kgo.ConsumeResetOffset(kgo.NewOffset().AtStart()),

        // ── OFFSET COMMIT MODE ────────────────────────────────────
        // By default, franz-go AUTO-COMMITS every 5 seconds.
        // This is convenient but dangerous: if your handler processes
        // a message and then the process crashes before the 5-second
        // timer fires, the offset is NOT committed. On restart, the
        // message is redelivered — which is fine (at-least-once).
        //
        // BUT the reverse is also true: if auto-commit fires BEFORE
        // your handler finishes, and then the process crashes, the
        // offset IS committed even though processing did not complete.
        // The message is then skipped on restart — silent data loss.
        //
        // For most microservices, the safest pattern is:
        // kgo.DisableAutoCommit() + manual CommitRecords after each
        // successful handler call. This gives you true at-least-once:
        // the offset only advances AFTER your business logic succeeds.
        kgo.DisableAutoCommit(),

        // ── REBALANCE CALLBACKS ───────────────────────────────────
        // kgo.OnPartitionsRevoked is called when Kafka takes partitions
        // AWAY from this consumer (rebalance, shutdown, new member joins).
        //
        // WHY this matters with DisableAutoCommit:
        // When a partition is revoked, the consumer stops receiving
        // messages from it. If you have processed messages but not
        // yet committed their offsets, those messages will be
        // redelivered to the new owner of that partition. They will
        // be processed TWICE. For idempotent handlers this is fine.
        // For non-idempotent handlers (e.g. sending an email), it
        // causes duplicates.
        //
        // The solution: commit your uncommitted offsets in this
        // callback BEFORE returning. This is the idiomatic franz-go
        // pattern for safe at-least-once processing.
        kgo.OnPartitionsRevoked(func(ctx context.Context,
            client *kgo.Client, assigned map[string][]int32) {
            // CommitUncommittedOffsets commits all offsets that have
            // been marked but not yet persisted to __consumer_offsets.
            // This is a blocking call — it waits for confirmation.
            // We must complete before returning from this callback,
            // because returning signals to Kafka that we are ready
            // for the partition to be reassigned.
            log.Printf("[%s] Partitions being revoked: %v — committing offsets",
                serviceName, assigned)
            if err := client.CommitUncommittedOffsets(ctx); err != nil {
                log.Printf("[%s] Commit on revoke failed: %v",
                    serviceName, err)
            }
        }),

        // kgo.OnPartitionsAssigned is called when Kafka GIVES partitions
        // to this consumer (on startup, rebalance, or when we take over
        // from a crashed instance).
        // You can use this to initialise per-partition state — for
        // example, loading a cache for the data in those partitions.
        kgo.OnPartitionsAssigned(func(ctx context.Context,
            client *kgo.Client, assigned map[string][]int32) {
            log.Printf("[%s] Partitions assigned: %v",
                serviceName, assigned)
        }),

        // ── HEARTBEAT TUNING ──────────────────────────────────────
        // kgo.SessionTimeout sets the coordinator's dead-consumer
        // detection window. If the coordinator does not hear from
        // this client within this duration, it triggers a rebalance.
        // 45 seconds is the standard production value.
        kgo.SessionTimeout(45 * time.Second),

        // kgo.HeartbeatInterval sets how often the library sends
        // a heartbeat to the coordinator. Must be less than
        // SessionTimeout / 3. 3 seconds is the standard.
        kgo.HeartbeatInterval(3 * time.Second),

        // kgo.RebalanceTimeout sets the maximum time allowed for
        // all group members to re-join after a rebalance is triggered.
        // If a member does not re-join within this window, it is
        // removed from the group. 60 seconds covers most scenarios.
        kgo.RebalanceTimeout(60 * time.Second),
    )
    if err != nil {
        return nil, fmt.Errorf("[%s] failed to create consumer: %w",
            serviceName, err)
    }

    // Verify connectivity before declaring the consumer ready.
    pingCtx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()
    if err := client.Ping(pingCtx); err != nil {
        client.Close()
        return nil, fmt.Errorf("[%s] Kafka unreachable: %w", serviceName, err)
    }

    gc.client = client
    log.Printf("[%s] Consumer ready. group=%s topics=%v",
        serviceName, groupID, topics)
    return gc, nil
}

// RegisterHandler maps a topic name to a handler function.
// Call this for each topic before calling Start().
func (gc *GroupConsumer) RegisterHandler(topic string, h MessageHandler) {
    gc.mu.Lock()
    defer gc.mu.Unlock()
    gc.handlers[topic] = h
    log.Printf("[%s] Handler registered for topic: %s", gc.serviceName, topic)
}

// Start begins the poll loop. It blocks until ctx is cancelled.
// Run this in a goroutine from main():
//   go consumer.Start(ctx)
//
// THE POLL LOOP — how franz-go differs from confluent:
//
// confluent: ReadMessage() returns ONE message at a time.
//   You process it, commit it, call ReadMessage again.
//
// franz-go: PollFetches() returns ALL available messages in a batch
//   called a Fetches. You iterate through the batch, process each
//   record, and commit when done. This is more efficient because
//   one network round trip returns many messages.
func (gc *GroupConsumer) Start(ctx context.Context) error {
    log.Printf("[%s] 👂 Poll loop started", gc.serviceName)

    for {
        // PollFetches asks Kafka for the next batch of records from
        // all assigned partitions. It blocks until:
        //   a) Records are available from any partition, OR
        //   b) The context is cancelled
        //
        // When ctx is cancelled, PollFetches drains any buffered
        // records and returns a final Fetches that may be non-empty.
        // We check IsClientClosed() below to detect this.
        //
        // Unlike confluent's 300ms poll timeout loop, franz-go's
        // PollFetches uses the context for cancellation. There is no
        // busy-wait polling — the goroutine sleeps until data arrives.
        fetches := gc.client.PollFetches(ctx)

        // IsClientClosed() returns true when the context was cancelled
        // AND the client has been closed. This is the clean exit signal.
        if fetches.IsClientClosed() {
            log.Printf("[%s] Client closed, poll loop exiting", gc.serviceName)
            return nil
        }

        // Check for fetch-level errors. These are non-retriable errors
        // (broker went down, authentication failed, partition was deleted)
        // that franz-go could not recover from internally.
        if errs := fetches.Errors(); len(errs) > 0 {
            for _, e := range errs {
                log.Printf("[%s] ⚠️  Fetch error topic=%s partition=%d: %v",
                    gc.serviceName, e.Topic, e.Partition, e.Err)
                // Non-fatal: log and continue. The library will
                // attempt to reconnect and re-fetch automatically.
            }
        }

        // Iterate through every record in this batch.
        // fetches.EachRecord is the idiomatic way to iterate — it
        // handles the nested structure (fetches → topic → partition
        // → records) and presents each record one at a time.
        fetches.EachRecord(func(record *kgo.Record) {
            gc.mu.RLock()
            handler, exists := gc.handlers[record.Topic]
            gc.mu.RUnlock()

            if !exists {
                log.Printf("[%s] No handler for topic %s — skipping",
                    gc.serviceName, record.Topic)
                // Mark this record so its offset gets committed.
                // We mark even records we skip — otherwise we would
                // be stuck replaying them forever.
                gc.client.MarkCommitRecords(record)
                return
            }

            log.Printf("[%s] 📨 Record topic=%s key=%s "+
                "partition=%d offset=%d",
                gc.serviceName, record.Topic, string(record.Key),
                record.Partition, record.Offset)

            // Call the business logic handler.
            if err := handler(
                record.Topic,
                string(record.Key),
                record.Value,
            ); err != nil {
                // Handler failed. We do NOT mark this record for commit.
                // Since franz-go uses auto-advance-on-commit semantics,
                // not marking this record means its offset will not be
                // committed in the next CommitMarkedOffsets call.
                // On the next poll (or after restart), this record will
                // be redelivered. This is at-least-once delivery.
                //
                // In production: track failure counts per record.
                // After N failures, move to a dead-letter topic instead
                // of blocking the partition indefinitely.
                log.Printf("[%s] ❌ Handler error topic=%s key=%s: %v "+
                    "— will be redelivered",
                    gc.serviceName, record.Topic, string(record.Key), err)
                return
            }

            // Handler succeeded. Mark this record's offset for commit.
            // kgo.MarkCommitRecords does NOT commit immediately —
            // it marks the record as "ready to commit". The actual
            // commit happens in CommitMarkedOffsets below.
            //
            // WHY the two-step mark-then-commit approach?
            // It allows us to process a whole batch of records and
            // then commit all their offsets in ONE network round trip,
            // rather than one commit per record. Much more efficient.
            gc.client.MarkCommitRecords(record)
            log.Printf("[%s] ✅ Processed topic=%s key=%s "+
                "partition=%d offset=%d",
                gc.serviceName, record.Topic, string(record.Key),
                record.Partition, record.Offset)
        })

        // Commit all marked offsets to Kafka's __consumer_offsets topic.
        // This advances the group's bookmark for every record we marked.
        // On the next restart or rebalance, consumers resume from
        // the committed position, not from the beginning.
        //
        // CommitMarkedOffsets is non-blocking — it schedules the commit.
        // To block until the commit completes, use CommitUncommittedOffsets.
        if err := gc.client.CommitMarkedOffsets(ctx); err != nil {
            log.Printf("[%s] Commit error: %v", gc.serviceName, err)
            // Non-fatal: the marks are still tracked. The next
            // CommitMarkedOffsets call will retry them.
        }
    }
}

// Close cleanly shuts down the consumer.
// This triggers OnPartitionsRevoked (which commits uncommitted
// offsets), then sends a LeaveGroup request to the coordinator
// (which triggers an immediate rebalance instead of waiting 45s
// for the session timeout to expire), then closes the connection.
func (gc *GroupConsumer) Close() {
    gc.client.Close()
    log.Printf("[%s] Consumer closed", gc.serviceName)
}

// Unmarshal is a helper for handler functions to deserialise
// raw JSON bytes into a typed event struct.
// Usage:
//   var evt OrderPlacedEvent
//   if err := consumer.Unmarshal(value, &evt); err != nil { return err }
func Unmarshal(data []byte, target interface{}) error {
    return json.Unmarshal(data, target)
}
```

## Part 5 — Direct Partition Consumer (`consumer/direct.go`)

```go
package consumer

import (
    "context"
    "fmt"
    "log"
    "time"

    "github.com/twmb/franz-go/pkg/kgo"
)

// DirectConsumer reads from SPECIFIC partitions at SPECIFIC offsets,
// bypassing the group coordination system entirely.
// Use for: replaying history, debugging, data migration, one-off jobs.
// Do NOT use for normal microservice consumers — use GroupConsumer.
type DirectConsumer struct {
    client      *kgo.Client
    serviceName string
}

// NewDirectConsumer creates a consumer that bypasses group coordination.
// Notice we do NOT pass kgo.ConsumerGroup() — without that option,
// franz-go creates a "direct" consumer that is not part of any group.
// This maps to confluent's Assign() pattern.
func NewDirectConsumer(brokers []string,
    serviceName string) (*DirectConsumer, error) {

    client, err := kgo.NewClient(
        kgo.SeedBrokers(brokers...),
        kgo.ClientID(serviceName + "-direct"),
        // No kgo.ConsumerGroup() → no group coordination.
        // No rebalancing. No automatic partition reassignment.
        // You own exactly the partitions you explicitly assign.
    )
    if err != nil {
        return nil, fmt.Errorf("failed to create direct consumer: %w", err)
    }

    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()
    if err := client.Ping(ctx); err != nil {
        client.Close()
        return nil, fmt.Errorf("Kafka unreachable: %w", err)
    }

    return &DirectConsumer{client: client, serviceName: serviceName}, nil
}

// ReadFromPartition reads from one specific partition starting at a
// specific offset. This directly answers "can I read from a specific
// partition?" — yes, this is exactly how.
//
// Parameters:
//   topicName   — the topic to read from
//   partition   — exact partition number: 0, 1, 2, ...
//   startOffset — where to start:
//                   kgo.NewOffset().AtStart()     → oldest available
//                   kgo.NewOffset().AtEnd()        → only new messages
//                   kgo.NewOffset().At(42)         → specific offset 42
//                   kgo.NewOffset().Relative(-100) → 100 before end
//   maxMessages — stop after this many messages (-1 = read forever)
//   handler     — called for each record
func (dc *DirectConsumer) ReadFromPartition(
    ctx context.Context,
    topicName string,
    partition int32,
    startOffset kgo.Offset,
    maxMessages int,
    handler MessageHandler,
) error {

    // AddConsumePartitions tells the client to read from exactly
    // these partitions at these offsets. Unlike ConsumeTopics (which
    // subscribes to all partitions of a topic via group coordination),
    // this directly assigns specific partition+offset combinations.
    //
    // You can pass multiple topics and partitions in one call:
    //   map[string]map[int32]kgo.Offset{
    //     "order.placed": {0: startOffset, 2: startOffset},
    //     "user.registered": {1: startOffset},
    //   }
    dc.client.AddConsumePartitions(map[string]map[int32]kgo.Offset{
        topicName: {partition: startOffset},
    })

    log.Printf("[%s] Reading %s partition %d from offset %v",
        dc.serviceName, topicName, partition, startOffset)

    count := 0
    for {
        if maxMessages > 0 && count >= maxMessages {
            log.Printf("[%s] Reached maxMessages=%d — stopping",
                dc.serviceName, maxMessages)
            return nil
        }

        select {
        case <-ctx.Done():
            log.Printf("[%s] Context cancelled after %d messages",
                dc.serviceName, count)
            return nil
        default:
        }

        fetches := dc.client.PollFetches(ctx)
        if fetches.IsClientClosed() {
            return nil
        }

        fetches.EachRecord(func(record *kgo.Record) {
            if err := handler(
                record.Topic,
                string(record.Key),
                record.Value,
            ); err != nil {
                log.Printf("[%s] Handler error at offset %d: %v",
                    dc.serviceName, record.Offset, err)
            }
            count++
        })
    }
}

// SeekAndRead demonstrates kgo's offset-seeking capability.
// After assigning a partition, you can Seek to reposition the
// read cursor to any specific offset within that partition.
//
// Use case: you normally run as a group consumer, but you need
// to re-process messages from 2 hours ago for a specific partition.
func (dc *DirectConsumer) SeekAndRead(
    ctx context.Context,
    topicName string,
    partition int32,
    seekToOffset int64,
    handler MessageHandler,
) error {
    // First, assign the partition starting at the beginning.
    dc.client.AddConsumePartitions(map[string]map[int32]kgo.Offset{
        topicName: {partition: kgo.NewOffset().AtStart()},
    })

    // Now seek to the desired offset. This is a local operation —
    // it repositions the client's internal read cursor without
    // any network call. The next PollFetches will request from
    // this exact offset.
    dc.client.SetOffsets(map[string]map[int32]kgo.EpochOffset{
        topicName: {
            partition: {
                Offset: seekToOffset,
                Epoch:  -1, // -1 means "any epoch" (don't validate)
            },
        },
    })

    log.Printf("[%s] Seeked to offset %d on %s[%d]",
        dc.serviceName, seekToOffset, topicName, partition)

    for {
        select {
        case <-ctx.Done():
            return nil
        default:
        }

        fetches := dc.client.PollFetches(ctx)
        if fetches.IsClientClosed() {
            return nil
        }
        fetches.EachRecord(func(record *kgo.Record) {
            if err := handler(
                record.Topic, string(record.Key), record.Value,
            ); err != nil {
                log.Printf("handler error at offset %d: %v",
                    record.Offset, err)
            }
        })
    }
}

func (dc *DirectConsumer) Close() {
    dc.client.Close()
    log.Printf("[%s] Direct consumer closed", dc.serviceName)
}
```

```go
package main

import (

func main() {
    brokers := []string{getEnv("KAFKA_BROKERS", "localhost:9092")}
    groupID := getEnv("KAFKA_GROUP_ID", "order-service")

    // ── STEP 1: CREATE TOPICS ─────────────────────────────────────
    admin, err := topic.NewAdminClient(brokers)
    if err != nil {
        log.Fatalf("Admin client failed: %v", err)
    }
    defer admin.Close()

    err = admin.EnsureTopics([]topic.TopicSpec{
        {
            Name:              config.TopicOrderPlaced,
            NumPartitions:     6,   // up to 6 consumers in parallel
            ReplicationFactor: 1,   // 1 for dev, 3 for production
            Configs: map[string]*string{
                "retention.ms":    topic.StringPtr("604800000"), // 7 days
                "cleanup.policy":  topic.StringPtr("delete"),
            },
        },
        {
            Name:              config.TopicInventoryReserved,
            NumPartitions:     6,
            ReplicationFactor: 1,
        },
        {
            Name:              config.TopicNotification,
            NumPartitions:     3,
            ReplicationFactor: 1,
        },
    })
    if err != nil {
        log.Fatalf("Topic creation failed: %v", err)
    }

    admin.DescribeTopics(config.TopicOrderPlaced)

    // ── STEP 2: CREATE PRODUCER ───────────────────────────────────
    prod, err := producer.NewProducer(brokers, "order-service")
    if err != nil {
        log.Fatalf("Producer creation failed: %v", err)
    }
    defer prod.Close()

    // ── STEP 3: CREATE CONSUMER ───────────────────────────────────
    cons, err := consumer.NewGroupConsumer(
        brokers,
        groupID,
        "order-service",
        config.TopicInventoryReserved, // topics to subscribe to
        config.TopicNotification,
    )
    if err != nil {
        log.Fatalf("Consumer creation failed: %v", err)
    }
    defer cons.Close()

    cons.RegisterHandler(config.TopicInventoryReserved, handleInventoryReserved)
    cons.RegisterHandler(config.TopicNotification, handleNotification)

    // ── STEP 4: RUN ───────────────────────────────────────────────
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    go func() {
        if err := cons.Start(ctx); err != nil {
            log.Printf("Consumer stopped: %v", err)
        }
    }()

    // Publish test events to see the full flow.
    go func() {
        time.Sleep(2 * time.Second) // wait for consumer to be ready
        for i := 1; i <= 5; i++ {
            orderID := fmt.Sprintf("ord-%04d", i)
            if err := prod.PublishOrderPlaced(ctx,
                orderID, "user-99",
                []producer.OrderItem{
                    {ProductID: "prod-001", Name: "Keyboard",
                        Quantity: 1, Price: 129.99},
                },
                129.99,
            ); err != nil {
                log.Printf("Publish failed: %v", err)
            }
            time.Sleep(500 * time.Millisecond)
        }
    }()

    // Graceful shutdown on Ctrl+C or docker stop
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit
    log.Println("Shutdown signal received")
    cancel()
}
```

## franz-go vs confluent-kafka-go — Side by Side

```
  ┌──────────────────────────┬────────────────────────┬───────────────────────┐
  │  Feature                 │  confluent-kafka-go    │  franz-go             │
  ├──────────────────────────┼────────────────────────┼───────────────────────┤
  │  Pure Go (no CGO)        │  ❌ requires librdkafka │  ✅ 100% pure Go      │
```
